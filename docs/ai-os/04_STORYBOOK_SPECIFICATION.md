# 04 — Storybook Specification

> **Storybook is the ONLY source of truth and is IMMUTABLE for visual design.** Figma is a
> faithful, pixel-perfect reproduction of Storybook, never a redesign. Every story is a *view*
> of a component's `*.meta.ts`; nothing visual is authored twice. If Storybook and Figma
> disagree, Storybook wins and the Figma bundle is regenerated — never the reverse.

Cross-references: [00_MASTER_RULES.md](./00_MASTER_RULES.md) ·
[03_DESIGN_SYSTEM.md](./03_DESIGN_SYSTEM.md) · [05_FIGMA_SPECIFICATION.md](./05_FIGMA_SPECIFICATION.md) ·
[07_COMPONENT_SPECIFICATION.md](./07_COMPONENT_SPECIFICATION.md) ·

## 1. Purpose

Storybook is the human- and AI-facing rendering of the TDS design system: 60 components
(24 atoms, 27 molecules, 9 organisms) plus Foundations pages, each rendered live in light/dark
themes and four font families. This document is the **complete rulebook** for authoring those
stories so that:

- Every story is **derived from metadata** (`X.meta.ts` → `argTypesFromMeta` / `argsFromMeta` /
  `metaParameters`), never hand-configured, so controls, defaults and autodocs never drift from
  the component or the Figma manifest.
- Storybook remains the **single, immutable design source** from which
  `src/generated/design-system.manifest.json` and `figma/tds.plugin.json` are generated by
  `npm run ds:build` and reproduced by the Figma plugin.
- Any AI agent can add or edit a story deterministically, with zero visual invention, and prove
  compliance via `npm run lint`, `tsc -b`, and `npm run storybook`.

## 2. Responsibilities

This specification owns:

1. **Story authoring contract** — the exact CSF3 shape every `*.stories.tsx` file MUST follow.
2. **Metadata bridge rules** — how `metaParameters(meta)`, `argTypesFromMeta(meta)` and
   `argsFromMeta(meta)` from `@core/storybook` (`src/core/storybook.ts`) are wired into every story.
3. **Global configuration** — the meaning and immutability of `.storybook/main.ts` (stories glob,
   addons, aliases) and `.storybook/preview.tsx` (theme/font globals, viewports, `storySort`,
   controls matchers, a11y).
4. **Naming, folder, import/export, and typing rules** for stories and MDX.
5. **Plugin mapping** — how each story surface (controls, variant axes, states) corresponds to the
   Figma bundle, so stories stay legible to `@figma-plugin`.
6. **Validation, checklists, and failure recovery** for the Storybook layer.

It does **not** own component internals ([07_COMPONENT_SPECIFICATION.md](./07_COMPONENT_SPECIFICATION.md)),
token definitions ([08_TOKEN_SPECIFICATION.md](./08_TOKEN_SPECIFICATION.md)), the Figma bundle contract
([05_FIGMA_SPECIFICATION.md](./05_FIGMA_SPECIFICATION.md)), or the plugin algorithm
([06_PLUGIN_SPECIFICATION.md](./06_PLUGIN_SPECIFICATION.md)).

## 3. Scope

**In scope**

- All files matched by the stories glob in `.storybook/main.ts`:
  `../src/**/*.mdx` and `../src/**/*.stories.@(ts|tsx)`.
- Component stories under `src/components/{atoms,molecules,organisms}/X/X.stories.tsx`.
- Foundations / example stories under `src/stories/` (Foundations, TypeSystem, FigmaMapping,
  and the Board / Dashboard / List / BottomSheet examples).
- `.storybook/main.ts` and `.storybook/preview.tsx` configuration.
- The `@core/storybook` helpers and their contract with `ComponentMeta`.

**Out of scope**

- Visual/design decisions (locked — see [03_DESIGN_SYSTEM.md](./03_DESIGN_SYSTEM.md)).
- The React component `.tsx`/`.css` implementation ([07_COMPONENT_SPECIFICATION.md](./07_COMPONENT_SPECIFICATION.md)).
- Token authoring and generated CSS ([08_TOKEN_SPECIFICATION.md](./08_TOKEN_SPECIFICATION.md)).
- Any backend/DB/API/auth surface (**PLANNED** — does not exist today; see
  [15_API_GUIDE.md](./15_API_GUIDE.md), [17_SUPABASE_GUIDE.md](./17_SUPABASE_GUIDE.md)).
- A Storybook test-runner / visual-regression / snapshot pipeline (**PLANNED**; see §4.14, §4.15
  and [24_TESTING_GUIDE.md](./24_TESTING_GUIDE.md)).

## 4. Rules

Rules are **MUST/SHOULD/NEVER** and enforceable via lint, typecheck, or a running Storybook.
Each mandated topic is its own subsection.

### 4.1 Folder Rules

- **R-FOLDER-1 (MUST):** A component's story lives beside its source as
  `src/components/<tier>/<Name>/<Name>.stories.tsx`, in the same folder as `<Name>.meta.ts`,
  `<Name>.tsx`, `<Name>.css`, and `index.ts`. One story file per component.
- **R-FOLDER-2 (MUST):** Cross-cutting/narrative stories (Foundations, TypeSystem, FigmaMapping,
  usage Examples) live in `src/stories/`. NEVER put component stories there.
- **R-FOLDER-3 (NEVER):** Do not add stories outside the glob (`src/**`), and NEVER place stories
  under generated dirs (`src/generated/`, `src/tokens/generated/`, `figma/`).

### 4.2 Story Rules

- **R-STORY-1 (MUST):** Every component's story `default export` is a CSF3 `Meta<typeof Component>`
  object; every named export is a `StoryObj<typeof Component>`.
- **R-STORY-2 (MUST):** The first named story is `Playground` — an empty `{}` object that renders
  purely from `args`/`argTypes` so the Controls panel drives the full variant matrix.
- **R-STORY-3 (SHOULD):** Provide dedicated showcase stories for each axis a component exposes:
  `Types` (the `type` A/B/C axis), `Styles` (the `variant`/"Style" axis), `Tones`, `Sizes`,
  `States`, plus composition stories (`WithIcons`, etc.) as the component warrants.
- **R-STORY-4 (MUST):** Showcase `render` functions use inline layout **only via token CSS vars**
  (e.g. `gap: 'var(--tds-space-3)'`). NEVER hardcode px/hex in a render wrapper.

### 4.3 CSF Rules (CSF3)

- **R-CSF-1 (MUST):** Use **Component Story Format 3** exclusively — object stories, no
  `Template.bind({})`, no CSF2 function stories.
- **R-CSF-2 (MUST):** The `meta` object MUST set, in this order: `title`, `component`,
  `tags: ['autodocs']`, `parameters: metaParameters(meta)`, `argTypes: argTypesFromMeta(meta)`,
  `args: { ...argsFromMeta(meta), ...storyDefaults }`.
- **R-CSF-3 (MUST):** Import the three helpers from the `@core/storybook` barrel and the meta from
  the local `./X.meta` module. NEVER inline-duplicate what a helper produces.
- **R-CSF-4 (SHOULD):** Prefer `args` + Controls over bespoke `render` where the default render
  suffices; use `render` only to demonstrate composition or multiple instances at once.

### 4.4 MDX Rules

- **R-MDX-1 (MAY):** `*.mdx` docs pages are matched by the glob and permitted for **narrative
  foundations documentation** (token overviews, mapping explainers). No component `.mdx` exists
  today; component docs come from **autodocs**, not MDX.
- **R-MDX-2 (MUST, when used):** An MDX page MUST use Storybook `Meta`/`Canvas`/`Story` blocks and
  reference existing CSF stories by id — it MUST NOT redefine component props or re-declare design
  values. MDX documents; it never becomes a second source of truth.
- **R-MDX-3 (NEVER):** Do not embed hardcoded design values (hex, px, shadows) in MDX prose that
  contradict tokens; reference `var(--tds-*)` or the token id.

### 4.5 Controls (`argTypesFromMeta`)

- **R-CTRL-1 (MUST):** All controls are generated by `argTypesFromMeta(meta)` in
  `src/core/storybook.ts`. Do not hand-write `argTypes` entries that duplicate meta-derived ones.
- **R-CTRL-2 (derived behavior):** Each `meta.variantProps[]` entry becomes a `select` control with
  `options`, `table.category: 'Variants'`, `defaultValue` = the variant default, and a `type.summary`
  of the option union. Each `meta.componentProps[]` becomes a control by `type`:
  `boolean → boolean`, `number → number`, `select → select`, `text → text`, and
  `instanceSwap → control: false` (rendered in Figma as `INSTANCE_SWAP`, not a Storybook control),
  all under `table.category: 'Properties'`.
- **R-CTRL-3 (SHOULD):** Extra non-meta args a story needs (e.g. `children: 'Button'`) MAY be added
  to `args`, but any prop that is a *design axis* MUST come from meta, never a hand-added control.
- **R-CTRL-4 (config):** The global `controls.matchers` (color/date) and `controls.expanded: true`
  in `.storybook/preview.tsx` are fixed; do not disable expanded controls per story without cause.

### 4.6 Args (`argsFromMeta`)

- **R-ARGS-1 (MUST):** Default args come from `argsFromMeta(meta)`, which seeds every variant axis
  with its `default` and every component prop that declares a `default`. Spread it first, then
  layer story-specific args: `args: { ...argsFromMeta(meta), children: 'Button' }`.
- **R-ARGS-2 (MUST):** Per-story overrides use the story's own `args`, or props passed in `render`
  (`<Button {...args} type="C" />`). NEVER mutate the shared `meta.args`.
- **R-ARGS-3 (MUST):** A story's default arg set MUST render a valid default variant combination —
  the same combination `defaultsFromMeta(meta)` produces and the plugin treats as the base frame.

### 4.7 Decorators

- **R-DEC-1 (MUST):** The global `withTheme` decorator in `.storybook/preview.tsx` is the canonical
  theme/font wrapper. It stamps `data-theme` and `data-font` on `document.documentElement` (so
  portals inherit theme) and wraps the story in a `ThemeFrame` using `var(--tds-color-bg-canvas)`,
  `var(--tds-color-fg-default)`, `var(--tds-space-6)` padding, and `var(--tds-font-family-body)`.
- **R-DEC-2 (SHOULD):** Per-story decorators are allowed only for **layout framing** (e.g. constrain
  width to demo responsive behavior) and MUST use token vars. NEVER add a decorator that changes
  theme, color, or typography — that is the global decorator's job and a Design-Lock boundary.

### 4.8 Parameters (`metaParameters`)

- **R-PARAM-1 (MUST):** `parameters` come from `metaParameters(meta)`, which sets
  `docs.description.component` to `docDescription(meta)` (a generated autodocs blurb listing
  category, responsive note, variant→Figma-property mapping, states, keyboard, and the Figma
  variant count) and, when `meta.a11y.role` exists, scopes the a11y addon to
  `{ element: '[role="…"]' }`.
- **R-PARAM-2 (SHOULD):** Story-local parameters (e.g. `layout`, `viewport.defaultViewport`) may be
  added by merging, but MUST NOT remove or override the autodocs description or the a11y element
  selector produced by `metaParameters`.
- **R-PARAM-3 (config):** Global preview parameters are fixed: `layout: 'fullscreen'`,
  `controls.expanded: true`, `options.storySort.order = ['Foundations','Atoms','Molecules','Organisms']`,
  `a11y: { test: 'todo' }`, four viewports, and `backgrounds: { disable: true }` (theme provides bg).

### 4.9 Variants

- **R-VAR-1 (MUST):** Variant axes shown in Storybook are exactly `meta.variantProps`:
  `type` (layout preset **A/B/C**), `variant` (labelled **"Style"** — solid/outline/ghost/soft/link),
  `tone` (brand/neutral/success/warning/danger), `size` (sm/md/lg), `shape` (rounded/pill/square).
  See [09_VARIANT_SPECIFICATION.md](./09_VARIANT_SPECIFICATION.md).
- **R-VAR-2 (MUST):** A component with a `type` A/B/C axis SHOULD provide a `Types` story
  demonstrating each preset; the `variant` axis SHOULD be a `Styles` story. Do not conflate the two
  — `type` = layout, `variant` = fill.
- **R-VAR-3 (NEVER):** NEVER invent a variant value not in `meta.variantProps[].options`. To add a
  value, edit the `.meta.ts` and run `npm run ds:build`; the control updates automatically.

### 4.10 States

- **R-STATE-1 (MUST):** Interaction states (`default/hover/active/focus/disabled/error/success/
  loading/readonly`, per `meta.states`) are demonstrated via a `States` story that renders instances
  with the relevant props (`disabled`, `loading`, `fullWidth`, …) or CSS `:hover`/`:focus` where
  applicable.
- **R-STATE-2 (policy):** The **state axis is deliberately NOT a Storybook control** and NOT a Figma
  variant (it would explode combinations — e.g. Button 4,050+). `disabled`/`loading` survive as
  BOOLEAN component props; other states are shown by example. See [11_INTERACTION_SPECIFICATION.md](./11_INTERACTION_SPECIFICATION.md).

### 4.11 Responsive

- **R-RESP-1 (MUST):** Responsive behavior is exercised via the four global viewports defined in
  `.storybook/preview.tsx` — `mobile` 375×812, `tablet` 768×1024, `desktop` 1280×800, `wide`
  1440×900. Use the Viewport toolbar; do not hardcode viewport widths in stories.
- **R-RESP-2 (SHOULD):** If a component's `meta.responsive` note is set, the story SHOULD include a
  demonstration (or set `parameters.viewport.defaultViewport`) so the behavior is visible. Layout is
  driven by tokens/`useMediaQuery` — see [12_RESPONSIVE_SPECIFICATION.md](./12_RESPONSIVE_SPECIFICATION.md).

### 4.12 Animations

- **R-ANIM-1 (MUST):** Motion in stories uses **motion tokens** (`var(--tds-*)` duration/easing)
  only; NEVER inline ad-hoc transitions. Respect `prefers-reduced-motion`. See
  [10_ANIMATION_SPECIFICATION.md](./10_ANIMATION_SPECIFICATION.md).
- **R-ANIM-2 (note):** Animation is a Storybook-only concern (Figma cannot reproduce runtime motion);
  it is out of the Figma bundle. Do not treat a missing Figma animation as drift.

### 4.13 Accessibility

- **R-A11Y-1 (MUST):** `@storybook/addon-a11y` is installed and enabled globally (`a11y: { test: 'todo' }`).
  Every component story is subject to axe checks; violations MUST be resolved in the component, not
  suppressed in the story.
- **R-A11Y-2 (MUST):** When `meta.a11y.role` is present, `metaParameters` scopes a11y to that role's
  element; keep the role accurate. Keyboard affordances declared in `meta.a11y.keyboard` MUST match
  the component's actual behavior. See [13_ACCESSIBILITY_SPECIFICATION.md](./13_ACCESSIBILITY_SPECIFICATION.md).

### 4.14 Testing

- **R-TEST-1 (available now):** `@storybook/addon-interactions` is installed. Interactive components
  SHOULD add a CSF3 `play` function to assert focus/keyboard/disclosure behavior; play steps run in
  the Interactions panel.
- **R-TEST-2 (PLANNED):** A Storybook **test-runner** (`@storybook/test-runner`) and CI wiring do
  **not exist today** — there is no `test` npm script and no test framework. Broad interaction/e2e
  coverage is PLANNED; see [24_TESTING_GUIDE.md](./24_TESTING_GUIDE.md). The only headless gate that
  exists today is the Figma plugin harness `figma/plugin/test/harness.ts` (run by `npm run plugin:test`),
  which is a *plugin* coverage gate, not a Storybook test.

### 4.15 Snapshots

- **R-SNAP-1 (PLANNED):** Visual-regression/snapshot tooling (Chromatic, storyshots, Playwright
  screenshots) is **not configured**. When added, snapshots MUST be captured per theme (light/dark)
  and per relevant viewport. Until then, "no visual drift" is enforced by review against Storybook,
  not by automated snapshots.

### 4.16 Naming

- **R-NAME-1 (MUST):** Story `title` is `"<Tier>/<Name>"` where Tier ∈ `Atoms | Molecules |
  Organisms` (e.g. `'Atoms/Button'`), or `"Foundations/<Topic>"` for `src/stories/` pages
  (e.g. `'Foundations/Design Tokens'`). Tiers MUST match the `storySort` order.
- **R-NAME-2 (MUST):** Named exports are `PascalCase`, action-descriptive (`Playground`, `Types`,
  `Styles`, `Tones`, `Sizes`, `States`, `WithIcons`). The `default export` is the `meta` object.
- **R-NAME-3 (MUST):** Story `title` component name MUST equal the component's `meta.name` /
  folder name. NEVER rename in the title.

### 4.17 Imports

- **R-IMPORT-1 (MUST):** Import the component from its **local module** in its own story
  (`import { Button } from './Button'`) and its meta from `./X.meta`. Import cross-component
  helpers used inside a render (e.g. `Icon`) from the **single barrel `@/components`**, never a deep
  path. Import Storybook helpers from `@core/storybook`. See [19_CODE_CONVENTION.md](./19_CODE_CONVENTION.md).
- **R-IMPORT-2 (MUST):** Use `import type { Meta, StoryObj } from '@storybook/react'` (type-only)
  per `verbatimModuleSyntax`/`consistent-type-imports`. Aliases available: `@/*`, `@core/*`,
  `@components/*`, `@tokens/*` (declared in `tsconfig.*` and `.storybook/main.ts` `viteFinal`).

### 4.18 Exports

- **R-EXPORT-1 (MUST):** Exactly one `export default meta`; every other export is a named
  `StoryObj`. No side-effect exports, no re-exports of the component from a story file.

### 4.19 Types

- **R-TYPE-1 (MUST):** `Meta<typeof Component>` and `StoryObj<typeof Component>` are the required
  generics; `type Story = StoryObj<typeof Component>` is the local alias. Foundations pages that
  render no single component use bare `Meta`/`StoryObj`.
- **R-TYPE-2 (MUST):** `render` args are typed by the story generic; do not cast to `any`. Const
  option arrays use `as const` (e.g. `(['sm','md','lg'] as const)`).

### 4.20 Documentation (autodocs)

- **R-DOC-1 (MUST):** `tags: ['autodocs']` on every component `meta` generates the Docs page from
  `docDescription(meta)`. NEVER write a parallel prose description that can drift from the meta.
- **R-DOC-2 (SHOULD):** Add a JSDoc comment above a showcase story (e.g. `/** Type A/B/C — content
  layout presets. */`) — Storybook surfaces it as the story's description. Keep it factual, no design
  claims. See [33_DOCUMENTATION_GUIDE.md](./33_DOCUMENTATION_GUIDE.md).

### 4.21 Design Tokens

- **R-TOKEN-1 (MUST):** Every color/space/radius/shadow/typography value referenced in a story
  (render wrappers, decorators, Foundations swatches) MUST be a `var(--tds-*)` token. NEVER hardcode.
  Token id dot-notation maps to `--tds-` dash CSS vars (`color.bg.canvas` → `--tds-color-bg-canvas`).
- **R-TOKEN-2 (MUST):** Foundations token stories read from the token registry (`allCollections`,
  `cssVar`, `resolveValue` from `@/tokens`) — they render the generated tokens, they do not restate
  values. See [08_TOKEN_SPECIFICATION.md](./08_TOKEN_SPECIFICATION.md).

### 4.22 Composition

- **R-COMP-1 (MUST):** Compound components (Tabs, Accordion, Card, Sidebar, Menu, …) are storied via
  their sub-parts assembled in `render`, importing sub-components from `@/components`. Demonstrate the
  canonical composition, not a redesigned one.
- **R-COMP-2 (SHOULD):** `src/stories/` example pages (Board, Dashboard, List, BottomSheet) compose
  many components to show real usage; they MUST reuse existing components only (reuse-first) — see
  [docs/COMPONENTS.md](../COMPONENTS.md).

### 4.23 Performance

- **R-PERF-1 (MUST):** Keep render functions cheap and static — no data fetching, timers, or heavy
  loops. Zero-runtime CSS (token vars) keeps stories fast. NEVER import backend/API modules
  (none exist; PLANNED) into stories.
- **R-PERF-2 (SHOULD):** Prefer `Playground` + Controls over dozens of hardcoded permutations; the
  Controls matrix covers combinations without extra story weight. See [23_PERFORMANCE_GUIDE.md](./23_PERFORMANCE_GUIDE.md).

### 4.24 Plugin Mapping

- **R-MAP-1 (MUST):** What a story exposes MUST correspond to what `npm run ds:build` emits into
  `figma/tds.plugin.json`: `meta.variantProps` → Figma **VARIANT** properties (the axes in your
  `Types`/`Styles`/`Tones`/`Sizes` stories); `meta.componentProps` → **BOOLEAN/TEXT/INSTANCE_SWAP**
  component properties (the boolean/text controls); the **state axis** stays a demonstration, not a
  Figma variant. A component with a `type` A/B/C axis is split into one Component Set per preset.
- **R-MAP-2 (MUST):** Storybook is authoritative; if a story shows a combination Figma lacks, fix the
  source (`.meta.ts`/tokens) and regenerate — NEVER hand-edit `figma/tds.plugin.json` or
  `src/generated/design-system.manifest.json`. See [05_FIGMA_SPECIFICATION.md](./05_FIGMA_SPECIFICATION.md),
  [06_PLUGIN_SPECIFICATION.md](./06_PLUGIN_SPECIFICATION.md).

## 5. Workflow

### 5.1 Author or edit a component story

1. **Read metadata first** — open `X.meta.ts` (or the catalog [docs/COMPONENTS.md](../COMPONENTS.md))
   to learn the axes/props. Do not read the whole `.tsx` unless composition details are needed.
2. **Scaffold CSF3** — create/edit `X.stories.tsx` following the Template (§11): `Meta<typeof X>`
   with `title`, `component`, `tags`, and the three helper wires.
3. **Add `Playground`** (`export const Playground: Story = {};`) then axis showcases
   (`Types`, `Styles`, `Tones`, `Sizes`, `States`, composition) using token-var layout wrappers.
4. **Add interactions (SHOULD)** — for interactive components add a `play` function
   (`@storybook/addon-interactions`) asserting keyboard/focus/disclosure.
5. **Run Storybook** — `npm run storybook` (dev on 6006). Toggle Theme (light/dark) and Font
   toolbars and the four viewports; confirm the a11y panel is clean.
6. **Regenerate outputs** — if you changed the meta, run `npm run ds:build` so
   `figma/tds.plugin.json`, `src/generated/design-system.manifest.json` and
   `docs/COMPONENTS.md` stay in sync; then `npm run figma:build` to typecheck/build/verify the plugin.
7. **Validate** — `npm run lint` and `tsc -b` (via `npm run build`) MUST pass. Update the relevant
   `.ai/` manifests ([.ai/COMPONENT_INDEX.json](../../.ai/COMPONENT_INDEX.json),
   [.ai/DESIGN_MANIFEST.json](../../.ai/DESIGN_MANIFEST.json)).

### 5.2 Author a Foundations / example page

1. Create the story under `src/stories/` with `title: 'Foundations/<Topic>'` (or an Examples title).
2. Read from the token registry / `@/components` barrel only; never restate design values.
3. Set `parameters.layout` and `options.showPanel` as the Foundations pages do; keep within `storySort`.

### 5.3 Global config change (rare, gated)

Edits to `.storybook/main.ts` (glob, addons, aliases) or `.storybook/preview.tsx` (globals,
viewports, storySort, controls, a11y) are **infrastructure changes** — they require explicit approval
and a review gate ([30_REVIEW_SYSTEM.md](./30_REVIEW_SYSTEM.md)); they MUST NOT alter locked design
(theme colors are token-driven, not config).

## 6. Examples

### 6.1 Canonical component story (real — `src/components/atoms/Button/Button.stories.tsx`)

```tsx
import type { Meta, StoryObj } from '@storybook/react';
import { Button } from './Button';
import { buttonMeta } from './Button.meta';
import { argTypesFromMeta, argsFromMeta, metaParameters } from '@core/storybook';
import { Icon } from '../Icon';

const meta: Meta<typeof Button> = {
  title: 'Atoms/Button',
  component: Button,
  tags: ['autodocs'],
  parameters: metaParameters(buttonMeta),
  argTypes: argTypesFromMeta(buttonMeta),
  args: { ...argsFromMeta(buttonMeta), children: 'Button' },
};
export default meta;

type Story = StoryObj<typeof Button>;

export const Playground: Story = {};

/** Type A/B/C — content layout presets. */
export const Types: Story = {
  render: (args) => (
    <div style={{ display: 'flex', gap: 'var(--tds-space-3)', flexWrap: 'wrap', alignItems: 'center' }}>
      <Button {...args} type="A">Type A — Label</Button>
      <Button {...args} type="B" iconStart={<Icon name="download" size="sm" />}>Type B — Icon + Label</Button>
      <Button {...args} type="C" iconStart={<Icon name="settings" size="sm" />}>Settings</Button>
    </div>
  ),
};

/** Fill styles (the `variant` axis, labelled "Style"). */
export const Styles: Story = {
  render: (args) => (
    <div style={{ display: 'flex', gap: 'var(--tds-space-3)', flexWrap: 'wrap' }}>
      {(['solid', 'outline', 'ghost', 'soft', 'link'] as const).map((v) => (
        <Button key={v} {...args} variant={v}>{v}</Button>
      ))}
    </div>
  ),
};
```

### 6.2 What the helpers generate (from `src/core/storybook.ts`)

- `argTypesFromMeta(buttonMeta)` → a `select` control per variant axis (`type`, `variant`, `tone`,
  `size`, `shape`) under **Variants**, plus `boolean`/`text`/`number` controls per component prop
  (`disabled`, `loading`, `fullWidth`, …) under **Properties**; `instanceSwap` props get `control: false`.
- `argsFromMeta(buttonMeta)` → `{ type: 'A', variant: 'solid', tone: 'brand', size: 'md', … }` seeded
  from each axis `default`.
- `metaParameters(buttonMeta)` → `docs.description.component` = the generated autodocs blurb, and
  `a11y.element = '[role="button"]'` when the meta declares that role.

### 6.3 Foundations page (real — `src/stories/Foundations.stories.tsx`)

```tsx
import type { Meta, StoryObj } from '@storybook/react';
import { allCollections, cssVar, resolveValue } from '@/tokens';

const meta: Meta = {
  title: 'Foundations/Design Tokens',
  parameters: { layout: 'fullscreen', options: { showPanel: false } },
};
export default meta;
type Story = StoryObj;
// Colors/Space/Type stories read allCollections and render generated tokens — no restated values.
```

## 8. Checklist

Per-story, before commit:

- [ ] File is `src/components/<tier>/<Name>/<Name>.stories.tsx` (or `src/stories/` for Foundations).
- [ ] `title` = `"<Tier>/<Name>"`, tiers align with `storySort`.
- [ ] CSF3: single `export default meta`, named `StoryObj` exports, `Meta<typeof X>`/`StoryObj<typeof X>` generics.
- [ ] `tags: ['autodocs']`; `parameters: metaParameters(meta)`; `argTypes: argTypesFromMeta(meta)`;
      `args: { ...argsFromMeta(meta), …}`.
- [ ] `Playground` first; axis showcases (`Types`/`Styles`/`Tones`/`Sizes`/`States`) as applicable.
- [ ] Imports: component from local module, cross-component from `@/components`, helpers from `@core/storybook`, `import type` for Storybook types.
- [ ] Every style value is `var(--tds-*)`; no hardcoded hex/px/shadow.
- [ ] No invented variant values (all from `meta.variantProps.options`).
- [ ] Toggled Theme (light/dark) + Font toolbars and all four viewports; a11y panel clean.
- [ ] Interactive component has a `play` (SHOULD).
- [ ] Meta changed → ran `npm run ds:build` (+ `npm run figma:build`); `.ai/` manifests updated.
- [ ] `npm run lint` and `tsc -b` pass.

## 9. Failure Recovery

| Symptom | Diagnosis | Fix | Resume |
| --- | --- | --- | --- |
| Controls panel empty / missing an axis | `argTypesFromMeta` not wired, or axis missing from `meta.variantProps` | Wire `argTypes: argTypesFromMeta(meta)`; add the axis to `X.meta.ts` | Reload Storybook; re-run §5.1 step 5 |
| Default render is a broken/blank combo | `args` lacks `argsFromMeta(meta)` or a required child prop | Spread `argsFromMeta(meta)` first, then add `children`/required props | Verify `Playground` renders |
| Autodocs description missing/stale | `tags: ['autodocs']` absent, or `parameters` overrode `metaParameters` | Add the tag; merge (don't replace) `metaParameters(meta)` | Open the Docs tab |
| Theme/font toolbar has no effect | Story bypassed `withTheme` (custom root) or removed the decorator | Rely on the global decorator; don't set theme locally | Toggle toolbar |
| a11y violations flagged | Real component a11y gap or wrong `meta.a11y.role` | Fix the component / correct the role; never suppress in the story | Re-run axe panel |
| Figma lacks a shown combination | Story shows values the meta/plugin doesn't emit | Fix source `.meta.ts`/tokens, `npm run ds:build`, `npm run figma:build` | Re-run `plugin:test` |
| `plugin:test` fails after a story/meta change | Coverage mismatch in `figma/plugin/test/harness.ts` | Reconcile meta ↔ bundle; never edit generated files | Re-run `npm run figma:build` |
| Lint/type error on a story | CSF2 pattern, value import of a type, or `any` cast | Convert to CSF3 objects, use `import type`, type via generic | `npm run lint`; `tsc -b` |

If context is lost mid-task, rehydrate from [.ai/SESSION_SUMMARY.md](../../.ai/SESSION_SUMMARY.md) and
[.ai/TASKS.json](../../.ai/TASKS.json) per [28_RESUME_SPECIFICATION.md](./28_RESUME_SPECIFICATION.md).

## 11. Template

Copy-paste skeleton for a new component story (`src/components/<tier>/<Name>/<Name>.stories.tsx`):

```tsx
import type { Meta, StoryObj } from '@storybook/react';
import { <Name> } from './<Name>';
import { <name>Meta } from './<Name>.meta';
import { argTypesFromMeta, argsFromMeta, metaParameters } from '@core/storybook';
// import { Icon } from '@/components'; // if composition needs sibling components

const meta: Meta<typeof <Name>> = {
  title: '<Atoms|Molecules|Organisms>/<Name>',
  component: <Name>,
  tags: ['autodocs'],
  parameters: metaParameters(<name>Meta),
  argTypes: argTypesFromMeta(<name>Meta),
  args: { ...argsFromMeta(<name>Meta) /* , children: '…' */ },
};
export default meta;

type Story = StoryObj<typeof <Name>>;

export const Playground: Story = {};

/** Type A/B/C — layout presets. */
export const Types: Story = {
  render: (args) => (
    <div style={{ display: 'flex', gap: 'var(--tds-space-3)', flexWrap: 'wrap' }}>
      {/* one instance per meta type option */}
    </div>
  ),
};

/** Fill styles (the `variant` axis, labelled "Style"). */
export const Styles: Story = { /* map meta.variantProps.variant.options */ };
export const States: Story = { /* demonstrate disabled/loading/etc via props */ };

// Interactive components SHOULD add a play() (addon-interactions):
```
