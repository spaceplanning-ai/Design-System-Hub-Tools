# 원패스 플레이북 — 작업을 한 번에 끝내는 법

> 왜 이 문서가 있나: 라이브러리 도입 한 건이 **수십 번의 순차 전달**로 번진 적이 있다.
> 원인은 능력 부족이 아니라 **순서**였다. 게이트가 뒤늦게 작동하니 결함이 10번째
> 전달에서야 드러났고, 그때마다 새 담당을 순차로 붙였다. 아래 순서를 지키면
> 같은 일이 **착수 시점에 한꺼번에** 드러나 한 번에 처리된다.

## 규칙 1 — 착수 전에 `pnpm verify:all` 을 먼저 돌린다

**빨간불의 목록을 먼저 확보한다.** 무엇을 만들기 전에, 지금 리포가 어디서 깨지는지를 안다.
- `verify:all` = 전체 게이트 무중단 체인 (codegen 최신성 · 레지스트리 · 계약 · 경계 · 4자 일치 · 커버리지 · 클린코드 · 네이밍 · tsc · lint · format · 유닛). 브라우저 불필요.
- `verify:full` = 위 + `pnpm e2e` (Playwright — 동작 보존 안전망).

**끝날 때도 `verify:all` 한 번.** 조각조각 재실행하지 않는다 — 한 명령이 전부를 판정한다.

## 규칙 2 — 독립 작업은 처음부터 병렬로 fan-out 한다

경로가 안 겹치면 **동시에** 띄운다. 하나 띄우고 → 보고 받고 → 다음 띄우는 순차 패턴이
길이의 진짜 원인이었다. 소유 경계(P1)가 병렬 안전을 보장한다:

| 동시에 가능 | 이유 |
|---|---|
| `apps/admin/src` (프론트 구현/리팩터) ∥ `packages/ui/src` (컴포넌트 엔지니어) ∥ `tools/*` (도구 소유자) | 경로 무교차 |
| `contracts/**` (계약 엔지니어) ∥ `specs/**` (기능 명세) ∥ `e2e/**` (E2E 테스트) | 경로 무교차 |

**직렬로 묶어야 하는 유일한 경우**: 산출물이 다음 단계의 입력일 때.
계약 → 구현 → 페이지 교체는 순서가 강제된다 (계약 없이 교체하면 화면이 깨진다).

## 규칙 3 — 게이트를 만들면 즉시 CI·pre-commit 에 물린다

작동하지 않는 게이트는 게이트가 아니다. 도구를 새로 만들면 그 자리에서:
- 루트 `package.json` 스크립트 등록 → `verify:all` 체인에 편입
- `.github/workflows/pr-gates.yml` required check 등록 (CI·CD)
- 검출 검증(의도적 위반 주입 → 잡히는지 → 삭제) — **한 번도 빨간불을 못 켜는 도구는 초록불만 켜는 도구와 똑같이 무용하다**

## 규칙 4 — 공허 통과를 처음부터 의심한다

이 리포가 가장 비싸게 배운 것. 초록불이 "검사가 통과했다"가 아니라 "검사가 없다"인 경우:

| 함정 | 냄새 |
|---|---|
| `--passWithNoTests` | 테스트 0건인데 exit 0 |
| 단언 없는 play/test | `expect`·스파이가 0개 → 실패할 수 없음 |
| 측정 불가를 통과로 | dist·기준이미지·명세 부재 시 조용히 exit 0 |
| 분모 세탁 | 커버리지 분모가 줄어 % 가 저절로 오름 |

**측정할 수 없으면 `NOT_VERIFIED`(exit 2)이지 PASS 가 아니다.** 새 검증 도구는 이 원칙 위에 짓는다.

## 규칙 5 — 리팩터 전에 E2E 안전망을 먼저 깐다

대규모 변경(모듈화·라이브러리 교체) 전에 **보이는 동작**을 E2E 로 고정한다.
그러면 "동작이 안 바뀌었다"를 주장이 아니라 **기계로** 증명한다. 리팩터 후에 쓰면 기준선이 없다.
E2E 는 role·접근 가능한 이름·텍스트로 찾는다 — 내부 구조(클래스명)에 걸면 리팩터가 곧 테스트를 깬다.

## 규칙 6 — 낡은 주석을 만료시킨다

`"@tds/ui 는 아직 비어 있다"` 주석 하나가 **손코딩 복사본 18개**를 낳았다. 한때 참이었고
아무도 지우지 않았다. 조건이 바뀌면 그 조건을 적어둔 주석부터 찾아 지운다.

---

## 체크리스트 (작업 1건)

```
□ pnpm verify:all        # 착수 전 — 빨간불 목록 확보
□ 독립 작업 병렬 fan-out  # 경로 무교차면 동시에
□ 계약→구현→교체는 직렬   # 산출물이 입력인 경우만
□ 리팩터면 E2E 먼저
□ 새 게이트는 즉시 CI 물림 + 검출 검증
□ pnpm verify:full       # 종료 시 — 한 명령으로 전부 판정
□ 커밋 & 푸시
```
