# SCR-001. 로그인

## 1. 화면 개요

어드민 허브의 유일한 인증 진입점이다. 미인증 사용자가 이메일과 비밀번호로 인증해 세션을 발급받는다.
인증 성공 시 대시보드(SCR-002)로 이동하며, 보호된 URL로 접근하다 리다이렉트된 경우에는 원래 목적지로 복귀한다.
로그인 실패 누적에 따른 계정 잠금, 제출 중 중복 제출 차단, '이메일 저장' 편의 기능을 포함한다.

| 항목 | 내용 |
|---|---|
| 목적 | 어드민 허브 사용자의 이메일/비밀번호 인증 및 세션 발급 |
| 주 사용자(역할) | 미인증 상태의 전체 어드민 역할(시스템 관리자 · 운영자 · 뷰어) |
| 진입 경로 | (1) 루트 URL 직접 접근(미인증 시), (2) 보호된 URL 접근 시 미인증 리다이렉트, (3) 세션 만료 후 재인증 리다이렉트 |
| 관련 화면 | SCR-002 대시보드(인증 성공 시 이동처) |

### 5.2 유효성 규칙

| 필드 | 규칙 (형식·길이·필수 여부) | 위반 시 에러 메시지 |
|---|---|---|
| 이메일 | 필수. 형식: `로컬부@도메인부.TLD`(공백 불가, `@` 정확히 1개, 도메인부에 `.` 1개 이상). 최대 254자 | 미입력: "이메일을 입력해 주세요." / 형식 위반: "이메일 형식이 올바르지 않습니다." |
| 비밀번호 | 필수. 8자 이상 64자 이하(구성 문자 제한 없음 — 강도 규칙은 계정 생성 화면 소관) | 미입력: "비밀번호를 입력해 주세요." / 길이 위반: "비밀번호는 8자 이상 64자 이하로 입력해 주세요." |
| 이메일 저장 (체크박스) | 선택(boolean, 기본값: 직전 로그인에서 저장된 이메일이 있으면 체크, 없으면 해제) | 에러 없음 — 선택 항목 |

검증 시점 규칙:

- 필드 blur 시점에 해당 필드 형식 검사 → 위반 시 필드 하단 인라인 에러 표시.
- 제출 시점에 전체 필드 재검사 → 위반 필드가 1개 이상이면 서버 요청을 보내지 않고, 첫 번째 위반 필드로 포커스 이동.
- 클라이언트 검증 통과 후에만 인증 API를 호출한다.

### 5.3 기타 규칙

1. **실패 카운트/잠금**: 동일 계정 기준 연속 실패 5회 도달 시 30분 잠금(서버 판정). 인증 성공 시 실패 카운트 0으로 리셋. 잠금 중 제출 시도는 §3 등록-에러 (b) 문구로 응답.
2. **'이메일 저장'**: 체크 상태로 인증 성공 시 이메일 값만 브라우저 저장소에 보관. 체크 해제 상태로 인증 성공 시 기존 저장값 삭제. 비밀번호는 어떤 경우에도 저장하지 않는다.
3. **returnUrl 보존**: 보호된 URL에서 리다이렉트 진입한 경우 원래 URL을 쿼리 파라미터로 보존, 인증 성공 시 해당 URL로 복귀. returnUrl이 외부 도메인이면 무시하고 SCR-002로 이동(오픈 리다이렉트 차단).
4. **비밀번호 표시 토글**: 비밀번호 필드 우측 토글 버튼으로 마스킹/평문 전환. 토글은 `aria-pressed` 상태를 노출하고, 전환 시 입력값과 커서 위치를 유지한다. 기본 상태는 마스킹.
5. **제출 수단**: 제출 버튼 클릭 및 폼 내 Enter 키 입력 2가지. 로딩 중에는 2가지 모두 차단(§3 등록-로딩).
6. **제출 타임아웃**: 10초. 초과 시 요청 중단 후 §3 등록-에러 (d) 처리.
7. **세션 만료 리다이렉트 진입 시**: 폼 상단에 Alert(info) "세션이 만료되었습니다. 다시 로그인해 주세요." 표시.
