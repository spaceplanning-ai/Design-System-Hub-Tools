# SCR-002. 대시보드

## 1. 화면 개요

로그인(SCR-001) 성공 직후 도착하는 어드민 허브의 홈 화면이다. 선택한 기간의 핵심 지표(KPI) 4종과
지표 추이 차트, 최근 활동 기록을 한 화면에서 조회한다. 읽기 전용 화면으로 데이터 생성·수정·삭제 기능은 없다.
차트 영역은 본 릴리스에서 플레이스홀더로 예약하며 시각화 구현은 별도 ADR로 결정한다.

| 항목 | 내용 |
|---|---|
| 목적 | 운영 핵심 지표(주문·매출·가입·문의)와 최근 활동을 기간 기준으로 조회 |
| 주 사용자(역할) | 시스템 관리자 · 운영자 · 뷰어(전 역할 진입 가능, 활동 테이블은 §5.1 조건부) |
| 진입 경로 | (1) 로그인 성공 직후 기본 이동처, (2) 전역 내비게이션의 '대시보드' 메뉴, (3) 루트 URL 접근(인증 상태) |
| 관련 화면 | SCR-001 로그인(미인증 시 리다이렉트 출발점), SCR-003 상품 등록(전역 내비게이션으로 이동) |

### 3.1 KPI 스탯 카드 상태 (보충 — CRUD 매트릭스 외 위젯)

KPI 스탯 카드 4종(오늘/기간 주문 수 · 매출액 · 신규 가입 회원 수 · 미처리 문의 수)은 아래 4개 상태를 카드 단위로 갖는다.

| 상태 | 정의 |
|---|---|
| 정상 | 기간 내 집계값 + 직전 동일 길이 기간 대비 증감률(소수점 1자리, 부호 포함). 증감률은 Badge로 표시(증가/감소/변동 없음 3분류) |
| 빈/0값 | 집계값 0은 빈 상태가 아니라 정상 값 "0"으로 표시. 직전 기간 값이 0이라 증감률 계산 불가면 증감 Badge 대신 "비교 불가" 표기 |
| 에러 | 실패한 카드만 값 자리에 "—" + 카드 내 [다시 시도] 텍스트 버튼. 나머지 카드는 정상 유지 |
| 로딩 | 카드 값 영역 스켈레톤. 4장 각각 독립 로딩(하나의 응답 지연이 다른 카드를 막지 않음) |

### 3.2 차트 영역 상태 (보충)

| 상태 | 정의 |
|---|---|
| 정상(본 릴리스) | 플레이스홀더 패널: "지표 추이 차트 — 준비 중" 문구 고정 표시. 시각화 구현은 별도 ADR 결정 후 후속 SCR 개정으로 정의 |
| 빈/에러/로딩 | N/A — 플레이스홀더는 데이터 요청을 하지 않으므로 3개 상태가 발생하지 않는다. 차트 구현 시점에 본 절을 개정한다 |

### 5.3 기타 규칙

1. **정렬**: 최근 활동 테이블은 발생 시각 내림차순 고정. 컬럼 헤더 클릭 정렬은 제공하지 않는다(본 릴리스 범위 밖).
2. **페이지 크기**: 10건 고정. 사용자 변경 기능 없음. 최대 조회 범위는 선택 기간 내 1,000건(초과분은 "1,000건 초과 — 기간을 좁혀 주세요" 안내를 테이블 하단에 표시).
3. **기간 정의**: '오늘' = 당일 00:00부터 현재까지(서버 표준시 Asia/Seoul). '최근 7일'/'최근 30일' = 당일 포함 직전 7일/30일. 증감률 비교 기준은 직전 동일 길이 기간.
4. **갱신 정책**: 자동 갱신(폴링) 없음. 데이터 갱신은 화면 재진입, 기간 필터 변경, 위젯별 [다시 시도] 3가지 경로만 존재.
5. **위젯 독립성**: KPI 카드 4장 · 차트 · 테이블은 각각 독립 요청/독립 상태. 한 위젯의 에러·로딩이 다른 위젯을 차단하지 않는다.
6. **금액 표기**: 매출액 KPI는 원화, 천 단위 구분 기호 포함, 1억 원 이상이면 "N.n억 원" 축약 표기(소수점 1자리).
