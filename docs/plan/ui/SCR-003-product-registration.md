# SCR-003. 상품 등록

## 1. 화면 개요

새 상품을 생성하는 단일 등록 폼 화면이다. 기본 정보(상품명·상품 설명), 가격, 재고, 카테고리 선택,
이미지 업로드의 5개 폼 섹션으로 구성된다. [임시저장]과 [등록]의 이중 액션을 제공하며,
저장되지 않은 입력이 있는 상태의 이탈 시도는 브라우저 confirm(beforeunload)과 라우트 가드로 경고한다.
화면 재진입 시 임시저장본이 있으면 복원 여부를 묻는다. 기존 상품의 수정은 별도 SCR(미작성) 소관이다.

| 항목 | 내용 |
|---|---|
| 목적 | 새 상품 1건의 정보 입력·이미지 업로드·임시저장·등록 |
| 주 사용자(역할) | 시스템 관리자 · 운영자 (뷰어는 진입 시 권한없음 안내 — §5.1) |
| 진입 경로 | (1) 전역 내비게이션의 '상품 등록' 메뉴(SCR-002 등에서 이동), (2) 상품 등록 URL 직접 접근(인증 상태) |
| 관련 화면 | SCR-001 로그인(미인증 시 리다이렉트 이동처), SCR-002 대시보드(전역 내비게이션 출발점), 상품 목록(별도 SCR 미작성 — 등록 성공 후 이동처로 개정 예정) |

### 3.1 이미지 업로드 상태 (보충 — CRUD 매트릭스 외 위젯)

파일 선택 즉시 파일별로 업로드를 시작하며(개별 비동기), [등록] 제출 시에는 업로드 완료된 이미지 ID만 전송한다.

| 상태 | 정의 |
|---|---|
| 대기(선택 전) | 드롭존 + [파일 선택] 버튼 + 규칙 안내 문구 고정 표시: "JPG·PNG·WebP, 1장당 5MB 이하, 최대 5장" |
| 업로드 중 | 파일별 진행률 표시 + 해당 파일 [취소] 버튼. 1장이라도 업로드 중이면 [등록] 제출 차단(§3 등록-에러 (d)). 파일당 60초 타임아웃(§5.3 규칙 5) |
| 성공 | 섬네일 + 파일명 + [삭제] 버튼 표시. 첫 번째 이미지에 '대표' Badge 표시(§5.3 규칙 6) |
| 실패 | 실패한 파일 행에만 인라인 에러(사유별 §5.2 이미지 메시지) + [다시 시도] · [삭제] 버튼. 다른 파일의 상태에 영향 없음 |
| 제거 | [삭제] 클릭 시 확인 절차 없이 즉시 목록에서 제거(파일 재선택으로 복구 가능). 제거로 0장이 되면 제출 시 §5.2 필수 규칙 위반으로 처리 |

### 3.2 임시저장·복원 상태 (보충)

| 상태 | 정의 |
|---|---|
| 저장 성공 | [임시저장] 클릭 → 완화 검증(§5.2 검증 시점 규칙) 통과 시 서버 저장. Alert(success) "임시저장되었습니다. (HH:mm)" + dirty 해제 |
| 저장 실패 | Alert(danger) "임시저장하지 못했습니다. 다시 시도해 주세요." + 입력값 전부 유지(dirty 유지) |
| 저장 로딩 | [임시저장] 버튼 loading 상태(스피너 + aria-busy), [등록] 버튼 비활성(동시 요청 최대 1건 — §5.3 규칙 4). 10초 내 무응답 시 요청 중단 후 저장 실패로 전환 |
| 복원 안내 | 화면 진입 시 임시저장본 존재하면 폼 상단 Alert(info) "임시저장된 상품이 있습니다. (저장 시각: YYYY-MM-DD HH:mm)" + [이어서 작성] · [삭제하고 새로 작성] 버튼 2개 |
| 복원 실행 | [이어서 작성] → 저장 시점의 전 필드 값·이미지 목록 복원. 사용자가 이미 입력을 시작한 상태(dirty)면 브라우저 confirm "현재 입력한 내용이 임시저장본으로 대체됩니다. 계속할까요?" 확인 시에만 복원, 취소 시 안내 Alert 유지 |
| 복원 거부 | [삭제하고 새로 작성] → 임시저장본 서버 삭제 + 안내 Alert 닫힘 + 빈 폼 유지. 삭제 요청 실패 시 Alert(danger) "임시저장본을 삭제하지 못했습니다. 다시 시도해 주세요." |

- 클라이언트 검증 통과 후에만 등록/임시저장 API를 호출한다.

### 5.3 기타 규칙

1. **이중 액션**: 폼 하단 우측에 [임시저장](Button variant: secondary) · [등록](Button variant: primary) 2개 버튼을 배치한다.
   폼 내 Enter 키 제출은 비활성한다(여러 줄 입력과 필드 수가 많아 오제출 위험) — 제출 수단은 버튼 클릭 1가지.
2. **임시저장 보관 정책**: 서버 저장, 계정당 1건. 재저장 시 최신본이 기존 본을 덮어쓴다. 등록 성공 시 임시저장본 삭제.
   보관 기한 30일(초과분은 서버가 삭제).
3. **저장 안 된 이탈 경고**: dirty 판정 기준 = 마지막 임시저장/등록 성공 이후 필드 값 또는 이미지 목록 변경 1건 이상.
   (a) 브라우저 이탈(탭 닫기·새로고침·외부 URL 이동): beforeunload 이벤트로 브라우저 기본 confirm 표시.
   (b) 앱 내 라우트 이동: 라우트 가드가 confirm "저장되지 않은 변경 사항이 있습니다. 이 화면을 벗어나면 입력한 내용이 사라집니다. 계속할까요?"
   표시 — 확인 시 이동, 취소 시 잔류. 등록/임시저장 성공 직후(dirty 해제 상태)에는 두 경고 모두 표시하지 않는다.
4. **중복 제출 차단**: [등록]·[임시저장]의 동시 요청은 최대 1건 — 한쪽이 진행 중이면 두 버튼 모두 비활성(§3 등록-로딩, §3.2 저장 로딩).
5. **타임아웃**: 등록 30초 / 임시저장 10초 / 이미지 업로드 파일당 60초. 초과 시 요청 중단 후 각 에러 상태로 전환.
6. **대표 이미지**: 업로드 순서상 첫 번째 이미지가 대표('대표' Badge 표시). 이미지 순서 변경 기능은 본 릴리스 범위 밖 —
   대표 변경은 삭제 후 재업로드로 수행한다.
7. **숫자 입력 표시**: 판매가·재고 수량은 숫자만 입력 가능(그 외 문자는 입력 무시). blur 시 천 단위 구분 기호를 표시하고,
   포커스 시 구분 기호를 제거한 원값을 편집한다.
8. **카테고리 로드**: 화면 진입 시 1회 로드(§3.3). 화면 체류 중 자동 갱신 없음 — 재로드는 [다시 시도] 버튼 1가지 경로.
