# Badge

<code>atom</code> · <code>Data Display</code> · <code>taxonomyItem: badge</code> · <code>v1.0.0</code> · <code>status: beta</code>

카운트 뱃지 — 제목/라벨 옆에 개수를 표시하는 원형 pill. 출처:
`apps/admin/src/pages/dashboard/components/Card.tsx` 의 `CountBadge` (`count<=0` 이면 렌더하지 않음).

**비대화형 표시 전용이며 클릭 이벤트를 갖지 않는다.**

## Props

<Controls />

| prop | 타입 | 기본값 | 의미 |
|---|---|---|---|
| `count` | `number` | — (필수) | 표시할 개수. `hideWhenZero=true` 이고 `count<=0` 이면 아무것도 렌더하지 않는다 (구현: `if (count <= 0) return null`). |
| `tone` | `neutral \| danger \| success` | `neutral` | 시각 의미. `neutral` = 텍스트색 배경 위 서피스색 숫자(현행 구현), `danger`/`success` = feedback 토큰 페어. |
| `hideWhenZero` | `boolean` | `true` | `count<=0` 일 때 렌더 생략 여부. `false` 면 0 도 표시한다. |

## 접근성 계약

- **role**: `status`
- **keyboard**: `none` — 비대화형. 포커스 순서에 들어가지 않는다.
- **focusVisible**: `false`
- **contrastMin**: 4.5

계약 `a11y.aria`:

- `aria-live`: 카운트가 런타임에 갱신되는 문맥에서는 polite. **정적 렌더 시 불필요.**
- `labelling`: **숫자만으로는 의미가 없다** — 인접 제목(CardTitle 등)이 접근 가능한 문맥을 제공해야 한다.

> StatusBadge 와 대비되는 판정이다. StatusBadge 는 목록에 수십 개가 렌더되므로 `role=status` 를 **두지 않는다**.
> Badge 는 카운트 단독 표시라 `role=status` 를 갖는다 (StatusBadge 계약 `description` 이 이 대비를 명시한다).
