# Button

<code>atom</code> · <code>Actions</code> · <code>taxonomyItem: button</code> · <code>v1.1.0</code> · <code>status: beta</code>

기본 액션 버튼. 이 계약에서 React Props 타입 · Storybook argTypes · Figma Component Properties · Docs 가
자동 생성된다 (`tools/codegen`). G3 승인 시 Frozen — 변경은 변경 요청 → G3 재진입 + SemVer 재판정.

스토리는 96칸 조합을 전수로 갖지만 이 문서는 **대표 스토리만** 싣는다. 전수는 사이드바의 스토리 목록에서 본다.

계약이 선언한 prop 의 의미는 아래와 같다. 표의 설명은 `contracts/Button.contract.json` 의 `props` 를 옮긴 것이다.

| prop | 타입 | 기본값 | 의미 |
|---|---|---|---|
| `variant` | `primary \| secondary \| ghost \| danger` | `primary` | 시각 위계. 디자인 스펙의 variant 목록과 완전 일치해야 한다. 기본값 `primary` 는 유지한다 — `apps/admin` 의 `<Button>` 호출부는 전수 감사 결과 100% variant 를 명시하고 있어(무지정 호출부 0건) 기본값 차이로 시각이 뒤집히는 호출부가 없다. |
| `type` | `button \| submit \| reset` | `button` | 네이티브 button type. 그 외 값은 구현이 `button` 으로 좁힌다. **기본값 `button` 은 HTML 기본값(`submit`)을 의도적으로 뒤집은 DS 결정**이다 — 폼 안의 보조 버튼이 실수로 제출하지 않게 한다. |
| `size` | `sm \| md \| lg` | `md` | 크기 축. Figma Component Property `Size` 에 대응한다. |
| `loading` | `boolean` | `false` | 로딩 중 스피너 표시 + `onClick` 차단 + `aria-busy`. |
| `disabled` | `boolean` | `false` | 비활성. `onClick` 차단 + `aria-disabled`. |
| `isFullWidth` | `boolean` | `false` | 컨테이너 100% 폭. 기본은 내용 폭(`inline-flex`). |
| `iconLeft` | `slot<Icon>` | `null` | 좌측 아이콘 슬롯. `loading` 중에는 스피너로 대체되어 숨는다. |
| `iconRight` | `slot<Icon>` | `null` | 우측 아이콘 슬롯. 레이블 뒤에 온다(예: '다음 ›'). `loading` 중에는 숨는다. |
| `children` | `node` | — (필수) | 버튼 레이블. |

### `type` 이 enum 이 아닌 이유 — 계약이 명시한 판단

`type` 은 시각 변형이 아니라 HTML 시맨틱이라 Figma Component Property 대응이 없다. enum 으로 선언하면
스키마가 `figmaProperty` 를 강제해(G3) **시각차 0 인 3값 Figma Variant 축**이 생기고, `contract-test` 의
조합 커버리지 요구가 3배로 뛴다. 그래서 허용 값은 `values` 로만 기술하고 Figma/Variant 축은 만들지 않는다.

### `isFullWidth` 이름 — naming-guard 규칙

boolean prop 은 `is`/`has`/`can` 접두 또는 상태 형용사 화이트리스트만 허용된다
(naming-guard `boolean-prop` 규칙 · ADR-0005). CR 원안의 `fullWidth` 는 규칙 위반이라 `isFullWidth` 로
좁혔다. **Figma 쪽 이름은 `FullWidth` 로 유지한다** — 두 이름이 다른 것은 의도된 것이다.

## 접근성 계약

계약 `a11y` 가 선언한 것:

- **role**: `button` (네이티브 `<button>` 이 암묵 role 을 갖는다)
- **keyboard**: `Enter` · `Space`
- **focusVisible**: `true`
- **aria-disabled**: `disabled` 일 때
- **aria-busy**: `loading` 일 때
- **contrastMin**: 4.5

### 네이티브 속성 패스스루 — 계약 prop 이 아니다

계약 `description` 과 `a11y.aria.native-passthrough` 가 명시한 **의도된 예외**다.
계약 props 외의 표준 HTML/ARIA 속성(`aria-label` · `aria-describedby` · `aria-busy` · `title` …)은 구현이
`<button>` 으로 그대로 전달한다 (Card 선례: `CardProps & Omit<HTMLAttributes,'style'|'children'|'className'>`).
이 속성들은 시각 변형이 아니고 Figma Component Property 대응도 없으므로 **계약 prop 으로 열거하지 않는다.**
실사용: RolePanel(`aria-describedby`·`title`) · PointsCard(`aria-label`) · ConfirmDialog(`aria-busy`).

`className`/`style` 은 토큰 규칙 보호를 위해 **차단한다.**

호출부가 명시한 `aria-busy` 는 `loading` 파생값보다 **우선한다** — ConfirmDialog 는 스피너 없이
`aria-busy` 만 필요로 한다.

<Story of={ButtonStories.AriaBusyOverride} />
<Story of={ButtonStories.NativeAttributePassthrough} />

### `type="submit"` 경고

계약 `a11y.aria.type-submit`: `type="submit"` 은 폼 제출 시맨틱이다 — 이 버튼이 `<form>` 안에 있으면
**Enter 키 제출도 함께 살아난다.** `type` 을 하드코딩하면 폼이 조용히 죽는다.
