# Card

배경·테두리·라운드·내부 여백을 제공하는 최소 단위 표면(surface) 컨테이너입니다.

## 언제 사용하나요?

- 콘텐츠를 하나의 표면 위에 묶어 보여줄 때
- 대시보드 요약·패널처럼 영역을 시각적으로 구분할 때
- 강조가 필요한 카드를 배경 위로 살짝 띄우고 싶을 때 (`elevation="raised"`)
- 데이터를 불러오는 동안 로딩 상태를 알려야 할 때 (`busy`)

## Props

<Controls />

| Prop        | Description                                                        |
| ----------- | ----------------------------------------------------------------- |
| `children`  | 카드 본문. 컨테이너는 flex column 이라 어떤 내용이든 담깁니다.     |
| `padding`   | 내부 여백. `md`(기본) 또는 `lg`.                                   |
| `elevation` | 표면 높이. `flat`(기본, 그림자 없음) 또는 `raised`(배경 위로 부상).|
| `busy`      | 로딩 중일 때 `true`. `aria-busy="true"` 가 부여됩니다.            |

## Behavior

<Story of={CardStories.States} />

- `busy={true}` 이면 `aria-busy="true"` 로 로딩 상태를 알립니다.

## Accessibility

- 카드 자체는 포커스 대상이 아니며, 내부 요소가 탭 순서를 가집니다.
- region 랜드마크로 노출하려면 조립하는 쪽에서 `aria-labelledby`(제목 id) 또는 `aria-label` 로 이름을 줍니다. 이름이 없으면 generic 으로 노출되며 정상 동작입니다.
- 문서 방향(RTL)을 따라 자연스럽게 흐릅니다.

## Notes

- 헤더/본문 구조나 도메인 데이터는 카드가 강제하지 않습니다 — 조립은 organism(StatsCard·TodoCard·ListCard)이 담당합니다.
