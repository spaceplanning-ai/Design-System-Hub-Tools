# HelpTip

<code>atom</code> · <code>Feedback</code> · <code>taxonomyItem: tooltip</code> · <code>v1.0.0</code> · <code>status: beta</code>

ⓘ 도움말 — 라벨 옆에 붙는 **disclosure.** 출처: `apps/admin/src/shared/ui/HelpTip.tsx`
(소비 3곳: 폼 라벨의 부가 설명).

### 왜 툴팁이 아니라 disclosure 인가

계약 `description`: **hover 전용 툴팁은 키보드·터치에서 열리지 않으므로** 버튼을 눌러 여닫는 disclosure 로
만든다 — `aria-expanded` 로 열림 상태를, `aria-controls` 로 설명 문단을 잇는다.

열림 상태는 **컴포넌트 내부(`useState`)가 소유한다** — 외부 제어 이벤트를 두지 않는다.

## Props

| prop | 타입 | 기본값 | 의미 |
|---|---|---|---|
| `label` | `string` | — (필수) | 스크린 리더용 트리거 이름 — 아이콘만 있으므로 `aria-label` 로 무엇에 대한 도움말인지 밝힌다 ('그룹 유형 설명' 등). |
| `children` | `node` | — (필수) | 패널 본문 — 열렸을 때 아이콘 아래로 뜨는 설명 문단(`<p>`)의 내용. |

**이벤트가 없다.** 계약에 `events` 항목 자체가 없으며, 이것은 위의 '열림 상태를 내부가 소유한다' 는
설계 결정의 결과다.

## 접근성 계약

- **role**: `button` (트리거 기준)
- **keyboard**: `Enter` · `Space` · `Tab`
- **focusVisible**: `true`
- **contrastMin**: 4.5

계약 `a11y.aria`:

- `disclosure`: hover 툴팁이 아니라 **버튼 토글 disclosure** 다 — `aria-expanded` 로 열림 상태,
  `aria-controls` 로 설명 문단(`<p id>`)을 잇는다 (키보드·터치에서도 열린다).
- `label`: 트리거는 아이콘(ⓘ)만 있으므로 접근 가능한 이름을 `aria-label` 로 준다.
- `panel-hidden`: 닫혔을 때 `hidden` 속성으로 감춘다 — **언마운트하지 않아** `aria-controls` 대상이 사라지지 않는다.

### 패널을 언마운트하지 않는 이유

계약 `description`: 닫혔을 때 요소를 지우면 **`aria-controls` 가 가리키는 대상이 사라진다** —
`hidden` 속성으로 감추기만 한다.
