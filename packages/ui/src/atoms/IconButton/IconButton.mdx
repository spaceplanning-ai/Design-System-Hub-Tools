# IconButton

<code>atom</code> · <code>Actions</code> · <code>taxonomyItem: icon-button</code> · <code>v1.0.0</code> · <code>status: beta</code>

아이콘만 보이는 정사각 버튼. **없던 것을 만든 게 아니라 갈라져 있던 두 벌을 합쳤다.**

| 출처 | 자리 |
|---|---|
| 이메일 빌더 | `apps/admin/src/pages/marketing/message-templates/email/styles.ts` 의 `iconButtonStyle` |
| 문자·알림톡 편집기 | `apps/admin/src/pages/marketing/message-templates/components/EditorToolbar.tsx:74` 의 비공개 `function IconButton` |

두 화면 모두 `var(--tds-*)` 토큰은 옳게 썼다. 그런데도 값이 갈라진 이유는 **토큰을 잘못 써서가 아니라
컴포넌트가 없어서**다 — 크기(`space.7` vs `space.6`) · 포커스 링 · `title` 속성이 파일마다 달랐다.

## Props

<Controls />

| prop | 타입 | 기본값 | 의미 |
|---|---|---|---|
| `icon` | `ReactNode` | — (필수) | 버튼이 담는 유일한 글리프. `aria-hidden` 래퍼에 들어간다. |
| `label` | `string` | — (필수) | 접근 가능한 이름. `aria-label` 과 `title` 양쪽에 같은 값으로 들어간다. |
| `size` | `sm \| md` | `md` | 정사각 변 길이. `md`=`space.7`(이메일 빌더) · `sm`=`space.6`(문자 편집기). |
| `pressed` | `unset \| on \| off` | `unset` | 토글 상태. `unset` 이면 `aria-pressed` 를 **아예 내지 않는다.** |
| `disabled` | `boolean` | `false` | 네이티브 `disabled`. onClick 차단 + 포커스 순서에서 제외. |

## `label` 이 필수인 이유

아이콘만 보이므로 접근 가능한 이름을 공급할 것이 다른 데 없다. 비면 스크린리더에 "버튼" 이라고만 읽힌다.

같은 문자열이 `aria-label`(스크린리더)과 `title`(마우스 툴팁) 양쪽에 들어간다 —
**값이 하나이므로 두 사용자 집단이 얻는 정보가 갈라질 수 없다.** `title` 은 문자 편집기 쪽에만 있던 것이고,
이메일 쪽에는 없어 마우스 사용자가 아이콘의 뜻을 알 길이 없었다.

## `pressed` 가 boolean 이 아니라 3값인 이유

`aria-pressed` 는 세 상태를 갖는다: **없음**(일반 액션) · `true` · `false`.

<Story of={IconButtonStories.States} />

위 줄의 **왼쪽 둘이 `unset` 과 `off`** 다. 픽셀은 같고 낭독이 다르다.

> 실행취소·다시하기 같은 일반 액션에 `aria-pressed="false"` 를 달면 **거짓 시맨틱**이다 —
> 스크린리더가 "토글 버튼, 안 눌림" 이라고 읽어 사용자가 누르면 무언가 켜질 것이라 기대하게 된다.

문자 편집기의 수제 구현은 이 함정을 이미 알고 있었다 (`EditorToolbar.tsx:84` 주석과 `:96` 의 조건부 spread).
계약에는 boolean 을 optional 로 표현할 수단이 없어(boolean 은 `default` 와 `figmaProperty` 를 요구하고,
`default` 를 주면 속성이 항상 나간다) **부재를 값으로 올린 것이 `unset`** 이다.

## 눌린 시각의 소유자는 `aria-pressed` 다

`pressed` 전용 클래스를 두지 않는다. CSS 가 `[aria-pressed='true']` 를 직접 본다.

> **접근성 상태와 픽셀이 한 원천에서 나오므로 "색만 칠하고 속성은 빠뜨림"(그 반대도)이
> 원리적으로 불가능하다.**

## size

<Story of={IconButtonStories.Sizes} />

두 값 모두 실물에서 왔다. 어느 한쪽을 틀렸다고 판정할 근거가 없어(둘 다 오너 확정 화면이다) **축으로 올렸다.**
기본값 `md` 는 파일럿 소비자(이메일 빌더)의 현행 시각을 유지하기 위한 것이며,
시각 회귀 없이 이관하려는 화면은 자기 값을 명시한다.

## 왜 별도의 `ToggleButton` 이 없나

분류표 `Actions/toggle-button` 키는 **비워 두었다.** `loading` 키 선례와 같은 판단이다.

- 두 툴바의 토글은 전부 '눌릴 수 있는 아이콘 버튼' 이다. 눌림 여부만 다른 컴포넌트를 하나 더 만들면
  **CSS 가 두 벌이 되어 이 컴포넌트가 없애려던 표류가 그대로 되살아난다.**
- 글자 라벨이 붙는 토글(문자 편집기의 '치환변수' 알약)은 `Button` 이 이미 덮는다 —
  Button 계약의 네이티브 속성 패스스루가 `aria-pressed` 를 그대로 흘린다.
- **상호배타 묶음**(캔버스 폭 데스크톱/모바일)은 `SegmentedControl` 의 영역이다(`role="radiogroup"`).
  독립 토글(패널 접기)은 어떤 해석으로도 그쪽 영역이 아니었다.

## 툴바 조립

<Story of={IconButtonStories.InToolbar} />

`Divider` 와 함께 쓴다. 툴바 자체는 컴포넌트가 아니라 **조립**이다 — 자세한 근거는 `Divider` 문서를 참조한다.

## 접근성 계약

- **role**: `button` — 네이티브 `<button>` 의 암묵 role.
- **keyboard**: `Tab`(진입 · disabled 면 제외) · `Enter`/`Space`(활성화 — 네이티브가 click 으로 바꾼다).
- **focusVisible**: `true` — 키보드 포커스에만 링. 두께는 단일 토큰 `border-width.medium` 에서 온다.
- **ariaDisabled**: 쓰지 않는다. **네이티브 `disabled` 를 건다** — 툴바 버튼은 비활성일 때 포커스 순서에
  남을 이유가 없다(메뉴 항목과 다르다).
- **contrastMin**: 3
- **네이티브 패스스루**: `aria-expanded` · `aria-haspopup` · `aria-controls` 는 계약 prop 이 아니라
  `<button>` 으로 그대로 흐른다. 변수 드롭다운 트리거가 이것을 쓴다.

## 토큰

`space.7`/`space.6`(md/sm 변 길이) · `radius.sm` · `border-width.thin`(테두리) ·
`color.transparent`(기본 배경·테두리) · `color.surface.raised`(hover · pressed 배경) ·
`color.action.primary.default`(pressed 테두리·글자색) · `color.text.default` · `color.text.disabled` ·
`color.border.focus` + `border-width.medium`(포커스 링) · `motion.duration.fast` + `motion.easing.standard`.

기본 테두리를 `color.transparent` 로 **항상 같은 두께로** 두는 이유: 눌렸을 때 테두리가 생기면서
글리프가 밀리지 않게 하기 위해서다. 의존 없음(atom).

`responsive.behavior` 는 `size-fixed` — 브레이크포인트에 따라 크기가 변하지 않는다.
