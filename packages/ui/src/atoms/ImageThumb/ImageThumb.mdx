# ImageThumb

<code>atom</code> · <code>Media</code> · <code>taxonomyItem: image</code> · <code>v1.0.0</code> · <code>status: beta</code>

목록 썸네일 — 작은 이미지. 출처: `apps/admin/src/shared/ui/ImageThumb.tsx`
(소비 3곳: 로고 목록·인증서 목록·리뷰 미리보기).

`src` 가 비었거나(trim 후 빈 문자열) 로드에 실패하면 **빈칸/깨진 이미지 대신 이미지 아이콘 placeholder 를
한 곳에서 보장한다.**

**도메인을 모른다** — 무슨 이미지인지 알지 못하고 `src`(URL 문자열)와 접근성 이름(`alt`)만 받는다.

## Props

<Controls />

| prop | 타입 | 기본값 | 의미 |
|---|---|---|---|
| `src` | `string` | — (필수) | 이미지 URL. **앞뒤 공백을 제거한 뒤** 빈 문자열이면 placeholder 를 렌더한다. 로드에 실패해도(`onError`) placeholder 로 폴백한다. |
| `alt` | `string` | — (필수) | 접근성 이름 — 실제 이미지의 `alt` 이자 placeholder(`role=img`)의 `aria-label` 이다. **두 경로 모두 같은 이름으로 읽힌다.** |

**이벤트가 없다** — 계약에 `events` 항목 자체가 없다. 비대화형 표시 전용이다.

### 로드 실패 복구

계약 `description`: `src` 가 바뀌면 실패 플래그를 초기화한다 — **이전 URL 의 실패가 새 URL 로 새지 않는다.**

## 접근성 계약

- **role**: `img`
- **keyboard**: `none` — 비대화형 이미지. 포커스 순서에 들어가지 않는다.
- **focusVisible**: `false`
- **contrastMin**: 4.5

계약 `a11y.aria`:

- `alt-both-paths`: 실제 이미지는 `alt` 로, placeholder(빈 `src`·로드 실패)는 `role="img"` +
  `aria-label={alt}` 로 **같은 접근 가능한 이름**을 준다.
- `decorative-icon`: placeholder 안의 이미지 아이콘은 `aria-hidden` 장식이다 —
  접근 가능한 이름은 컨테이너(`role=img`)가 제공한다.

## 토큰

`color.surface.raised`(placeholderSurface) · `color.text.muted`(placeholderText) ·
`color.border.default`(placeholderBorder) · `radius.sm`(placeholderRadius) · `border-width.thin`.

**토큰이 전부 placeholder 경로의 것이다** — 실제 이미지는 스스로 그려지므로 색 토큰이 필요 없다.

`responsive.behavior` 는 `size-fixed`. 의존 없음 — 다만 **FileChip 이 이 컴포넌트에 의존한다**.
