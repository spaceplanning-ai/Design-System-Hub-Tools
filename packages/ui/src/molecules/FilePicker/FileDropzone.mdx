# FileDropzone

클릭하거나 파일을 끌어다 놓아 파일 한 건을 고르는 가로형 선택 영역입니다.

## 언제 사용하나요?

- 파비콘·대표 이미지처럼 파일 한 건을 올려야 할 때
- 클릭과 드래그 두 방식을 함께 지원하고 싶을 때
- 선택한 파일을 옆의 FileChip 과 나란히 보여줄 때

## Props

<Controls />

| Prop          | Description                                                             |
| ------------- | ----------------------------------------------------------------------- |
| `label`       | 접근 가능한 이름의 뿌리입니다. `aria-label` 문구가 여기서 만들어집니다. |
| `title`       | 첫째 줄 안내 문구입니다. 예: '파일 선택 또는 끌어다 놓기'.              |
| `meta`        | 형식·크기 안내 둘째 줄입니다. 비우면 렌더하지 않습니다.                 |
| `accept`      | 탐색기의 형식 필터입니다. 보증이 아니므로 검증은 호출부가 합니다.       |
| `describedBy` | 오류/힌트 문단의 id 를 잇습니다. 문단 자체는 호출부가 소유합니다.       |
| `disabled`    | 클릭·드롭을 함께 막고 흐리게 표시합니다.                                |
| `isInvalid`   | 테두리를 danger 색으로 바꿉니다. 오류 문구는 그리지 않습니다.           |

파일 검증(확장자·용량·해상도)과 업로드는 호출부가 담당합니다. 이 컴포넌트는 파일을 고르는 일까지만 합니다.

## Behavior

- 탐색기 선택과 드래그·드롭 모두 같은 `onSelect` 로 파일 한 건을 내보냅니다.
- 드래그 중에는 테두리가 primary 색으로 강조됩니다.
- `disabled` 는 클릭과 드롭을 함께 막습니다.
- `isInvalid` 는 테두리 색만 바꾸고, 오류 문구는 `describedBy` 로 이은 문단이 담당합니다.
- 같은 파일을 다시 골라도 다시 발화합니다(선택 후 입력 값을 비웁니다).

## Accessibility

- `Tab` 으로 포커스, `Enter`·`Space` 로 탐색기를 엽니다.
- `aria-label` 이 클릭·드래그 두 조작 경로를 함께 안내합니다.
- 진짜 `<input type="file">` 은 숨겨 두어 탭 정지점이 버튼 하나만 남습니다.
