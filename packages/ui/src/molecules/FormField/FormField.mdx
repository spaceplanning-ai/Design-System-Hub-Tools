# FormField

라벨·도움말·에러를 입력 컨트롤에 감싸는 폼 필드 래퍼입니다.

## 언제 사용하나요?

- 입력 컨트롤에 라벨을 붙일 때
- 필수 표식이나 글자수 카운터가 필요할 때
- 에러 메시지나 보조 힌트를 함께 보여줄 때
- 라벨 옆 ⓘ 도움말을 제공할 때

## Props

<Controls />

| Prop       | Description                                                                    |
| ---------- | ----------------------------------------------------------------------------- |
| `htmlFor`  | 컨트롤의 id — 라벨과 컨트롤을 잇습니다. 힌트·에러 `<p>` 의 id 도 여기서 파생됩니다. |
| `label`    | 필드 라벨 텍스트입니다. 컨트롤의 접근 가능한 이름이 됩니다.                     |
| `required` | 필수 필드 표식(`*`)을 붙이고, 자식 컨트롤에 `aria-required` 를 넣습니다.       |
| `error`    | 인라인 에러 메시지입니다. 값이 있으면 힌트 대신 에러를 표시합니다.             |
| `hint`     | 보조 안내 문구입니다. 에러가 없을 때만 표시됩니다.                             |
| `counter`  | 라벨 오른쪽에 표시할 글자수 문자열(`'12/500'` 등)입니다.                       |
| `help`     | 라벨 옆 ⓘ 도움말 패널의 본문입니다.                                            |
| `children` | 폼 컨트롤 슬롯 — 감싸지 않고 그대로 렌더합니다.                                |

동반 헬퍼 `errorIdOf(htmlFor)` · `hintIdOf(htmlFor)` 로 힌트·에러 `<p>` 의 id 를 얻어 컨트롤의
`aria-describedby` 에 물릴 수 있습니다.

## Behavior

- `error` 가 있으면 `role=alert` 로 에러 `<p>` 를 그리고, 힌트는 숨깁니다.
- 에러는 색뿐 아니라 메시지 텍스트로도 전달합니다.
- `required` 표식(`*`)은 장식이며, 필수 여부는 자식 컨트롤의 `aria-required` 로 전달됩니다.

## Accessibility

<Story of={FormFieldStories.AriaWiring} />

- 라벨은 `htmlFor` 로 컨트롤과 연결됩니다.
- 힌트·에러 `<p>` id 를 `aria-describedby` 에 물려 컨트롤에 연결합니다.
- 컨테이너 `<div>` 자체에는 ARIA role 을 두지 않습니다.
- `dir="rtl"` 문서 방향을 따릅니다.
