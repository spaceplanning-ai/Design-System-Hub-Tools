# Pagination

<code>molecule</code> · <code>Navigation</code> · <code>taxonomyItem: pagination</code> · <code>v1.1.0</code> ·
<code>status: beta</code>

페이지네이션 — 범위 요약('전체 N건 중 x–y') · 이전 / 번호 창 / 다음 · 페이지 크기 선택.

**현재 페이지 주변 최대 5개 번호만** 보여준다(전부 그리면 줄이 넘친다).
**도메인을 모른다** — 회원·운영자·적립금 내역 어느 목록이든 `page`·`totalPages`·`onChange` 와
nav 접근성 `label` 만 받는다. 출처: `apps/admin/src/shared/ui/Pagination.tsx` (소비 11곳).

| ----------------- | -------- | -------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `page`            | `number` | — (필수)             | 현재 페이지 (1-based). **번호 창이 이 값을 가운데 두려 민다.**                                                         |
| `totalPages`      | `number` | — (필수)             | 전체 페이지 수. **1 이하이면 컴포넌트가 렌더되지 않는다** (단, 아래 예외 참조).                                        |
| `label`           | `string` | `'회원 목록 페이지'` | nav 의 접근성 이름(`aria-label`). 회원 목록이 기본값 — 다른 목록이 재사용할 때만 바꾼다.                               |
| `total`           | `number` | `0`                  | 전체 레코드 수 — 범위 요약의 N. **`pageSize` 와 함께 줘야 의미가 있다.** 데이터 prop.                                  |
| `pageSize`        | `number` | `0`                  | 페이지당 행 수. **`0`(기본)이면 범위 요약과 크기 선택을 그리지 않는다** — 아래 opt-in 절 참조. 데이터 prop.            |
| `pageSizeOptions` | `array`  | `[]`                 | 페이지 크기 선택지(예: `[10, 25, 50, 100]`). **비어 있으면(기본) 크기 선택 `<select>` 를 그리지 않는다.** 데이터 prop. |
| `sizeLabel`       | `string` | `'페이지당'`         | 크기 선택 `<select>` 의 가시 라벨.                                                                                     |

### 범위·크기 표면은 opt-in — 하위호환 장치

계약 `description`: 한국 ERP grid 는 가시 record 범위와 조정 가능한 page size 를 기대한다(ERP-05).
그러나 **기존 소비자 11곳은 `page`·`totalPages`·`label`·`onChange` 만 넘긴다** — 그래서 범위/크기 표면은
**`pageSize` 를 받았을 때만** 그린다(`pageSize` 기본 `0` = 미지정).

`pageSize` 미지정이면 렌더 결과가 **1.0.0 과 완전히 동일하다**(번호 줄만, `totalPages ≤ 1` 이면 `null`).

**`pageSize` 와 `pageSizeOptions` 는 서로 다른 두 표면을 따로 켠다** — 범위 요약만 쓰는 소비자를 위해서다.

### `totalPages ≤ 1` 이어도 범위는 남는다

계약 `description`: `pageSize` 를 받은 경우엔 단일/0 페이지에서도 범위 요약('전체 0건' · '전체 3건 중 1–3')을
그린다 — **번호 줄만 감춘다.** 0건에서 '전체 0건'을 보여주는 것이 ERP 운영자에게 필요한 정보다.

## 이벤트

| 이벤트             | payload  | 의미                                                                                                             |
| ------------------ | -------- | ---------------------------------------------------------------------------------------------------------------- |
| `onChange`         | `number` | 선택된 페이지 번호. **이전/다음/번호 버튼 모두 이 콜백으로 귀결된다.**                                           |
| `onPageSizeChange` | `number` | 고른 새 page size. **페이지 되감기(보통 1로)는 호출부가 소유한다** — 이 컴포넌트는 값만 알린다(도메인을 모른다). |

둘 다 `blockedWhen` 이 없다.

## States

계약 `states`: `default` · `hover` · `focus-visible` · `disabled`.

## 접근성 계약

- **role**: `navigation`
- **keyboard**: `Tab` · `Enter` · `Space`
- **focusVisible**: `true`
- **contrastMin**: 4.5

계약 `a11y.aria`:

- `aria-label`: nav 의 접근성 이름 — `label` prop.
- `aria-current`: 현재 페이지 번호 버튼에 `aria-current="page"`.
- `disabled`: 첫 페이지에서 '이전', 마지막 페이지에서 '다음' 버튼은 **native `disabled`.**
- `range-summary`: 범위 요약은 **`role=status`(`aria-live=polite`)로 그린다** — 페이지/크기를 바꾼 뒤
  '지금 몇 번째 행을 보고 있는지'가 스크린리더에도 전달돼야 한다.
  시각 사용자는 텍스트로, AT 사용자는 announce 로 **같은 정보를 받는다.**
- `size-select`: 크기 선택은 `<label htmlFor>` 로 가시 라벨(`sizeLabel`)과 연결한 네이티브
  `<select>`(SelectField) 다 — **라벨 없는 select 를 만들지 않는다.** `useId` 로 id 를 파생해
  한 화면에 여러 Pagination 이 있어도 충돌하지 않는다.
- `nav-scope`: **범위 요약과 크기 선택은 nav 밖(감싸는 `<div>`)에 둔다** — nav 의 접근성 이름은
  '페이지 이동' 랜드마크의 것이고, 요약/크기 선택은 페이지 이동 링크가 아니다.
