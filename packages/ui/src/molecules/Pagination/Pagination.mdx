# Pagination

목록을 여러 페이지로 나누어 이동하게 해주는 컴포넌트입니다.

## 언제 사용하나요?

- 목록이 한 화면에 다 들어오지 않을 때
- 페이지 단위로 데이터를 넘겨 보여줄 때
- 페이지당 표시 개수를 사용자가 고르게 할 때
- '전체 N건 중 x–y' 처럼 보고 있는 범위를 알려줄 때

## Props

<Controls />

| Prop              | Description                                                          |
| ----------------- | ------------------------------------------------------------------- |
| `page`            | 현재 페이지 번호입니다 (1부터 시작).                                |
| `totalPages`      | 전체 페이지 수입니다. `1` 이하이면 번호 줄을 그리지 않습니다.       |
| `label`           | 페이지 이동 영역의 접근성 이름입니다 (`aria-label`).                |
| `total`           | 전체 레코드 수입니다. 범위 요약의 N 값으로 쓰입니다.                |
| `pageSize`        | 페이지당 행 수입니다. 값을 주면 범위 요약과 크기 선택이 켜집니다.   |
| `pageSizeOptions` | 페이지 크기 선택지입니다 (예: `[10, 25, 50, 100]`).                 |
| `sizeLabel`       | 크기 선택 상자의 라벨입니다.                                        |
| `onChange`        | 페이지를 옮길 때 새 페이지 번호와 함께 호출됩니다.                  |
| `onPageSizeChange`| 페이지 크기를 바꿀 때 새 크기와 함께 호출됩니다.                   |

## Behavior

- 번호 창은 현재 페이지 주변 최대 5개만 보여줍니다 (전부 그리면 줄이 넘칩니다).
- 첫 페이지에서는 '이전', 마지막 페이지에서는 '다음' 버튼이 잠깁니다.
- `totalPages` 가 `1` 이하이면 번호 줄이 사라집니다.
- `pageSize` 를 주면 범위 요약과 크기 선택이 함께 나타납니다. 안 주면 번호 줄만 그립니다.
- `pageSize` 가 있으면 0건·단일 페이지에서도 '전체 0건' 같은 범위 요약은 남습니다.

## Accessibility

- `Tab` 으로 버튼을 이동하고 `Enter` · `Space` 로 누릅니다.
- 현재 페이지 버튼에 `aria-current="page"` 가 붙습니다.
- 잠긴 이전/다음 버튼은 native `disabled` 입니다.
- 범위 요약은 상태 알림으로 안내되어 페이지를 바꿔도 스크린리더에 전달됩니다.

## Notes

- 페이지 크기를 바꾼 뒤 1페이지로 되감는 처리는 호출부가 담당합니다.
