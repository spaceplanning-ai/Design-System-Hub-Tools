# PasswordField

<code>molecule</code> · <code>Inputs</code> · <code>taxonomyItem: password</code> · <code>v1.1.0</code> · <code>status: beta</code>

비밀번호 입력 필드 — TextField + **표시/숨김 토글 버튼.**

표시/숨김은 순수 표현 관심사이므로 이 컴포넌트가 소유하되(**폼 상태로 끌어올리지 않는다**),
계약상 `revealed` 는 제어 가능한 prop 으로 노출한다. **토글 전환 시 입력값·커서 위치를 유지한다.**
출처 구현: `apps/admin/src/pages/login/components/PasswordField.tsx`.

## Props

<Controls />

| prop | 타입 | 기본값 | 의미 |
|---|---|---|---|
| `id` | `string` | — (필수) | input 의 id. `label` 의 `htmlFor` 및 **토글 버튼의 `aria-controls`** 가 참조한다. |
| `label` | `string` | — (필수) | 필드 레이블. 시각적으로 노출되며 input 의 접근 가능한 이름이 된다. |
| `value` | `string` | — (필수) | 제어 컴포넌트 입력값. |
| `error` | `string` | `''` | 인라인 에러 메시지. 값이 있으면 `aria-invalid` + `aria-describedby` 연결. |
| `disabled` | `boolean` | `false` | 비활성. **입력·토글 모두 차단** + `aria-disabled`. |
| `required` | `boolean` | `false` | 필수 입력. native `required` → `aria-required`. **레이블에 시각 마커(\*)를 주입하지 않는다** — 아래 절 참조. |
| `name` | `string` | `''` | 폼 제출 키이자 **비밀번호 관리자의 필드 판정 근거.** 실사용: LoginForm(`name="password"`). |
| `autoComplete` | `string` | `''` | 자동완성 힌트 (`current-password` · `new-password`). **없으면 비밀번호 관리자 채우기가 퇴행한다.** |
| `placeholder` | `string` | `''` | 보조 예시 텍스트. |
| `revealed` | `boolean` | `false` | 표시/숨김 상태. `true` 면 `input type=text`(평문) + 눈 감김 아이콘, `false` 면 `type=password` + 눈 아이콘. |

### 계약이 자기 자신을 정정한 항목 — `required` 오기(erratum)

계약 `props.required.description` 이 직접 적은 것이다:

> **이 문장의 이전 판("레이블에 필수 표식")은 오기(erratum)였다.**

TextField `required` 와 **동일 판정**: 라벨 `textContent` = 접근 가능한 이름이며, 마커는 `getByLabel`
정확일치를 깨고 오너 확정 로그인 화면에 없던 표식을 만든다.

### `placeholder` — 끊겨 있던 표면

계약 `props.placeholder.description`: TextField 에는 있으나 **PasswordField 가 전달하지 않아 끊겨 있던
표면**이다 — 자식 TextField 로 그대로 내려보낸다.

### 계약이 명시한 예외 — `ref`

input 참조는 계약 prop 이 아니라 **`forwardRef` 로 노출한다** (TextField 와 동일 판정 —
현행 구현이 커서 복원을 위해 내부 ref 를 이미 갖고 있으므로 그것을 그대로 전달한다).
**호출부가 `document.getElementById(id)` 로 DOM 을 더듬는 우회를 없앤다.**

## 접근성 계약

- **role**: `textbox`
- **keyboard**: `Tab` · `Enter` · `Space`
- **focusVisible**: `true`
- **aria-disabled**: `disabled` 일 때
- **contrastMin**: 4.5

계약 `a11y.aria`:

- `aria-invalid`: `error` 가 빈 문자열이 아닐 때 `true`.
- `aria-describedby`: `error` 가 있을 때 에러 메시지 요소 id.
- `aria-required`: `required` 일 때 `true`.
- `aria-pressed`: **토글 버튼** — `revealed` 값.
- `aria-controls`: **토글 버튼** — input 의 `id`.
- `aria-label`: **토글 버튼** — `revealed` 에 따라 `'비밀번호 숨기기'` / `'비밀번호 표시'`.
