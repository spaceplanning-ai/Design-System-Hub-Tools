# PasswordField

비밀번호 입력 필드에 표시/숨김 토글 버튼을 더한 컴포넌트입니다.

## 언제 사용하나요?

- 로그인·회원가입 등에서 비밀번호를 입력받을 때
- 입력한 비밀번호를 잠깐 눈으로 확인하고 싶을 때
- 인라인 에러 메시지와 함께 필수/선택 입력을 다룰 때

## Props

<Controls />

| Prop           | Description                                                                 |
| -------------- | --------------------------------------------------------------------------- |
| `id`           | input 의 id. 라벨과 토글 버튼이 이 값을 참조합니다. (필수)                   |
| `label`        | 필드 라벨. input 의 접근 가능한 이름이 됩니다. (필수)                        |
| `value`        | 제어 컴포넌트 입력값. (필수)                                                 |
| `error`        | 값이 있으면 인라인 에러 메시지가 표시됩니다.                                 |
| `disabled`     | 입력과 토글을 모두 차단합니다.                                              |
| `required`     | 필수 입력 표시. 라벨에 별표(\*) 마커는 붙지 않습니다.                        |
| `name`         | 폼 제출 키. 비밀번호 관리자가 이 필드를 인식하는 근거가 됩니다.             |
| `autoComplete` | 자동완성 힌트(`current-password` · `new-password`). 비밀번호 채우기에 필요. |
| `revealed`     | 표시/숨김 상태. `true` 면 평문으로 보입니다.                                 |

## Behavior

- 토글 버튼을 누르면 `password` ↔ `text` 로 전환됩니다.
- 전환 후에도 입력값과 커서 위치가 그대로 유지됩니다.
- `disabled` 면 입력·`onChange`·`onBlur`·토글이 모두 막힙니다.

## Accessibility

- 라벨은 `id` 로 input 과 연결되어 접근 가능한 이름이 됩니다.
- 토글 버튼에는 상태에 따라 `비밀번호 표시` / `비밀번호 숨기기` 접근 이름이 붙습니다.
- 에러가 있으면 `aria-invalid` 와 `aria-describedby` 로 메시지가 연결됩니다.
- Tab 은 입력 → 토글 버튼 순으로 포커스를 옮깁니다.
