# Result

<code>molecule</code> · <code>Feedback</code> · <code>taxonomyItem: result-page</code> · <code>v1.0.0</code> · <code>status: beta</code>

결과 화면 — 요청한 내용을 **그릴 수 없을 때 그 자리를 대신 채우는** 상태 블록.
제목 한 줄, 설명 한 문단, 신고용 참조 코드, 빠져나갈 액션으로 이루어진다.

## Props

<Controls />

| prop | 타입 | 기본값 | 의미 |
|---|---|---|---|
| `title` | `string` | — (필수) | 무슨 일이 일어났는지 한 줄로. `h2` 로 렌더된다. **제품 카피라 DS 가 만들지 않고 받는다.** |
| `description` | `string` | `''` | 제목을 보충하는 한 문단. **빈 문자열이면 렌더하지 않는다.** |
| `reference` | `string` | `''` | 신고에 붙일 짧은 상관관계 코드 (EXC-20). 빈 문자열이면 렌더하지 않는다. |
| `actions` | `slot` | `null` | 빠져나갈 수단 — `<Button>` 하나 또는 여럿. **주지 않으면 액션 줄 자체를 그리지 않는다.** |

## 무엇이 DS 이고 무엇이 앱인가

계약 `description` 이 세운 경계다. `Sidebar` 가 정한 기준을 그대로 따른다 — **DS 는 그리고 앱이 정한다.**

| 관심사 | 소유 | 근거 |
|---|---|---|
| 블록의 시각·배치 | **DS** | 이 컴포넌트 |
| 무슨 일이 일어났는지의 문구 | 앱 | `title` · `description` 으로 들어온다 |
| 오류 객체에서 참조 코드 추출 | 앱 | HTTP 오류 모양을 아는 쪽이 계산해 문자열로 넘긴다 |
| 어디로 빠져나갈지와 그 이동 | 앱 | `actions` 슬롯 — 라우터를 아는 쪽이 `<Button>` 을 넣는다 |

문구를 DS 가 들면 다른 제품에서 쓸 수 없다.

## Empty 와 무엇이 다른가

| | `Empty` | `Result` |
|---|---|---|
| 언제 | data view 가 0행일 때 | 화면 전체가 실패했을 때 |
| 사유의 종류 | **유한하다** (없음/검색/필터) | 유한하지 않다 (렌더 예외·권한·점검·정지…) |
| 문구 | **스스로 조립한다** (`label` + context) | **받는다** (`title`·`description`) |
| 정렬 | 가운데 (표 안을 채운다) | 왼쪽 (본문을 대신한다) |

두 계약이 갈라지는 지점이 여기다.

## 빈 문자열 센티널 — 줄을 지운다

`description` · `reference` 는 빈 문자열이 곧 '그리지 않음' 이다.
참조 코드가 없는 결과 화면(권한 없음 등)이 **빈 줄을 남기지 않게** 하는 것이 이 센티널의 목적이다.

## 상태(status) 축을 만들지 않았다

403 은 자물쇠, 500 은 경고 삼각형처럼 **사유별 아이콘·색**을 주는 설계가 흔하다.
그러나 승격 원본인 어드민의 두 화면은 지금 **시각적으로 완전히 동일**하고 사유는 오직 제목 문구로만 갈린다.

없는 구분을 계약에 넣으면 검증되지 않은 축이 조합 행렬만 늘리고 승격이 픽셀을 바꾼다 —
`Sidebar` 가 접기(collapse) 축을 만들지 않은 것과 같은 규율이다.
**아이콘·색으로 사유를 구분할지는 제품 판단이므로 오너 검수 항목으로 올렸다.**

## 무엇을 보이지 않는가 — EXC-20

raw 서버 body · stack trace · status 코드를 **산문으로 노출하지 않는다.**
내부 구조를 흘리고 고장처럼 읽히기 때문이다. 대신 로깅된 오류와 대조 가능한 짧은 코드 하나만 보인다.

**계약에 그런 prop 이 없는 것이 그 규율의 강제 수단이다.**

참조 코드는 운영자가 그대로 복사해 티켓에 붙이는 것이 유일한 용도이므로
`font-variant-numeric: tabular-nums` 로 자릿수를 고정하고 `user-select: all` 로 한 번에 통째로 잡히게 한다.

## States

계약 `states` 는 `default` 하나뿐이다 — 위 분기는 state 가 아니라 **prop 조합**이다.

## 접근성 계약

- **role**: `alert`
- **focusVisible**: `true`
- **contrastMin**: 4.5

계약 `a11y.aria`:

- `role`: `alert`(=`aria-live` assertive) — 화면이 그려지지 못했다는 사실은 **즉시** 전달돼야 한다.
  이 값은 승격 전 두 화면(`ErrorScreens.tsx`)이 이미 쓰던 것을 **그대로 승계**한 것이며 동작을 바꾸지 않았다.
- `heading`: 제목은 `h2` 다. 페이지의 `h1` 은 화면 이름을 가진 `Header` 가 소유하므로
  여기서 `h1` 을 더 만들면 **한 문서에 제목이 둘**이 된다.
- `no-raw-detail`: stack trace·서버 body·status 코드를 노출하지 않는다.
