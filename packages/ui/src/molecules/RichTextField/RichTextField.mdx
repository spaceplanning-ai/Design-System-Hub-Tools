# RichTextField

서식 있는 텍스트 편집기 — 볼드·기울임·제목·목록·링크 등을 담는 본문 입력 필드입니다.

## 언제 사용하나요?

- 상품 상세설명처럼 서식이 필요한 긴 본문을 입력받을 때
- 굵게·기울임·제목·글머리/번호 목록·링크를 사용자가 직접 넣어야 할 때
- 평문만 받으면 되는 경우에는 TextareaField 를 씁니다.

## Props

<Controls />

| Prop          | Description                                                                        |
| ------------- | --------------------------------------------------------------------------------- |
| `label`       | 필드 레이블입니다.                                                                |
| `value`       | 현재 본문 값입니다. HTML 문자열이며, 제어 컴포넌트로 사용합니다.                   |
| `onChange`    | 본문이 바뀔 때 새 HTML 문자열을 넘겨줍니다. `disabled` 상태에서는 호출되지 않습니다. |
| `maxLength`   | 최대 글자 수입니다. 서식 마크업이 아니라 눈에 보이는 평문 길이만 셉니다.           |
| `required`    | 필수 필드로 표시하고 레이블에 마커(\*)를 붙입니다.                                 |
| `disabled`    | 편집을 막고 툴바 버튼을 비활성화합니다.                                            |
| `error`       | 인라인 오류 메시지입니다. 값이 있으면 오류 상태로 표시합니다.                      |
| `hint`        | 보조 안내 문구입니다. 오류가 없을 때만 표시됩니다.                                 |
| `placeholder` | 본문이 비어 있을 때 보여 줄 예시 텍스트입니다.                                     |
| `rows`        | 편집 영역의 최소 표시 행 수입니다.                                                 |

## Behavior

- 툴바 버튼으로 굵게·기울임·제목·목록·링크 등의 서식을 넣습니다. 켜진 서식은 버튼에 눌린 상태로 표시됩니다.
- `disabled` 이면 편집 영역이 잠기고 툴바 버튼도 비활성화되어 `onChange` 가 발생하지 않습니다.
- 글자 수 카운터는 서식을 제외한 평문 길이(`N/max`)만 셉니다.

## Accessibility

- 편집 영역은 레이블로부터 접근 가능한 이름을 얻습니다.
- 툴바 버튼은 `Tab`·`Shift+Tab` 으로 순서대로 이동하며, 현재 서식 상태를 스크린리더에 알립니다.
- 오류가 있으면 오류 메시지와 연결되고, 필수 필드는 접근성 트리에 필수로 노출됩니다.
- RTL(오른쪽에서 왼쪽) 문서 방향에서도 툴바·본문 정렬이 방향을 따릅니다.

## Notes

- 툴바의 이미지 버튼은 로컬 미리보기만 넣고 실제 업로드는 하지 않습니다.
