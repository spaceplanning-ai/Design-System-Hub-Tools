# SearchField

돋보기 아이콘을 겹친 검색 입력 컴포넌트입니다.

## 언제 사용하나요?

- 목록 상단 툴바에서 검색어를 입력받을 때
- 이름·SKU·주소 등으로 항목을 걸러낼 때
- 시각적으로 감춘 라벨로 '무엇을 검색하는지' 알려야 할 때

## Props

<Controls />

| Prop          | Description                                                          |
| ------------- | ------------------------------------------------------------------- |
| `value`       | 현재 검색어입니다. 제어 컴포넌트로 사용합니다.                       |
| `label`       | 시각적으로 감추지만 스크린 리더가 읽는 이름입니다 ('공지 제목 검색' 등). |
| `placeholder` | 입력 안내 문구입니다. 미지정 시 '검색'.                             |
| `onChange`    | 입력이 바뀔 때 새 문자열 값을 넘겨줍니다.                            |

## Behavior

- `onChange` 는 네이티브 이벤트가 아니라 새 문자열 값(`event.target.value`)을 넘깁니다 — `onChange={setKeyword}` 형태로 바로 연결합니다.
- `name`·`aria-*`·`autoFocus` 등 표준 `<input>` 속성은 그대로 전달됩니다.
- 긴 검색어는 레이아웃을 밀지 않고 입력 안에서 가로로 스크롤됩니다.

## Accessibility

- 라벨은 감추되 DOM 에 남겨 입력에 접근 가능한 이름을 줍니다.
- 네이티브 `<input type="search">` 라 `searchbox` 역할로 동작합니다.
- 돋보기 아이콘은 `aria-hidden` 장식이며 클릭은 아래 입력으로 통과합니다.
- `Tab` 으로 포커스가 들어오면 포커스 링이 표시됩니다.
