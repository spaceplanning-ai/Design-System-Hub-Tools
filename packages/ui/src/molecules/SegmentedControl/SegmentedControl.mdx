# SegmentedControl

몇 개 옵션 중 하나를 고르는 분절 토글입니다.

## 언제 사용하나요?

- 서로 배타적인 2~5개 옵션 중 하나를 고를 때
- 목록 필터·보기 방식처럼 값이 바로 반영되는 전환일 때
- 라디오 버튼보다 압축된 토글 형태가 어울릴 때

## Props

<Controls />

| Prop        | Description                                                          |
| ----------- | ------------------------------------------------------------------- |
| `value`     | 선택된 세그먼트의 id. `options[].id` 중 하나여야 합니다.            |
| `options`   | 세그먼트 목록 `{ id, label, icon?, labelHidden? }`.                 |
| `size`      | 세그먼트 크기입니다. `sm` 또는 `md`.                                |
| `disabled`  | 그룹 전체를 비활성화합니다.                                         |
| `ariaLabel` | 그룹의 접근 가능한 이름입니다. 시각 레이블이 없으므로 필수입니다.   |
| `onChange`  | 세그먼트를 선택하면 선택된 id 로 호출됩니다.                        |

## Behavior

- 세그먼트를 누르면 `onChange` 가 그 id 로 발화합니다.
- 화살표 키는 선택과 포커스를 함께 옮깁니다 (선택이 포커스를 따름).
- `disabled` 이면 선택이 차단됩니다.

## Accessibility

- `ArrowLeft` · `ArrowRight` 로 세그먼트를 이동합니다.
- 현재 선택된 세그먼트만 탭 순서에 들어옵니다 (그룹당 한 칸).
- 아이콘만 보이는 세그먼트도 `label` 이 접근 가능한 이름으로 남습니다.
