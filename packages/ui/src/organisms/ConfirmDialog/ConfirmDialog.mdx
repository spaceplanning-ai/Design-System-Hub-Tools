# ConfirmDialog

<code>organism</code> · <code>Dialogs & Overlays</code> · <code>taxonomyItem: confirm-dialog</code> · <code>v1.0.0</code> · <code>status: beta</code>

확인 다이얼로그 — **CRUD 확인의 단일 창구.** Modal 을 조립해 **의도(`intent`)별 톤·라벨·아이콘을 강제한다**
(ADR-0003).

**도메인을 모른다** — 무엇을 확인하는지는 `title`/`message` 로 받는다.

## `intent` 가 톤·기본 라벨·아이콘을 정한다

계약 `description`: 호출부가 매번 색과 문구를 고르면 **같은 '삭제'가 화면마다 다른 색으로 보인다** —
그래서 **의도만 받는다.**

| intent | 기본 라벨 | 톤 | 아이콘 |
|---|---|---|---|
| `create` | '만들기' | primary | 원+ |
| `update` | '저장' | primary | 연필 |
| `delete` | '삭제' | **danger** | 휴지통 |
| `discard` | '나가기' | **danger** | 삼각형 |

## 실패는 다이얼로그를 닫지 않는다

계약 `description`: `error` 를 주면 다이얼로그 안에 danger 배너(Alert)로 뜨고 **확인 버튼이 되살아난다** —
**재클릭이 곧 재시도다.**

> 모달이 떠 있는 동안 **토스트는 시선 밖이고 닫히면 사라지므로** 여기서는 인라인 배너를 쓴다.

## `busy` 잠금

`busy` 면 확인 버튼이 비활성(`aria-busy`)되어 중복 클릭을 막는다 — **확인은 요청 1건만 만든다.**
**busy 중에도 취소/Esc/딤 클릭은 살아 있다**(진행 중 요청의 abort 경로는 호출부 `onCancel` 이 소유).

## 취소 토스트는 앱의 것

계약 `description`: 취소 시 '작업이 취소되었습니다' 토스트는 **앱(shared/ui 어댑터 + `useToast`)이 얹는다** —
**DS 는 토스트 큐를 모른다.** DS 는 `onCancel` 을 부르기만 한다.

## 이벤트 — 계약이 명시한 예외

계약 `description`: `[imperative props — 계약 밖 경계]` `onConfirm`(필수)·`onCancel`(필수)은
**명령형 배선이라 Figma 대응이 없다.** `error`/`confirmLabel`/`cancelLabel` 은
`exactOptionalPropertyTypes` 경계에서 `undefined` 허용으로 넓힌다 (B2 선례).

| 이벤트 | payload | blockedWhen |
|---|---|---|
| `onConfirm` | `void` | `["busy"]` — 구현은 확인 버튼을 `disabled` 로 잠근다 |
| `onCancel` | `void` | 없음 — **busy 중에도 살아 있다** |

## States

계약 `states`: `default` · `loading` · `error`.

## 접근성 계약

- **role**: `dialog`
- **keyboard**: `Escape` · `Enter` · `Tab` · `Shift+Tab`
- **focusVisible**: `true`
- **aria-busy**: 확인 버튼은 busy 일 때 `aria-busy=true` — **스피너 없이 `aria-busy` 만**,
  라벨이 '처리 중…' 으로 바뀐다.
- **contrastMin**: 4.5

계약 `a11y.aria`:

- `dialog`: **Modal 이** `role=dialog` + `aria-modal` + `aria-labelledby`(제목) +
  `aria-describedby`(본문 메시지)를 제공한다. ConfirmDialog 는 `message` 요소에 id 를 부여하고
  **`Modal.describedBy` 로 넘겨** open 시 제목과 확인 문구가 함께 announce 되게 한다 (A11Y-02).
- `error`: `error` 가 비어 있지 않으면 **Alert(`role=alert`, `tone=danger`)로 즉시 통지한다.**

> Button 계약의 `a11y.aria.native-passthrough` 가 이 컴포넌트를 실사용 예로 든다 —
> *"ConfirmDialog 는 스피너 없이 `aria-busy` 만 필요로 한다"*. 두 계약이 서로를 가리킨다.

## 토큰

`color.text.default`(messageText) · `color.action.primary.default`(iconPrimary) ·
`color.feedback.danger.text`(iconDanger) · `space.3`(bodyGap) · `typography.label.md`.

`variantTokens.intent` 가 아이콘 색을 갈아끼운다: `create`/`update` → `color.action.primary.default`,
`delete`/`discard` → `color.feedback.danger.text`.

**나머지 시각은 Modal 이 소유한다** — 이 계약의 토큰이 적은 이유다.

`responsive.behavior` 는 `fluid`. 의존: `Modal` · `Alert` · `Button`.
