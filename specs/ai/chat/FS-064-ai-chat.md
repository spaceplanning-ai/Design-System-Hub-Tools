# FS-064. AI 에이전트 새 채팅

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 관리자가 `@데이터` 를 지목하고 조건을 적으면, 그 조건으로 저장된 데이터를 걸러 실제 행을 돌려준다 |
| 역할(주 사용자) | 시스템 관리자 · 운영자 |
| 진입 경로 | 좌측 GNB > AI > AI 에이전트 > 새 채팅 (`nav-config.ts`, `to: '/ai/chat'`) |
| 관련 화면 | FS-065 대화 목록 · FS-003 회원 관리 목록(결과 링크 대상) · FS-026 1:1 문의 · FS-041 상품 |
| 대응 SCR | SCR-064 (미작성 — §7) |
| 구현 경로 | `apps/admin/src/pages/ai/NewChatPage.tsx` |

### 1.1 이 화면에 없는 기능 (요구사항)

> **이 절이 이 화면의 핵심이다.** 아래는 '아직 안 만든 것' 이 아니라 **데이터가 없어서 만들 수 없는 것** 이다.
> 화면은 이 사실을 숨기지 않고 사용자에게 그대로 말한다.

| 없는 것 | 이유 |
|---|---|
| 언어 모델(LLM) 응답 | 이 앱에 모델도 백엔드도 없다. 답변은 결정적 파서(`_shared/parser.ts`)와 픽스처 조회로만 만들어진다. 연동 자리는 `data-source.ts` 의 `askAgent` 머리말에 명시 |
| 요약 · 분석 · 예측 · 추천 | 판단이 필요한 요청이다. 조회로 흉내 내면 근거 없는 문장이 답인 것처럼 보인다 — 파서가 `unsupported-intent` 로 **거절**한다 (`parser.ts` `UNSUPPORTED_VERBS`) |
| 자유 문장 대화 | `@` 멘션이 없으면 어떤 데이터를 봐야 할지 결정할 수 없다. `no-mention` 으로 거절하고 할 수 있는 것을 대신 보여준다 |
| 기간별 구매 조회 | 회원 픽스처의 `totalPurchase` 는 **누적값이고 날짜가 없다**(`shared/fixtures/members.ts:126`). '이번달 구매' 는 계산할 수 없다 — §3 FS-064-EL-021 참조 |
| 대화 보관 | 저장소에 쓰지 않는다. 새로고침하면 사라진다 (FS-065 §1.1) |
| 응답 모드 3종(빠른·전문가·헤비) | 모델의 사고량을 고르는 축이다. 고를 수 있게 두면 방금 고른 것이 답을 바꿨다고 믿게 된다 — 비활성 + '미연결' 로 남긴다 (`_shared/modes.ts`) |
| 음성 입력 · 첨부 | 참조 디자인에 있으나 처리할 백엔드가 없다. 자리를 만들지 않는다 |

### 3.1 멘션 문법과 기간 결합 규칙

```
질의   := 멘션 조건* 지시어?
멘션   := '@' 도메인별칭
조건   := 기간 | 값조건 | 존재조건
기간   := '오늘' | '어제' | '이번주' | '이번달' | '지난달' | '올해' | '최근 N일'
값조건 := 필드값 라벨/별칭        (예: 'VIP' → 등급=vip)
존재조건 := 필드명사              (예: '구매' → 누적 구매액 > 0)
지시어 := 목록('뽑아줘'·'보여줘'…) | 개수('몇 명'·'건수'…). 없으면 목록으로 본다
```

| 도메인 | 별칭 | 걸 수 있는 축 |
|---|---|---|
| 회원 목록 | `@회원목록` `@고객목록` `@회원` `@고객` | 등급(값) · 가입일(**기간**) · 누적 구매액(존재) · 적립금(존재) |
| 상품 목록 | `@상품목록` `@상품` | 판매상태(값) · 전시상태(값). **기간 축 없음** — 상품 픽스처에 날짜가 없다 |
| 1:1 문의 | `@문의목록` `@문의` `@티켓` | 처리상태(값) · 우선순위(값) · 접수일(**기간**) |

**기간 결합** — 기간 표현은 혼자 뜻을 갖지 않는다. 표현 **뒤쪽** 12자 안에서 가장 가까운 필드 명사에 묶는다(한국어 수식은 앞에서 뒤로 간다). 못 찾으면 도메인 기본 날짜 필드에 붙는다.

| 입력 | 결합 대상 | 결과 |
|---|---|---|
| `이번달 가입한 VIP` | 가입일 (period 축 O) | 기간 조건이 걸린다 |
| `이번달 VIP` | 도메인 기본(가입일) | 기간 조건이 걸린다 |
| `이번달 구매 VIP` | 누적 구매액 (period 축 **X**) | **기간을 버리고** FS-064-EL-013 통지 + 나머지 조건으로 조회 |
| `@상품 이번달` | 결합 대상 없음, 기본 날짜 필드도 없음 | 기간을 버리고 도메인 단위 통지 |

> **사용자 예시 질의 `@고객목록 의 이번달 구매 VIP 뽑아줘` 의 판정**: '이번달' 이 '구매' 에 묶이고, 누적 구매액에는
> 기간 정보가 없다. 그래서 기간을 버리고 `등급 VIP · 누적 구매액 있음` 으로 조회한다(픽스처 기준 9건).
> 통지 문구가 표보다 먼저 뜨며, 후속 제안으로 `이번달 가입한 회원 보여줘` 를 준다.
> **이 화면이 누적 구매액을 '이번달 구매'로 표시하는 경로는 없다.**

### 3.2 응답 모드가 menu 인 근거 (listbox 아님)

| 판정 | 근거 |
|---|---|
| **menu** | 값(모드 4종)과 값이 아닌 명령(질의 문법 도움말)이 한 표면에 함께 있다. `listbox` 는 자식으로 `option` 만 가질 수 있어 이 구성을 표현할 수 없다. 어떤 폼 필드에도 묶이지 않는다 |
| 단일 선택 표현 | `menuitemradio` + `aria-checked` — '이 중 하나가 켜져 있다' 를 메뉴 안에서 말하는 표준 방법 |
| 대비 — `@` 자동완성은 combobox | 그쪽은 **값을 편집하는 텍스트 입력**에 붙은 목록이다. 포커스가 입력에 머물러야 계속 타이핑할 수 있으므로 `aria-activedescendant` 로 활성 항목만 가리킨다 |
