# BE-065. AI 에이전트 대화 목록 백엔드 기능 명세

## 1. 개요

| 항목 | 내용 |
|---|---|
| 대상 화면 | FS-065 AI 에이전트 대화 목록 (`/ai/conversations`) |
| 범위 | 대화 목록 조회 · 대화 삭제 · 보관(영속) 정책 |
| 범위 밖 | 질문 처리와 대화 상세 조회 — BE-064 가 소유한다 |
| 전제 | 인증은 세션 쿠키 기반. 모든 경로는 `/api` 프리픽스. 응답 본문은 `application/json; charset=utf-8`. 시각은 ISO 8601(오프셋 포함) |
| 프론트 어댑터 | `apps/admin/src/pages/ai/data-source.ts` |
| 도메인 타입 | `apps/admin/src/pages/ai/_shared/conversations.ts` |

> **엔드포인트를 새로 만들지 않는다.** 이 화면이 쓰는 두 호출은 BE-064-EP-01 · BE-064-EP-04 와 **같은 것**이다.
> 화면이 둘이라고 엔드포인트를 복제하면 목록의 정렬·페이징 규칙이 두 곳에서 갈라진다.
> 이 문서는 **이 화면 관점의 계약과 보관 정책**을 적고, 요청/응답 스키마는 BE-064 를 참조한다.

## 2. 공통 에러 봉투

BE-064 §2 와 동일하다. 봉투를 재정의하지 않는다.

### 2.1 공통 에러코드

BE-064 §2.1 을 그대로 따른다. 이 화면이 실제로 만나는 코드는 아래로 한정된다.

| HTTP | 에러코드 | 발생 조건 | 프론트 처리 |
|---|---|---|---|
| 401 | `UNAUTHENTICATED` | 세션 없음·만료 | `/login?returnUrl=` 로 이동 |
| 403 | `FORBIDDEN` | `/ai/conversations` read 권한 없음 | `ForbiddenScreen` |
| 403 | `CSRF_TOKEN_INVALID` | 삭제 시 CSRF 토큰 누락·불일치 | FS-065-EL-008 실패 토스트 |
| 400 | `VALIDATION_FAILED` | `ids` 빈 배열·100건 초과 | FS-065-EL-008 실패 토스트 |
| 429 | `RATE_LIMITED` | 분당 한도 초과 | FS-065-EL-007 / EL-008 |
| 500 | `INTERNAL_ERROR` | 서버 오류 | 조회 → EL-007, 삭제 → EL-008 |
| 504 | `REQUEST_TIMEOUT` | 조회 지연 | 조회 → EL-007 |

## 3. 인증 · 권한 모델

### 3.1 역할

| 역할 | 설명 | 비고 |
|---|---|---|
| `admin` | 본인 대화 조회·삭제 | 타인 대화를 보지 않는다 (§3.2) |
| `operator` | 본인 대화 조회·삭제 | 동일 |

> **대화는 소유자 단위로 격리된다.** `admin` 이라도 타인의 대화를 열람하지 않는다 — 대화에는 그 관리자가 무엇을 조회했는지가 남고, 그것은 개인의 업무 기록이다. 감사 목적의 열람은 감사 로그(FS-060)의 일이지 이 화면의 일이 아니다.

### 3.2 403 vs 404 은닉 정책

| 리소스 | 권한 부족 시 응답 | 근거 |
|---|---|---|
| 대화 컬렉션 `/api/ai/conversations` | **403** | 컬렉션 존재는 비밀이 아니다 |
| 삭제 대상 id | **404 를 쓰지 않는다** | 멱등 삭제라 없는 id 도 204 다. 타인 소유 id 도 204 로 응답해 **존재를 드러내지 않는다**(실제로는 지우지 않는다) |

### 3.3 CSRF

삭제(DELETE)는 `X-CSRF-Token` 헤더를 요구한다. 누락·불일치면 403 `CSRF_TOKEN_INVALID`.

### 3.4 타임아웃 상한

| 계열 | 서버 처리 상한 | 초과 시 | 프론트 상한(권고) |
|---|---|---|---|
| 목록 조회 | 3초 | 504 | 10초 |
| 삭제 | 3초 | 504 | 10초 |

## 7. 핵심 판정 (근거를 남긴다)

### 7.1 엔드포인트를 복제하지 않는다

FS-064 좌측 레일과 FS-065 목록은 **같은 데이터의 두 표면**이다. 각자 엔드포인트를 가지면 정렬·페이징·건수 계산이 두 곳에서 갈라지고, 같은 화면 안에서 순서가 다른 두 목록이 보인다. 그래서 BE-064-EP-01/EP-04 를 공유하고 이 문서는 그 위의 요구만 적는다.

### 7.2 대화는 소유자 단위로 격리한다 — `admin` 도 예외가 아니다

대화에는 그 관리자가 무엇을 조회했는지가 남는다. 조회 이력의 열람은 감사 로그(FS-060)가 통제된 형태로 제공할 일이지, 대화 목록이 열어 줄 일이 아니다.

### 7.3 보관 정책은 백엔드가 정한다 — 지금은 보관하지 않는다

현재 구현은 메모리이며 새로고침하면 사라진다. 화면이 이 사실을 배너로 밝힌다(FS-065-EL-002). 서버가 붙으면 보관 기간과 자동 삭제를 정해야 하며, 그때 배너 문구를 함께 걷어낸다 — 문구가 남으면 이번에는 그것이 거짓이 된다.
