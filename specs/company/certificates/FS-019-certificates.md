# FS-019. 인증서/특허 관리 (목록·구분 필터·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 회사가 보유한 인증서·특허를 발급일 최근순으로 훑고, 구분(인증서/특허)으로 좁혀 보며, 개별 항목을 이미지와 함께 등록·수정·삭제(단건·일괄)한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 기업 관리 > 인증서/특허 (`/company/certificates`) |
| 포함 화면 | 목록 `/company/certificates` · 등록 `/company/certificates/new` · 수정 `/company/certificates/:id/edit` |
| 구현 경로 | `apps/admin/src/pages/company/certificates/**` |
| 대응 SCR | SCR-019 (미작성 — §7) |
| 공통 컴포넌트 | `shared/crud/{CrudListShell,CrudTable,useCrudList,useCrudForm,FormPageShell,FormFeedback,createCrudAdapter,parseFilter,requiredText,requiredImage}` · `shared/ui/{ConfirmDialog,SelectionBar,useRowSelection,useUnsavedChangesDialog,ToastProvider,FormField,SelectField,ImageUploadField,ImageThumb,StatusBadge,RowActions,SeqCell,RowSelectCell,SelectAllHeaderCell,Empty,Alert,Button}` · `shared/useRowNavigation` |
| 상세 화면 | **없다** — 읽기 전용 상세 라우트가 없고 행/수정 버튼이 곧바로 수정 폼으로 간다 |

> **이 화면에 없는 것 (지어내지 않기 위해 명시한다)**: 검색 입력 · 페이지네이션 · 정렬 컨트롤 · 재정렬 · 노출 토글 · 상세 조회 라우트 · **이미지 업로드 엔드포인트**. 목록은 발급일 내림차순 고정이며 전 행을 한 화면에 렌더한다(§7 #2). 필터는 구분 select 하나뿐이다.

> **⚠ 이미지의 실체**: 폼의 `ImageUploadField` 는 **파일을 서버로 보내지 않는다** — `URL.createObjectURL(file)` 로 만든 `blob:` URL 을 폼 값에 넣는다. 업로드 심이 없어 저장된 값은 그 탭에서만 유효하다(§7 #1).
