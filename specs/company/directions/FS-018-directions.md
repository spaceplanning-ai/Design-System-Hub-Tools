# FS-018. 오시는 길 (단일 문서 편집)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 고객 화면에 노출되는 회사 위치 정보(주소·상세주소·좌표·교통편) 1건을 조회하고 수정한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 기업 관리 > 오시는 길 (`/company/directions`) |
| 포함 화면 | 단일 편집 폼 `/company/directions` **하나뿐**. 목록·상세·등록 라우트가 없다(회사당 1건) |
| 구현 경로 | `apps/admin/src/pages/company/directions/**` (`DirectionsPage.tsx` · `data-source.ts` · `types.ts` · `validation.ts` · `directions.test.ts`) |
| 대응 SCR | SCR-018 (미작성 — §7) |
| 공통 컴포넌트 | `shared/crud/{DocumentFormShell,useDocumentQuery,useSaveDocument,createDocumentStore,requiredText}` · `shared/ui/{FormField,errorIdOf,controlStyle,hintStyle,useToast,useUnsavedChangesDialog,ConfirmDialog,Card,CardTitle,Alert,Button}` · `shared/form/zodResolver` · `shared/async.isAbort` |
| 검증 정본 | `validation.ts` 의 `directionsSchema` (zod/mini) |

> **지도 임베드는 미구현이다.** 좌표를 지도 SDK 로 렌더하는 코드가 이 앱에 없다. 화면에는 `aria-hidden` placeholder(FS-018-EL-009)와 안내문(FS-018-EL-011)만 있고, `DirectionsPage.tsx:214-215` 의 `// TODO(backend): 지도 임베드` 주석이 그 사실을 명시한다. 이 문서는 **없는 기능을 명세하지 않는다** — BE-018 에도 지도 관련 엔드포인트가 없다(BE-018 §4 EP-03 · §7.5).

### 4.1 공통 예외 규칙

| 규칙 | 내용 |
|---|---|
| 네트워크 단절 | 조회 실패는 FS-018-EL-014 가 폼을 대체한다. 저장 실패는 FS-018-EL-003 배너. 오프라인 감지·전용 문구는 없다 — §7 #10 |
| 세션 만료 | 조회·저장이 401 이면 앱 전역 인터셉터가 세션을 폐기하고 `/login?returnUrl=/company/directions&reason=session_expired` 로 보낸다. **편집 중인 입력은 보존되지 않는다** — §7 #11 |
| 요청 타임아웃 | 프론트 타임아웃 상한 없음 — §7 #12. abort 는 화면 이탈에서만 발생한다 |
| 중복 제출 | 저장 버튼은 `!dirty \|\| saving \|\| loading` 이면 비활성이다. **그러나 동기 제출 잠금이 없어** 검증(비동기)이 도는 틈의 두 번째 Enter 가 두 번째 요청을 만든다 — §7 #9 |
| 실패 통지의 자리 | ① 조회 실패는 폼을 대체하는 인라인 배너(FS-018-EL-014) ② 저장 실패는 카드 안 배너(FS-018-EL-003) ③ 저장 **성공**은 토스트 ④ abort 는 아무 통지도 하지 않는다 |
| 낙관적 업데이트 | 없다. 저장은 비관적(pessimistic) — 서버 응답 후에만 `reset` 하고 토스트를 띄운다 |
| 동시 조회 | 문서 조회는 동시에 1건만 유지된다(react-query, `staleTime` 30초 · 자동 재시도 없음 · 창 포커스 재조회 없음) |
| 권한 없음 | 역할 기반 분기 없음. 이 라우트의 **읽기** 권한이 없으면 앱 전역 가드가 폼 대신 403 화면을 그린다. **쓰기 권한은 화면에서 게이팅하지 않는다** — 저장 버튼이 그대로 보이고 눌리며, 서버 403 이 FS-018-EL-003 로 떨어진다(§7 #8) |
| 검증 시점 | 제출 시에만 검증한다(`zodResolver` + RHF 기본 `onSubmit` 모드). 입력 중에는 에러가 뜨지 않고, 한 번 에러가 뜬 필드는 재제출 때 다시 판정된다 |
| 저장 후 상태 | 성공하면 저장값으로 `reset` 해 '변경 없음' 으로 되돌린다 — 저장 버튼이 다시 비활성되고 이탈 가드가 풀린다. 화면은 이동하지 않는다 |
