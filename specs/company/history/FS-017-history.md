# FS-017. 연혁 관리 (목록·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 회사 연혁(연·월·내용)을 최근순으로 훑고, 개별 항목을 등록·수정·삭제(단건·일괄)한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 기업 관리 > 연혁 (`/company/history`) |
| 포함 화면 | 목록 `/company/history` · 등록 `/company/history/new` · 수정 `/company/history/:id/edit` |
| 구현 경로 | `apps/admin/src/pages/company/history/**` |
| 대응 SCR | SCR-017 (미작성 — §7) |
| 공통 컴포넌트 | `shared/crud/{CrudListShell,CrudTable,useCrudList,useCrudForm,FormPageShell,FormFeedback,createCrudAdapter,requiredText}` · `shared/ui/{ConfirmDialog,SelectionBar,useRowSelection,useUnsavedChangesDialog,ToastProvider,FormField,SelectField,TextareaField,RowActions,SeqCell,RowSelectCell,SelectAllHeaderCell,Empty,Alert,Button}` · `shared/format.formatNumber` · `shared/useRowNavigation` |
| 상세 화면 | **없다** — 이 도메인에는 읽기 전용 상세 라우트가 없고 행/수정 버튼이 곧바로 수정 폼으로 간다 |

> **이 화면에 없는 것 (지어내지 않기 위해 명시한다)**: 검색 입력 · 필터 · 페이지네이션 · 정렬 손잡이(재정렬) · 노출 토글 · 상세 조회 라우트. 목록은 연·월 내림차순 고정이며 전 행을 한 화면에 렌더한다(§7 #2).

| 낙관적 업데이트 | **없다.** 이 화면의 모든 쓰기는 비관적이다 — 서버 응답 후에만 캐시를 무효화한다 |
| 취소는 실패가 아니다 | abort 는 공유 판정(`isAbort`)으로 걸러 토스트·배너를 띄우지 않고, 일괄 실패 건수에서도 제외한다 |
| 권한 없음 | 역할 기반 분기가 화면에 없다. read 권한이 없으면 라우트 가드가 403 화면으로 대체하지만, **쓰기 버튼은 권한과 무관하게 렌더된다**(§7 #5). 서버 403 은 조회=배너, 쓰기=각 요소 실패로 떨어진다 |
| 행 선택의 수명 | 선택은 화면 로컬. 필터·검색·페이지가 없어 해제 계기는 '선택 해제' 버튼과 **일괄 삭제 전원 성공**뿐이다 |
| 정렬 | 연·월 내림차순 고정(같은 연·월은 id 역순). 사용자가 바꿀 수 없다 |
