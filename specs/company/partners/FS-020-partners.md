# FS-020. 파트너사 관리 (목록·등록/수정 모달·삭제)

> **이 문서는 '로고 목록' 공용 골격의 정본 기술이다.** 파트너사와 고객사는 `pages/company/logo-list/**` 한 모듈을 config(`resource`·`entityLabel`·`adapter`)만 바꿔 공유한다. FS-021(고객사)은 이 골격을 인용하고 차이만 기술하되, 요소 번호와 예외 표는 자기 번호(FS-021-EL-0NN)로 완결된다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 고객 화면에 노출되는 파트너사 로고를 이름으로 좁혀 찾고, 목록에서 바로 노출을 켜고 끄거나 드래그/키보드로 정렬 순서를 바꾸며, 개별 파트너사를 등록·수정·삭제(단건·일괄)한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 기업 관리 > 파트너사 (`/company/partners`) |
| 포함 화면 | 목록 `/company/partners` 하나. 등록·수정은 **모달**이라 하위 라우트가 없다(IA-06 taxonomy 무게 규칙) |
| 구현 경로 | 래퍼 `apps/admin/src/pages/company/partners/{PartnersPage.tsx,data-source.ts}` · 실제 UI `apps/admin/src/pages/company/logo-list/{LogoListPage,LogoListTable,LogoFormModal,queries,adapter,types,validation}` |
| 대응 SCR | SCR-020 (미작성 — §7) |
| 공통 컴포넌트 | `shared/ui/{SearchField,Button,ConfirmDialog,Modal,SelectionBar,useRowSelection,useModalDirtyGuard,ToggleSwitch,RowActions,RowSelectCell,SelectAllHeaderCell,SeqCell,SeqHeaderCell,tableSelectionState,ReorderGripCell,ReorderGripHeaderCell,ReorderMoveButtons,useReorderableRows,FormField,errorIdOf,ImageUploadField,Alert,useToast}` · `shared/bulk.ts`(`settleAll`) · `shared/async.ts`(`wait`·`isAbort`) · `shared/crud/{dev.ts,validation.ts}` |

### 1.1 고객사(FS-021)와의 공유

`PartnersPage.tsx` 는 `<LogoListPage resource="partners" entityLabel="파트너사" adapter={partnersAdapter} />` 한 줄이다. 화면의 모든 동작·예외는 `logo-list/**` 가 소유하며, 파트너사/고객사는 **`resource`(쿼리 키 루트 겸 `?fail=` scope)·`entityLabel`(문구)·`adapter`(시드)** 세 가지만 다르다. 두 화면의 요소 구성·예외 규칙은 동일하고, 이 문서가 그 골격의 정본이다.
