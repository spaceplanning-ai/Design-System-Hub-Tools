# FS-012. 배너 관리 (목록·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 배너를 위치·키워드로 좁혀 찾고, 목록에서 바로 ON/OFF 를 켜고 끄거나(단건·일괄) 드래그로 순서를 바꾸며 삭제하고, 등록·수정 폼에서 우측 실시간 미리보기로 노출 모습을 확인하며 만든다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 콘텐츠 관리 > 배너 관리 (`/content/banners`) |
| 포함 화면 | 목록 `/content/banners` · 등록 `/content/banners/new` · 수정 `/content/banners/:id/edit` (**상세 조회 화면 없음**) |
| 구현 경로 | `apps/admin/src/pages/content/banners/**` |
| 대응 SCR | SCR-012 (미작성 — §7) |
| 공통 컴포넌트 | `shared/ui/{ToggleSwitch,tableReorder,SelectionBar,useRowSelection,SelectField,ImageUrlField,DateRangeField,ConfirmDialog,Pagination,SearchField,RowActions}` · `@tds/ui` `SegmentedControl` · `shared/bulk.ts` |

> 팝업(FS-011)과 '닮은 쌍'이다. **차이**: 목록 필터가 상태(ON/OFF)가 아니라 **위치(메인/서브)**, 5번째 숫자 열이 '우선순위'가 아니라 **'정렬 순서'**, 그리고 **DnD/키보드 재정렬**이 있다(FAQ 와 같은 규칙). 미리보기는 넓은 띠 모양이며 링크 문구가 '바로가기'다.

| 중복 제출 | 폼 제출·삭제·일괄 액션은 요청 중 버튼 비활성. **낙관적 ON/OFF 토글은 id 단위** 잠금. **재정렬은 진행 중 이동 버튼·드래그 잠금** |

## 5. 서버 연동 지점

| 요소번호 | 이름 | 읽기/쓰기 | 필요 데이터 | 프론트 어댑터 (data-source.ts) | 비고 |
|---|---|---|---|---|---|
| FS-012-EL-001 / EL-002 / EL-004 / EL-006 | 목록 조회 | R | 위치·검색 반영 배너 목록 + 총 건수 | `fetchBanners(query, signal)` | 페이지당 10건 |
| FS-012-EL-006.8 | ON/OFF 토글 | W | 배너 id · enabled | `setBannerEnabled(id, enabled)` | 낙관적 업데이트 |
| FS-012-EL-005.2 / EL-005.3 | 일괄 ON/OFF | W | 선택된 id 배열 · enabled | `setBannerEnabled(id, enabled)` × N (`settleAll`) | 부분 실패 건수 |
| FS-012-EL-006.10 / EL-006.12 | 재정렬 | W | 현재 페이지 새 순서 id 배열 | `reorderBanners(orderedIds, signal)` | 낙관적. 서버가 order 1..n 재매김 |
| FS-012-EL-006.11 / EL-011 | 삭제(단건) | W | 배너 id | `deleteBanner(id, signal)` | — |
| FS-012-EL-005.4 / EL-012 | 일괄 삭제 | W | 선택된 id 배열 | `deleteBanner(id, signal)` × N (`settleAll`) | 전원 성공일 때만 무효화 |
| FS-012-EL-010 | 페이지네이션 | R | 요청 페이지 | `fetchBanners(query, signal)` | 크기 10 |
| FS-012-EL-019 | 정렬 순서 자동 채움 | R | 현재 최대 order + 1 | `fetchNextBannerOrder(signal)` | 등록 모드에서만 |
| FS-012-EL-024 (등록) | 배너 등록 | W | 제목·이미지·링크·위치·기간·노출·정렬 순서 | `createBanner(input, signal)` | 성공 후 목록 이동 |
| FS-012-EL-024 (수정) | 배너 수정 | W | id + 위 입력 | `updateBanner(id, input, signal)` | 성공 후 목록 이동 |
| FS-012-EL-025 | 수정 폼 초기 로드 | R | 배너 전문 | `fetchBanner(id, signal)` | 폼 `reset` |

> **현재 구현 상태 (백엔드 명세 참고)**: `createBanner`·`updateBanner`·`deleteBanner` 는 resolve 만 하고 저장하지 않는다. `setBannerEnabled`·`reorderBanners` 는 브라우저 안 mutable 픽스처(`BANNERS`)를 갱신한다. 이미지는 URL 문자열 저장 — 업로드는 `// TODO(backend): POST /api/uploads`. 위 표는 백엔드 연결 후 의도된 동작이다.
