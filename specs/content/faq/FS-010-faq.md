# FS-010. FAQ 관리 (목록·카테고리 모달·상세·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | FAQ 를 카테고리·노출 여부·키워드로 좁혀 찾고, 목록에서 바로 노출을 켜고 끄거나 순서를 바꾸며, 카테고리를 관리하고, 개별 FAQ 를 등록·수정·삭제(단건·일괄)한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 콘텐츠 관리 > FAQ (`/content/faq`) |
| 포함 화면 | 목록 `/content/faq` · 카테고리 관리 모달(목록 안) · 상세 `/content/faq/:id` · 등록 `/content/faq/new` · 수정 `/content/faq/:id/edit` |
| 구현 경로 | `apps/admin/src/pages/content/faq/**` |
| 대응 SCR | SCR-010 (미작성 — §7) |
| 공통 컴포넌트 | `shared/ui` 배럴에서 — `ToggleSwitch` · `tableReorder`(`moveArrayItem`·`useReorderableRows`·`ReorderMoveButtons`·`ReorderGripCell`) · `SelectionBar` · `SelectField` · `ConfirmDialog` · `Modal` · `Pagination` · `SearchField` · `StatusBadge` · `RowActions` · `TextareaField` · `FormField`. **대부분은 `@tds/ui` 로 승격돼 `shared/ui` 가 재수출만 한다**(사본은 삭제됨 — `shared/ui/index.ts:71-124`); 진짜 로컬은 `shared/ui/ConfirmDialog.tsx` 와 `shared/bulk.ts`(`settleAll`) 다 |
| 조회 상태의 소유자 | `category`·`visibility`·`keyword`·`page` 와 **행 선택**은 `shared/crud` 의 `useListState` 가 쥔다(`FaqPage.tsx:17,117,124`). 필터/페이지 축은 URL 쿼리스트링에 산다(`useListState.ts:87-99` `useSearchParams`) — 그래서 새로고침·뒤로가기가 조건을 보존한다. 선택은 화면 로컬이며(`useListState.ts:186`) 조건이 바뀌면 자동 해제된다. **`shared/ui/useRowSelection` 은 이 화면이 쓰지 않는다** |
