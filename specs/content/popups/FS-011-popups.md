# FS-011. 팝업 관리 (목록·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 팝업을 상태·키워드로 좁혀 찾고, 목록에서 바로 ON/OFF 를 켜고 끄거나(단건·일괄) 삭제하며, 등록·수정 폼에서 우측 실시간 미리보기로 노출 모습을 확인하며 만든다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 콘텐츠 관리 > 팝업 관리 (`/content/popups`) |
| 포함 화면 | 목록 `/content/popups` · 등록 `/content/popups/new` · 수정 `/content/popups/:id/edit` (**상세 조회 화면 없음** — 목록에서 수정으로 바로 간다) |
| 구현 경로 | `apps/admin/src/pages/content/popups/**` |
| 대응 SCR | SCR-011 (미작성 — §7) |
| 공통 컴포넌트 | `shared/ui/{ToggleSwitch,SelectionBar,useRowSelection,SelectField,ImageUrlField,DateRangeField,ConfirmDialog,Pagination,SearchField,RowActions}` · `@tds/ui` `SegmentedControl` · `shared/bulk.ts` |

| 규칙 | 내용 |
|---|---|
| 네트워크 단절 | 목록 조회는 인라인 배너(FS-011-EL-009)로 대체. 수정 로드 실패는 폼 대체 화면(FS-011-EL-025). 쓰기는 각 요소 실패 처리 |
| 세션 만료 | 감지·리다이렉트 처리가 구현에 없다 — §7 |
| 요청 타임아웃 | 프론트 타임아웃 상한 없음. abort 는 조건 변경·화면 이탈·다이얼로그 닫기·토글 재요청 시 — §7 |
| 중복 제출 | 폼 제출·삭제·일괄 액션은 요청 중 버튼 비활성. **낙관적 ON/OFF 토글(FS-011-EL-006.7)은 id 단위로** 그 스위치를 잠근다 |
| 실패 통지의 자리 | ① 조회 실패 = 인라인 배너 ② 저장 실패 = 폼 카드 배너 ③ 삭제 실패 = 다이얼로그 안 배너 ④ 낙관적 토글·일괄 액션 실패 = 재시도가 붙은 토스트 ⑤ 저장·삭제 **성공** = 토스트 |
| 낙관적 업데이트 | ON/OFF 토글(FS-011-EL-006.7)은 요청 전 캐시를 즉시 바꾸고 실패 시 스냅샷으로 롤백한다. 일괄 토글은 낙관 반영 없이 서버 정산 후 목록 무효화 |
| 동시 조회 | 목록/상세 조회는 동시에 1건만 유지(react-query) |
| 권한 없음 | 역할 기반 분기 없음. 서버 권한 응답은 조회=인라인 배너, 쓰기=각 요소 실패로 떨어진다. 은닉 정책은 BE-011 이 확정 |
| 행 선택의 수명 | 선택은 화면 로컬 상태. 상태 필터·키워드·페이지 중 하나라도 바뀌면 전부 해제 |
| 이미지 미리보기 | 이미지 URL 필드(FS-011-EL-016)·폼 미리보기(FS-011-EL-022)는 `<img>` 의 `onError` 로 로드 실패를 감지해 안내 문구로 대체한다. URL 이 바뀌면 실패 상태를 재설정해 다시 시도한다. **이미지 접근성·유효성의 서버 검증은 BE-011 이 확정**한다 |

## 5. 서버 연동 지점

| 요소번호 | 이름 | 읽기/쓰기 | 필요 데이터 | 프론트 어댑터 (data-source.ts) | 비고 |
|---|---|---|---|---|---|
| FS-011-EL-001 / EL-002 / EL-004 / EL-006 | 목록 조회 | R | 상태·검색 반영 팝업 목록 + 총 건수 | `fetchPopups(query, signal)` | 페이지당 10건 |
| FS-011-EL-006.7 | ON/OFF 토글 | W | 팝업 id · enabled | `setPopupEnabled(id, enabled)` | 낙관적 업데이트 |
| FS-011-EL-005.2 / EL-005.3 | 일괄 ON/OFF | W | 선택된 id 배열 · enabled | `setPopupEnabled(id, enabled)` × N (`settleAll`) | 부분 실패 건수 보고 |
| FS-011-EL-006.9 / EL-011 | 삭제(단건) | W | 팝업 id | `deletePopup(id, signal)` | — |
| FS-011-EL-005.4 / EL-012 | 일괄 삭제 | W | 선택된 id 배열 | `deletePopup(id, signal)` × N (`settleAll`) | 전원 성공일 때만 목록 무효화 |
| FS-011-EL-010 | 페이지네이션 | R | 요청 페이지 | `fetchPopups(query, signal)` | 페이지 크기 10 |
| FS-011-EL-019 | 우선순위 자동 채움 | R | 현재 최대 우선순위 + 1 | `fetchNextPopupPriority(signal)` | 등록 모드에서만 |
| FS-011-EL-024 (등록) | 팝업 등록 | W | 제목·이미지·링크·위치·기간·노출·우선순위 | `createPopup(input, signal)` | 성공 후 목록 이동 |
| FS-011-EL-024 (수정) | 팝업 수정 | W | id + 위 입력 | `updatePopup(id, input, signal)` | 성공 후 목록 이동 |
| FS-011-EL-025 | 수정 폼 초기 로드 | R | 팝업 전문 | `fetchPopup(id, signal)` | 폼 `reset` |

> **현재 구현 상태 (백엔드 명세 참고)**: `createPopup`·`updatePopup`·`deletePopup` 는 resolve 만 하고 저장하지 않는다(백엔드 미구현). `setPopupEnabled` 는 브라우저 안 mutable 픽스처(`POPUPS`)를 갱신한다. 이미지는 URL 문자열로만 저장 — 파일 업로드는 `// TODO(backend): POST /api/uploads`. 위 표는 백엔드 연결 후 의도된 동작이다.
