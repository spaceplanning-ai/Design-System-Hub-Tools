# FS-013. 약관 관리 (버전 이력·상세·새 버전)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 약관 종류(이용약관·전자금융거래·마케팅 수신 동의 등)를 고르고, 그 종류의 버전 이력을 보며, 개별 버전의 전문을 조회하고, 새 버전을 등록·수정·삭제(단건·일괄)한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 콘텐츠 관리 > 약관 관리 (`/content/terms`) |
| 포함 화면 | 목록 `/content/terms` · 상세(전문) `/content/terms/:id` · 등록 `/content/terms/new?type=` · 수정 `/content/terms/:id/edit` |
| 구현 경로 | `apps/admin/src/pages/content/terms/**` (`TermsPage`·`TermsDetailPage`·`TermsFormPage`·`components/VersionForm`) |
| 대응 SCR | SCR-013 (미작성 — §7) |
| 공통 컴포넌트 | `shared/ui/{VersionHistoryTable,SelectionBar,useRowSelection,SelectField,ConfirmDialog,SearchField,StatusBadge,TextareaField,FormField}` · `shared/bulk.ts` |

> 개인정보 처리방침(FS-014)과 '버전 문서 쌍'이다. **차이**: 약관은 **종류(type) 선택**이 있고(좌측 사이드바) 개인정보는 단일 문서다. 버전 이력 표(`VersionHistoryTable`)는 공유하되 서로 import 하지 않는다.
