# FS-002. 대시보드

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 업무 영역(상품·문의·영업)별 처리 대기 건수와 최근 활동, 방문자·기간별 통계를 한 화면에서 조회 |
| 역할(주 사용자) | 인증된 관리자. 화면 요소의 노출 여부는 역할이 아니라 기능 권한 키(shared/permissions)의 ON/OFF로 결정된다 |
| 진입 경로 | (1) 로그인 성공 직후 기본 이동처, (2) 사이드바 '대시보드' 메뉴, (3) 루트 경로(`/`) 접근 시 리다이렉트, (4) 정의되지 않은 경로 접근 시 리다이렉트 |
| 관련 화면 | FS-001 로그인(인증 출발점). 사이드바·할일 항목·리스트 행에서 각 도메인 화면으로 이동 |
| 대응 SCR | SCR-002 |
| 구현 경로 | apps/admin/src/pages/dashboard/ (DashboardPage.tsx · components/{DashboardTabPanel,StatsSection,RowIcon}.tsx · api.ts · stats-api.ts · queries.ts · types.ts · stats-types.ts · useDashboardTabs.ts) · 셸: apps/admin/src/shared/layout/{AppShell,AppHeader,LogoPlaceholder}.tsx + nav-config.ts · 권한: apps/admin/src/shared/permissions/ |

대시보드는 레이아웃 셸(AppShell) 안에서 렌더된다 — 좌측 고정 사이드바 + 우측 콘텐츠(헤더 + 본문).
따라서 사이드바(SEC-01)와 헤더(SEC-02)도 이 화면의 요소로 넘버링한다.

> 조립 경로 주석(v1.2 실측): 탭바는 더 이상 이 화면의 사본이 아니라 **`@tds/ui` 의 `Tabs`** 다
> (`DashboardPage.tsx:14`). 탭 패널 내용(오늘의 할일 + 리스트 카드 2장)은 `DashboardTabPanel.tsx`,
> 통계는 `StatsSection.tsx` 가 소유하며 페이지에 남은 분기는 **무엇을 렌더할지 고르는 것뿐**이다
> (`DashboardPage.tsx:8-9`). 조회 상태는 손으로 만든 `useAsyncData` 가 아니라 **TanStack Query**
> (`queries.ts`)가 들고, 탭 해소는 `useDashboardTabs.ts` 가 맡는다. 아래 요소 번호는 화면에 보이는
> 것을 가리키며 파일 경계와 1:1 이 아니다.

> 구현 대조 주석: SCR-002 §3의 'KPI 스탯 카드 4종 · 기간 Select(오늘/최근 7일/최근 30일) · 최근 활동 테이블 ·
> 차트 플레이스홀더'는 현재 구현에 존재하지 않는다. 구현은 '업무 탭 3종 · 오늘의 할일 · 리스트 카드 2종 ·
> 방문자 차트(일/주/월) · 기간별 분석 표'로 되어 있다. 본 명세는 구현을 기술한다(명세는 구현의 거울).
> SCR-002 개정은 UI 기획(UI Planner) 소관이며, 이 불일치는 아키텍처 판정 대상으로 보고한다.

## 2. 영역 구성

| 영역번호 | 이름 | 설명 |
|---|---|---|
| FS-002-SEC-01 | 셸 사이드바 | 로고 자리표시 + 4개 섹션에 걸친 내비게이션 목록(최상위 항목 14개, 서브메뉴 67개). 권한이 꺼진 메뉴는 렌더되지 않는다 |
| FS-002-SEC-02 | 셸 헤더 | 좌측 'LOGO · 관리자' + 현재 페이지명, 우측 오늘 날짜 + 로그인 계정 |
| FS-002-SEC-03 | 업무 탭바 | 상품 / 문의 / 영업 3탭과 탭 패널 컨테이너 |
| FS-002-SEC-04 | 오늘의 할일 카드 | 제목 + 합계 뱃지 + 항목 링크 목록(탭별 항목 상이) |
| FS-002-SEC-05 | 리스트 카드 그리드 | 나란히 놓이는 리스트 카드 2장(탭별 내용 상이) |
| FS-002-SEC-06 | 통계 섹션 | 방문자 카드(기간 토글 + 범례 + SVG 차트 + 축 라벨) + 기간별 분석 카드(표 + 합계 행) |
| FS-002-SEC-07 | 화면 수준 상태 (비표시 기본) | 탭 데이터 조회 실패 배너, 대시보드 위젯 권한 소거, 활성 탭 폴백, 전 위젯 OFF 안내 |

셸의 콘텐츠 컬럼(`main`)은 SEC-03~SEC-07을 담는 레이아웃 컨테이너이며 자체 콘텐츠·상호작용·데이터가 없다 —
요소 번호를 부여하지 않는다(구현: `AppShell.tsx` 의 `<main><Outlet /></main>`).

| 네트워크 단절 | 탭 데이터 조회가 실패하면 FS-002-EL-041 배너('대시보드 데이터를 불러오지 못했습니다. 잠시 후 다시 시도해 주세요.'), 통계 조회가 실패하면 카드별 문구(FS-002-EL-035 · EL-040)를 렌더한다. 두 조회는 독립이므로 한쪽 실패가 다른 쪽을 가리지 않는다 |
| 세션 만료 | 현재 구현의 대시보드에는 세션 검사·로그아웃 요소가 없다. 세션 만료 처리는 로그인 화면 진입 시의 안내(FS-001-EL-023)로만 정의되어 있다 |
| 요청 타임아웃 | 프론트 타임아웃 상한이 설정되어 있지 않다. 진행 중 요청은 탭·기간 변경 또는 화면 이탈 시 `AbortController` 로 중단된다 |
| 중복 제출 | 이 화면에는 쓰기 요청이 없다(읽기 전용). 읽기 요청은 탭·기간별로 동시 1건만 유지되며, 새 요청 발행 전에 진행 중이던 요청을 중단한다 |
| 자동 갱신 | 폴링·자동 재조회가 없다. 데이터 갱신 경로는 탭 전환(탭 데이터), 기간 토글(통계), 화면 재진입 3가지다 |
| 권한 없음 (전 요소 공통) | 권한이 꺼진 요소는 비활성이 아니라 미렌더다 — 권한 없음 전용 화면을 띄우지 않는다. 권한 기본값은 전부 ON이며, 변경은 리로드 없이 즉시 반영되고 다른 브라우저 탭의 변경도 storage 이벤트로 따라온다. 메뉴·위젯이 숨어도 라우트 접근 자체는 차단되지 않는다 |
| 요청 취소 | 중단된(abort) 요청은 실패로 표시하지 않는다 — 에러 문구·배너를 띄우지 않고 늦게 도착한 응답도 상태에 반영하지 않는다 |
