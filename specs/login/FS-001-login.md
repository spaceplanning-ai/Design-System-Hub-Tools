# FS-001. 로그인

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 어드민 허브 사용자의 이메일·비밀번호 인증과 세션 발급 |
| 역할(주 사용자) | 미인증 사용자 (인증 후 역할: system_admin · operator · viewer) |
| 진입 경로 | (1) `/login` 직접 접근, (2) 보호된 URL 접근 시 미인증 리다이렉트(`?returnUrl=<원래 경로>`), (3) 세션 만료 리다이렉트(`?reason=session_expired`) |
| 관련 화면 | FS-002 대시보드 (인증 성공 시 기본 이동처 `/dashboard`) |
| 대응 SCR | SCR-001 |
| 구현 경로 | apps/admin/src/pages/login/ (LoginPage.tsx · components/LoginForm.tsx · api.ts · session.ts · validation.ts · validation.test.ts · login.css). **TextField·PasswordField·Checkbox·Alert·Button 은 `@tds/ui` 가 소유하고 LoginForm 이 조립한다** — 이 화면의 사본이 아니다(`LoginPage.tsx:5-7`). `icons.tsx` 는 존재하지 않는다 — 눈 아이콘은 `@tds/ui` `PasswordField` 안에(`PasswordField.tsx:19-33`), 경고/정보 아이콘은 `Alert` 안에 들어 있다. 상태·API 호출·유효성·타임아웃·리다이렉트는 전부 LoginPage 가 소유한다 |

화면은 좌우 5:5 그리드다(`gridTemplateColumns: '1fr 1fr'`). 좌측은 장식용 비주얼 패널(`aria-hidden="true"`),
우측은 폼 패널이다. 뷰포트 폭 768px 미만에서는 그리드가 1열로 바뀌고 좌측 패널이 `display: none` 된다.

### 4.1 공통 예외 규칙

| 규칙 | 내용 |
|---|---|
| 네트워크 단절 | 인증 요청이 네트워크 예외로 실패하면 FS-001-EL-022의 danger Alert('일시적인 오류로 로그인하지 못했습니다. 다시 시도해 주세요.')를 표시하고 입력값을 전부 유지한다. 복구 경로는 제출 버튼 재클릭 1가지다 |
| 세션 만료 | 다른 화면에서 세션이 만료되어 `/login?reason=session_expired` 로 진입하면 FS-001-EL-023의 info Alert('세션이 만료되었습니다. 다시 로그인해 주세요.')를 표시한다 |
| 요청 타임아웃 | 프론트 상한 10초(`LOGIN_TIMEOUT_MS`). 초과 시 요청을 중단하고 네트워크·서버 오류(FS-001-EL-022)와 동일하게 처리한다 |
| 중복 제출 | 동기 가드(`submitLockRef`)로 두 번째 제출 이벤트를 즉시 반환시키고, 제출 버튼 `disabled` 로 클릭·Enter를 함께 막는다. 응답 수신 전 인증 요청은 1건만 발행된다 |
| 권한 없음 (전 요소 공통) | 이 화면은 인증 이전 화면이므로 역할 기반 권한 판정이 성립하지 않는다 — 어떤 요소도 역할에 따라 숨거나 비활성화되지 않는다. 이미 인증된 사용자가 진입하면 폼을 노출하지 않고 `/dashboard` 로 즉시 이동한다(FS-001-EL-025) |
| 브라우저 저장소 접근 불가 | 세션·이메일 저장 읽기/쓰기가 예외를 던지면 해당 기능만 동작하지 않는다 — 세션 읽기 실패는 세션 없음으로 처리하고, 세션 쓰기 실패는 인증 결과를 무효화하지 않는다 |
