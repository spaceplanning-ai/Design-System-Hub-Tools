# FS-035. 이메일 발송 (목록·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 이메일 발송 캠페인을 발송상태·키워드로 찾고, 제목·발신자·수신자 세그먼트·본문·수신거부 링크·광고성·예약을 지정해 **초안 또는 예약으로 저장**하며, 삭제(단건·일괄)한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 마케팅 관리 > 이메일 발송 (`/marketing/email`) |
| 포함 화면 | 목록 `/marketing/email` · 등록 `/marketing/email/new` · 수정 `/marketing/email/:id/edit`. **상세 조회 라우트는 없다**(행 클릭 = 수정 진입) |
| 구현 경로 | `apps/admin/src/pages/marketing/email/**` · 공용 `apps/admin/src/pages/marketing/_shared/**` |
| 대응 SCR | SCR-035 (미작성 — §7) |
| 공통 컴포넌트 | `shared/crud/{CrudListShell,CrudTable,useCrudList,useCrudForm,createCrudAdapter,FormServerError,FormConflictDialog,requiredText}` · `shared/ui/{SearchField,SelectField,StatusBadge,Button,Card,CardTitle,FormField,TextareaField,ToggleSwitch,Alert,ConfirmDialog,SelectionBar,RowActions,Empty,ToastProvider,useUnsavedChangesDialog}` · marketing `_shared/{messaging,store,SegmentPicker,VariableInsertBar,EmailPreview}` |
| SMS 발송과의 관계 | **형제 구조다** — 같은 CRUD 프레임워크 · 같은 `_shared` 도메인 · 같은 2단 레이아웃(좌 입력 / 우 미리보기)을 쓴다. 차이는 이메일 고유 규칙뿐이다: **제목**이 있고(SMS 는 없다), **(광고) 표기는 본문이 아니라 제목**에 걸리며, **수신거부 링크 토글**이 있고, **야간 발송 제한이 없으며**, **바이트/유형(SMS·LMS·MMS) 개념이 없다**(본문 상한 5,000자) |
| **저장은 발송이 아니다** | 이 화면의 어떤 동작도 메일을 전송하지 않는다. 저장은 캠페인(초안/예약) 레코드를 만들 뿐이다(§5 '현재 구현 상태'). 화면 문구도 그렇게 고지한다(FS-035-EL-014·EL-030) |

### 4.1 공통 예외 규칙

| 규칙 | 내용 |
|---|---|
| **저장 ≠ 발송** | 이 화면의 어떤 경로도 메일을 보내지 않는다. 목록의 '발송중'·'발송완료'·'취소' 상태와 오픈율/클릭율은 **픽스처 시드 값**이며, 화면이 만들어 낸 것이 아니다. 실제 전송·예약 실행·오픈/클릭 집계는 백엔드 몫이다(§5) |

> **현재 구현 상태 (A63 참고)**: 실제 네트워크 호출은 이 화면 어디에도 없다. `emailAdapter` 는 `createCrudAdapter` 가 만든 **브라우저 안 mutable 배열**(시드 3건)이며, 400ms 지연 후 응답을 흉내 낸다. 발신자·세그먼트·템플릿은 어댑터조차 거치지 않는 **동기 픽스처 함수**다(`_shared/store.ts`). **연동 지점은 `data-source.ts:69-70` 의 주석 두 줄이 전부다**:
> ```
> // TODO(backend): GET/POST /api/marketing/email · GET/PUT/DELETE /api/marketing/email/:id
> //   발송 실행: POST /api/marketing/email/:id/send (예약은 스케줄러가 실행) — 프론트는 트리거만.
> ```
> **`/send` 심은 주석에만 있고 프론트 호출부가 0건이다** — 어댑터 함수도, 발송 버튼도, 발송 확인 다이얼로그도, 테스트 발송도 없다. 즉 **이 화면에서 메일을 보낼 방법이 현재 존재하지 않는다**(FS-035-EL-035). 목록의 '발송완료' 행과 오픈율/클릭율 수치는 픽스처 시드가 만든 것이다. 위 표는 백엔드 연결 후 의도된 동작이며, `/send` 는 **UI 가 함께 구현돼야 성립하는 계약**이다.
>
> **`/send` 심은 SMS 와 대칭이다** — `marketing/sms/data-source.ts:71` 과 `marketing/email/data-source.ts:70` 이 같은 문장 구조를 갖는다. 두 화면 모두 심만 있고 호출부가 없다.
