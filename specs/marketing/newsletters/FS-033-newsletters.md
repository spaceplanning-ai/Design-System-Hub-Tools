# FS-033. 뉴스레터 (발송회차 목록·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 뉴스레터 **발송회차(issue)** 를 상태·제목으로 좁혀 찾고, 회차를 등록·수정·삭제(단건·일괄)한다. 회차번호는 저장 시 자동 채번되고, 구독자 세그먼트·발신자·본문·예약 시각을 지정하며 우측에서 수신 메일을 미리 본다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 마케팅 관리 > 뉴스레터 (`/marketing/newsletters`) |
| 포함 화면 | 목록 `/marketing/newsletters` · 등록 `/marketing/newsletters/new` · 수정 `/marketing/newsletters/:id/edit` (**상세 화면 없음** — 행 클릭이 곧 수정) |
| 구현 경로 | `apps/admin/src/pages/marketing/newsletters/**` (+ `apps/admin/src/pages/marketing/_shared/**`) |
| 대응 SCR | SCR-033 (미작성 — §7) |
| 공통 컴포넌트 | `shared/crud/{CrudListShell,CrudTable,useCrudList,useCrudForm,useListState,useDebouncedSearch,parseFilter,createCrudAdapter,FormServerError,FormConflictDialog}` · `shared/ui/{SearchField,SelectField,StatusBadge,SelectionBar,RowActions,RowSelectCell,SeqCell,Empty,ConfirmDialog,Alert,Card,TextareaField,FormField,useUnsavedChangesDialog,useRowSelection}` · `marketing/_shared/{SegmentPicker,VariableInsertBar,EmailPreview,messaging,store}` |
| 저장은 발송이 아니다 | 이 화면의 어떤 동작도 메일을 보내지 않는다. 회차 레코드만 저장한다(`NewsletterFormPage.tsx:389` '저장은 발송이 아닙니다.'). 실제 전송 트리거는 **미구현** — §5 · §7 |
