# FS-025. 성공 사례 (목록·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 성공 사례(고객 과제 → 해결 → 성과)를 업종으로 좁혀 보고, 목록에서 바로 노출을 켜고 끄며, 개별 사례를 등록·수정·삭제(단건·일괄)한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 포트폴리오 관리 > 성공 사례 (`/portfolio/case-studies`) |
| 포함 화면 | 목록 `/portfolio/case-studies` · 등록 `/portfolio/case-studies/new` · 수정 `/portfolio/case-studies/:id/edit`. **상세(`/:id`) 라우트가 없다** — 행 클릭·행 액션의 수정 버튼이 곧바로 수정 폼으로 간다 |
| 구현 경로 | `apps/admin/src/pages/portfolio/case-studies/**` · 섹션 공용 `apps/admin/src/pages/portfolio/_shared/**` |
| 대응 SCR | SCR-025 (미작성 — §7) |
| 공통 컴포넌트 | FS-023 §1 과 동일 |

> **FS-023 의 골격을 그대로 상속한다.** 두 화면은 `useCrudList` + `CrudListShell` + `useCrudForm` + `FormPageShell` 이라는 같은 프레임워크 위에 서 있고, 폼 하단부(`_shared/PortfolioMediaFields.tsx`)와 목록의 노출 토글 열(`_shared/publishColumn.tsx`)을 **공유**한다. **이 문서는 FS-023 과 다른 것만 온전히 기술하고, 같은 것은 FS-023 을 참조한다.** 골격 동작(로딩·재조회·빈 상태·실패 배너·삭제 다이얼로그·미저장 가드·충돌 다이얼로그·언마운트 abort)의 정본은 FS-023 이다.

### 1.1 FS-023 과의 차이 (전수)

| 축 | FS-023 포트폴리오 | FS-025 성공 사례 |
|---|---|---|
| **분류 축** | **서버 관리형 카테고리** — `GET /api/portfolio/categories` 로 선택지를 받는다. 카테고리 화면(FS-024)이 CRUD 를 소유하고 항목이 `categoryId` 로 참조한다 | **고정 업종 enum** — `CASE_INDUSTRY_OPTIONS` 6종(제조·유통·금융·공공·의료·IT·서비스)이 **소스 상수**다(`types.ts:51-58`). **서버 조회가 없다** |
| **필터 select 옵션 출처** | 서버 쿼리(`['portfolio','category-options']`) → 실패 시 표면 없음(FS-023 §7 #10) | 상수 → **조회 실패 자체가 없다** |
| **필터 값 파싱** | 없음 — `event.target.value` 를 그대로 `useState` 에 넣는다 | `parseFilter(value, CASE_FILTER_VALUES, CASE_FILTER_ALL)` 로 좁힌다(타입 안전) |
| **배지 톤** | `categoryTone(categoryId)` — id 해시로 4색 **안정 배정**(의미 없음) | `industryTone(industry)` — 옵션 상수에 **톤이 명시**돼 있다(`types.ts:52-57`) |
| **어댑터** | `createStoreAdapter` — `_shared/store.ts` 의 공유 배열에 위임. **존재 검사 없음**(§7 참조) | `createCrudAdapter` — 자체 시드 배열을 들고 **없는 id 에 409 를 던진다**(`crud.ts:71`·`:82`) |
| **저장소 공유** | 카테고리 화면(FS-024)과 `_shared/store.ts` 를 공유(순환 import 회피) | 공유 없음 — 어댑터가 시드를 자체 보유(`data-source.ts:9-64`) |
| **`?fail=` scope** | `portfolio` | `case-studies` |
| **본문 필드** | 소개 1개(500자) | **과제·해결·성과 3개**(각 500자) |
| **목록 열** | 분류 · 제목 · 고객사 · 일자 · 노출 | 업종 · 제목 · 고객사 · **성과** · 노출 (**일자 열이 없다** — 정렬 기준인데 보이지 않는다) |
| **truncate 대상** | 고객사(폭 상한 + ellipsis) | **성과**(폭 상한 + ellipsis). 고객사는 `nowrap` 만 |
| **폼 첫 행** | 분류 select(빈 값 허용) + 일자 | 업종 select(**빈 옵션 없음** — 기본 'manufacturing') + 일자 |
| **entityLabel** | '포트폴리오' | '성공 사례' |
| **resource / 쿼리 키** | `portfolio` | `case-studies` |
