# FS-024. 포트폴리오 카테고리 관리 (목록·등록/수정 모달·삭제)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 포트폴리오 항목을 분류하는 카테고리(taxonomy)를 한 화면에서 조회·추가·수정·삭제한다. **사용 중인 카테고리는 삭제를 잠가** 고아 포트폴리오를 막는다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 포트폴리오 관리 > 카테고리 (`/portfolio/categories`) |
| 포함 화면 | 목록 `/portfolio/categories` · 등록/수정 모달(목록 안) · 삭제 확인 다이얼로그(목록 안). **하위 라우트 없음** — 짧은 taxonomy 엔티티라 모달로 편집한다(IA-06 무게 규칙) |
| 구현 경로 | `apps/admin/src/pages/portfolio/categories/**` (`PortfolioCategoriesPage.tsx` · `components/PortfolioCategoryFormModal.tsx` · `data-source.ts` · `types.ts` · `validation.ts` · `categories.test.ts`) |
| 도메인 정본 | `apps/admin/src/pages/portfolio/_shared/store.ts` — 카테고리 픽스처·사용량 계산(`countItemsUsingCategory`)·삭제 차단이 여기 있다. 포트폴리오 항목 화면과 같은 저장소를 공유한다 |
| 대응 SCR | SCR-024 (미작성 — §7) |
| 공통 컴포넌트 | `shared/ui/{Modal,ConfirmDialog,useModalDirtyGuard,ToastProvider,Alert,Button,Card,StatusBadge,FormField,PencilIcon,TrashIcon,PlusCircleIcon}` · `shared/crud/{useCrudListQuery,useCrudCreate,useCrudUpdate,useCrudDelete,createStoreAdapter,requiredText}` · `shared/async.isAbort` · `shared/format.formatNumber` |
