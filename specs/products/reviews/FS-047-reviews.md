# FS-047. 리뷰 (목록·상세 노출/답변)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 고객이 쓴 상품 리뷰를 별점·키워드로 좁혀 훑고(낮은 별점·신고 트리아지), 목록에서 바로 노출/숨김을 토글하며, 개별 리뷰에서 관리자 답변을 달거나 리뷰를 삭제한다. **답변과 노출 상태는 고객 상품 페이지에 그대로 공표된다** — 상세 우측 미리보기가 그 결과를 비춘다 |
| 역할(주 사용자) | 관리자 · **쓰기 권한 게이팅이 없다** — read 권한만 있는 역할도 노출 토글·답변·삭제 컨트롤을 그대로 보고 누른다(FS-047-EL-038) |
| 진입 경로 | 좌측 GNB > 상품 관리 > 리뷰 (`/products/reviews` — `nav-config.ts:151`) |
| 포함 화면 | 목록 `/products/reviews` · 상세 `/products/reviews/:id` (`App.tsx:233-234`) |
| **범위 밖** | **리뷰 작성** — 리뷰는 고객이 쓰고 관리자는 노출·답변·삭제만 한다. 화면에 등록 버튼·진입점이 없다(`reviewAdapter.create` 는 `createCrudAdapter` 가 인터페이스로 제공할 뿐 **호출부가 0건**). BE-047 §7.6 이 이를 계약 범위 밖으로 확정한다. **신고 접수·신고 처리(반려/승인)** — `reported`·`reportReason` 은 **읽기 전용 표시**이며 이 화면에 신고를 해소·기각하는 수단이 **없다**(§7 #6). **답글에 대한 고객 재답변(스레드)** — 답변은 `reply` 문자열 1개이며 스레드 구조가 없다 |
| 구현 경로 | `apps/admin/src/pages/products/reviews/**` |
| 대응 SCR | SCR-047 (미작성 — §7 #1) |
| 공통 컴포넌트 | `shared/ui/{SearchField,SelectField,StatusBadge,ToggleSwitch,ImageThumb,Alert,Button,Card,CardTitle,ChevronLeftIcon,ConfirmDialog,TextareaField,useToast,useUnsavedChangesDialog,pageTitleStyle,dlStyle,dtStyle,ddStyle,fieldStyle,fieldLabelStyle}` · `shared/crud/{CrudListShell,CrudTable,useCrudList,useCrudRowUpdate,useCrudUpdate,useCrudDelete,useListState,createCrudAdapter,dev(LATENCY_MS·failIfRequested)}` · `shared/async(isAbort)` · `shared/useRowNavigation`(CrudTable 경유) |

> **이 화면은 상품 도메인 저장소와 연결돼 있지 않다**: 형제 화면(FS-044 교환/반품)이 `pages/products/_shared/store.ts` 를 읽어 SKU·재고를 조회하는 것과 달리, **리뷰는 `_shared` 를 import 하지 않는다**(grep 0건). `Review.productId`(`types.ts:9`)는 저장·복사만 되고 **읽는 코드가 0건**이다 — 상품과의 조인이 화면·어댑터 어디에도 없다(§7 #12 · BE-047 §7.5). 목록·상세에 보이는 상품명은 리뷰가 든 **비정규화 사본**(`productName`)이다.
