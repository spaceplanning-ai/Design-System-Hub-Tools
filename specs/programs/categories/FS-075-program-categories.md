# FS-075. 프로그램 카테고리 (2Depth 목록·모달 등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 프로그램이 참조하는 **2단계 분류 체계**(대분류 → 중분류)를 관리한다 — 카테고리를 추가·이름/상위 수정·삭제하고, 각 카테고리를 **몇 개 프로그램이 쓰는지** 보며, **사용 중이거나 하위가 달린 카테고리는 삭제를 막아** 고아 프로그램·고아 중분류가 생기지 않게 한다 |

> **카테고리와 프로그램은 같은 저장소를 공유한다**(`programs/_shared/store.ts`): 프로그램이 카테고리를 참조하고, 카테고리 삭제는 그 카테고리를 쓰는 프로그램이 있으면 막힌다(`:513-521`). 그래서 이 화면의 목록 항목은 `ProgramCategory` 가 아니라 **`ProgramCategoryUsage`**(= 카테고리 + `programCount` + `hasChildren` — `:23-26`)다.

### 1.1 상품 카테고리(FS-042)와 무엇이 같고 무엇이 다른가

골격은 **의도적으로 같다**(`ProgramCategoriesPage.tsx:3-6`: "규칙이 같은 화면이 서로 다른 골격을 갖기 시작하면 한쪽만 고쳐지는 날이 온다"). 같은 것: 좌측 사용 여부 필터(전체·사용 중·미사용) · 건수 배지 + `aria-pressed` · `<ul>` 행 목록(표가 아니다) · 하나의 모달이 등록과 수정을 겸함 · `ConfirmDialog` 삭제 · `useModalDirtyGuard` 4경로 파기 확인 · 카드 하단 상시 안내문 · 검색·페이지네이션 없음.

**다른 것 — 이 화면에만 있는 것:**

| 차이 | 내용 | 근거 |
|---|---|---|
| **2Depth 트리** | 대분류 아래 중분류를 들여쓰고 펼침/접기 토글을 준다. 상품 카테고리는 **평면**(`parentId` 자체가 없다) | `ProgramCategoriesPage.tsx:350-362,467-512` vs `products/_shared/store.ts:17-20` |
| **삭제 차단 사유가 둘** | '사용 중'(프로그램이 쓴다) **또는** '하위 카테고리 있음' | `:219` · `_shared/store.ts:513-521` |
| **사용 건수 배지가 링크다** | `N개 프로그램` 배지를 누르면 `/programs?category=<id>` 로 간다 — **FS-042 §7 #6 이 요청한 바로 그것**이 여기에는 구현돼 있다 | `:249-255` |
| **상위 카테고리 셀렉트** | 모달에 이름 외에 '상위 카테고리' 필드가 있고, 하위를 가진 대분류는 그 셀렉트가 잠긴다 | `ProgramCategoryFormModal.tsx:59,171-192` |
| **'하위 카테고리 추가' 액션** | 대분류 행에만 `+` 버튼이 붙어 부모가 채워진 채로 모달이 열린다 | `:265-276` · `:488` |
| **조사 오류가 없다** | 상품 카테고리는 `'N개 상품라 삭제할 수 없습니다'`(FS-042 §7 #11). 여기는 `'<이름> — <사유>이라 삭제할 수 없습니다'` 로 **'이라'** 를 쓴다 — 사유가 '12개 프로그램'(받침 있음)이든 '하위 카테고리 있음'(받침 있음)이든 맞는다 | `:290-294` |
