# FS-037. 예약 관리 (목록·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 방문 예약을 상태·키워드로 좁혀 찾고, 같은 자원에 시간이 겹치는 예약(더블부킹)을 한눈에 알아보며, 개별 예약을 등록·수정·삭제(단건·일괄)한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 예약/신청 관리 > 예약 (`/reservations`) |
| 포함 화면 | 목록 `/reservations` · 등록 `/reservations/new` · 수정 `/reservations/:id/edit` |
| 구현 경로 | `apps/admin/src/pages/reservations/{ReservationListPage.tsx,ReservationFormPage.tsx,reservation-validation.ts}` · `apps/admin/src/pages/reservations/_shared/**` |
| 대응 SCR | SCR-037 (미작성 — §7) |
| 공통 컴포넌트 | `shared/crud/{CrudListShell,CrudTable,useCrudList,useCrudForm,useCrudListQuery,FormPageShell,FormFeedback,createCrudAdapter,requiredText}` · `shared/ui/{SearchField,SelectField,StatusBadge,Button,Alert,FormField,TextareaField,ConfirmDialog,SelectionBar,RowActions,useRowSelection,useUnsavedChangesDialog}` · `shared/useRowNavigation` · `shared/format.formatNumber` |
| 섹션 공용 모듈 | `_shared/booking.ts`(상태·라벨·톤·전이) · `_shared/calendar.ts`(날짜·시각·겹침) · `_shared/reservation.ts`(도메인 타입·더블부킹·필터·검색·정렬) · `_shared/reservation-store.ts`(어댑터) · `_shared/resources.ts`(자원·담당자 카탈로그). 예약 일정(FS-040)이 같은 모듈을 읽는다 |

### 1.1 이 화면이 하지 않는 것

- **페이지네이션이 없다.** 목록은 어댑터가 돌려준 전량을 클라이언트에서 필터·검색해 그대로 렌더한다(§7 #2).
- **검색·필터가 서버 쿼리가 아니다.** `searchReservations`/`filterReservations` 가 이미 받아 둔 배열을 좁힌다 — 조건이 바뀌어도 재조회하지 않는다.
- **상태 전이 규칙을 폼이 강제하지 않는다.** `_shared/booking.ts` 에 `canTransition`/`statusChoices` 가 있으나 폼은 5개 상태 전부를 그린다(§4 FS-037-EL-026 · §7 #1).
