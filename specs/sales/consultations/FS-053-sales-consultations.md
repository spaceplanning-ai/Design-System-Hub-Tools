# FS-053. 상담 이력 (영업 — 목록·읽기 전용 상세)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 거래처와의 상담 기록(유형·주제·내용·결과·후속조치)을 유형·후속조치 대기·키워드로 좁혀 훑고, 개별 상담의 전문과 후속조치를 확인한다. **감사 성격이라 읽기 위주다** |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 영업 관리 > 상담 이력 (`/sales/consultations` — `nav-config.ts:161`) |
| 포함 화면 | 목록 `/sales/consultations` · 상세 `/sales/consultations/:id` (`App.tsx:255-256`) |
| **범위 밖** | **등록·수정·삭제 전부.** 이 화면은 **완전한 읽기 전용**이다 — 화면에 폼·저장 버튼·삭제 버튼·행 선택이 **하나도 없다**. 헤더 주석이 그것을 선언한다: '상담 이력은 감사 성격이라 읽기 위주다(생성/수정/삭제 없음)'(`ConsultationListPage.tsx:3`) · '읽기 전용 상세 — 상담 정보 + 상담 내용 + 후속조치. 감사 이력이라 수정/삭제 없이 조회만 한다'(`ConsultationDetailPage.tsx:3`) · '생성/수정/삭제 UI 없이 이력을 조회한다'(`types.ts:3-4`). 어댑터는 `createCrudAdapter` 라 `create`/`update`/`remove` 가 **동작하는 채로 존재**하지만 **호출부가 0건**이며, `data-source.ts:4` 가 그 이유를 밝힌다 — 'build/patch 는 프레임워크 계약상 둔다'. BE-053 §1·§7.4 가 이를 계약 범위 밖으로 확정한다 |
| **후속조치도 읽기 전용이다** | 후속조치 완료 여부(`followUpDone`)를 **이 화면에서 체크할 수 없다**. 목록의 '후속조치 대기만' 체크박스는 **필터**이지 완료 처리가 아니다(§7 #7) |
| **관련 링크도 텍스트다** | `related` 필드가 '견적 Q-20260710-001'·'계약 CT-2'·'문의 INQ-20260711-002' 같은 **자유 텍스트 요약**이며 실제 링크가 아니다(`types.ts:28-29` — '관련(기회/문의/계약) 링크 요약')(§7 #8) |
| 구현 경로 | `apps/admin/src/pages/sales/consultations/**` (`ConsultationListPage.tsx` · `ConsultationDetailPage.tsx` · `types.ts` · `data-source.ts` · `consultations.test.ts`) |
| 대응 SCR | SCR-053 (미작성 — §7 #1) |
| 공통 컴포넌트 | `shared/ui/{Alert,Button,buttonStyle,Card,CardTitle,checkboxStyle,ChevronLeftIcon,dl/dt/dd,fieldLabelStyle,hintStyle,pageTitleStyle,SearchField,SelectField,SeqCell,SeqHeaderCell,StatusBadge,tableStyle,visuallyHiddenStyle}` · `shared/crud/{createCrudAdapter,useCrudListQuery,useListState,parseFilter,dev}` · `shared/useRowNavigation` · `shared/format(formatDateTime·formatNumber)` |
