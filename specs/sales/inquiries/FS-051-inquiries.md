# FS-051. 문의 (목록·상세 처리·견적 발행)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 영업 채널(웹·전화·이메일·방문)로 접수된 문의를 유형·채널·상태·키워드로 좁혀 훑고(트리아지), 개별 문의에서 담당을 배정하고 처리 상태를 바꾸며 고객답변·내부메모를 작성해 처리 이력에 남긴다. **상태를 '견적 발행'으로 바꾸면 이 문의를 승계한 견적이 자동 생성된다** |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 영업 관리 > 문의 (`/sales/inquiries` — `nav-config.ts:159`) |
| 포함 화면 | 목록 `/sales/inquiries` · 상세 처리 `/sales/inquiries/:id` (`App.tsx:250-251`) |
| **범위 밖** | **문의 등록·삭제** — 문의는 고객 채널이 접수하며 관리자 생성/삭제 진입점이 화면에 없다. 어댑터는 `createCrudAdapter` 라 `create`/`remove` 가 **동작하는 채로 존재**하지만(FS-026 의 티켓 어댑터처럼 거절 구현이 아니다) **호출부가 0건**이다 — 화면에 등록 버튼·행 선택·삭제·일괄 작업이 없다(`InquiryListPage.tsx:3-4` 헤더 주석: '문의는 고객 채널이 만들고 관리자는 처리·답변만 한다'). BE-051 §1·§7.6 이 이를 계약 범위 밖으로 확정한다 |
| **범위 경계** | **견적의 편집·발행 취소는 이 화면이 아니다** — 이 화면은 '견적 발행' 전이로 견적을 **만들기만** 하고, 그 뒤의 편집은 FS-050(견적 `/sales/quotes/:id/edit`)이 소유한다. 이 화면은 그리로 가는 링크만 제공한다(FS-051-EL-008.9 · EL-022) |
| 구현 경로 | `apps/admin/src/pages/sales/inquiries/**` (`InquiryListPage.tsx` · `InquiryDetailPage.tsx` · `types.ts` · `data-source.ts` · `components/InquiryTimeline.tsx` · `inquiries.test.ts`) |
| 대응 SCR | SCR-051 (미작성 — §7 #1) |
| 공통 컴포넌트 | `shared/ui/{Alert,alertActionRowStyle,Button,Card,CardTitle,ChevronLeftIcon,controlStyle,Empty,FormField,SearchField,SelectField,SeqCell,SeqHeaderCell,StatusBadge,TextareaField,Timeline,dl/dt/dd,hintStyle,pageTitleStyle,tableStyle,useToast,useUnsavedChangesDialog,visuallyHiddenStyle}` · `shared/crud/{createCrudAdapter,useCrudListQuery,useCrudUpdate,useListState,parseFilter,dev(LATENCY_MS·failIfRequested)}` · `shared/useRowNavigation` · `shared/async(isAbort)` · `shared/format(formatDateTime·formatNumber·directionParticle)` |

> **이름 충돌 주의**: 이 화면(`/sales/inquiries` — 영업 문의)은 **`/support/tickets`(FS-026 1:1 문의)와 다른 화면**이다. 둘 다 '문의'라 부르고 타임라인·담당 배정·상태 전이라는 결이 같지만, 도메인·저장소·타입(`Inquiry` vs `Ticket`)·라우트가 전부 별개다. 이 화면만 **견적 발행 전이**를 갖는다.
