# FS-027. 문의 유형 관리 (목록·등록/수정 모달·삭제)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 1:1 문의(티켓)와 답변 템플릿을 분류하는 문의 유형(taxonomy)을 조회·추가·수정·삭제한다. **두 축의 안전장치**를 갖는다 — ① 참조 중(티켓·템플릿)인 유형은 **삭제를 잠근다**(고아 참조 방지) ② 그래도 더 안 쓰고 싶으면 **사용여부(active)를 꺼서** 신규 선택지에서만 숨긴다(기존 참조는 유지) |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 고객센터 > 문의 유형 (`/support/categories`) |
| 포함 화면 | 목록 `/support/categories` · 등록/수정 모달(목록 안) · 삭제 확인 다이얼로그(목록 안). **하위 라우트 없음** — 짧은 taxonomy 엔티티라 모달로 편집한다(IA-06 무게 규칙) |
| 구현 경로 | `apps/admin/src/pages/support/categories/**` (`CategoriesPage.tsx` · `components/CategoryFormModal.tsx` · `data-source.ts` · `validation.ts`) |
| 도메인 정본 | `apps/admin/src/pages/support/_shared/**` — `domain.ts`(타입·`categoryInUse`·`categoryUsageLabel`·`CATEGORY_LABEL_MAX`) · `store.ts`(픽스처·저장소 API·삭제 차단) · `usage.ts`(`countCategoryUsage` — 티켓·템플릿 참조 집계). 1:1 문의·문의 답변 화면이 같은 저장소를 공유한다 |
| 대응 SCR | SCR-027 (미작성 — §7) |
| 공통 컴포넌트 | `shared/ui/{Modal,ConfirmDialog,useModalDirtyGuard,ToggleSwitch,ToastProvider,Alert,Button,Card,StatusBadge,FormField,PencilIcon,TrashIcon,PlusCircleIcon}` · `shared/crud/{useCrudListQuery,useCrudCreate,useCrudUpdate,useCrudDelete,createStoreAdapter,requiredText}` · `shared/async.isAbort` · `shared/format.formatNumber` |
