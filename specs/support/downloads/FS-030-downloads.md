# FS-030. 자료실 (목록 · 등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 고객센터 자료실에 배포할 파일(카탈로그·매뉴얼·양식·브로슈어)을 카테고리·노출 상태·키워드로 좁혀 찾고, 목록에서 바로 노출을 켜고 끄며, 자료를 등록·수정·삭제(단건·일괄)한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 고객센터 > 자료실 (`/support/downloads`) |

| 검증 정본 | `validation.ts` — `downloadSchema`(zod/mini) |

> **`_shared/` 대조 결과**: `apps/admin/src/pages/support/_shared/{domain,store,usage}.ts` 는 **1:1 문의(tickets)·문의 유형(categories)·문의 답변(replies)** 이 공유하는 모듈이며, 자료실은 이 중 **어느 것도 import 하지 않는다**(전 파일 확인). 자료실은 자기 `data-source.ts` 의 독립 픽스처(`SEED`)를 쓴다 — 문의 유형(`support/categories`)과 자료 카테고리(`DOWNLOAD_CATEGORY_OPTIONS`)는 별개 분류다(`types.ts:51` 주석 정본).

### 1.1 이 화면의 특이점 — 앱에서 **유일하게 업로드 심을 가진 화면**

`FileUploadField.tsx:8` 과 `data-source.ts:64` 가 **`POST /api/uploads`** 를 선언한다. 앱의 다른 첨부 화면(공용 `ImageUploadField`)은 업로드 심이 없어 `blob:` URL 로 대체하지만, 자료실은 **정식 업로드 엔드포인트를 계약으로 갖는다**(BE-030-EP-06).

**그러나 현재 코드는 파일을 보내지 않는다** — `FileUploadField.acceptFile`(`:150-158`)이 검증 통과 후 `onSelect(file.name, file.size)` 로 **파일명·용량만** 폼에 넘기고 `File` 객체를 버린다. `fetch`·`FormData`·`createObjectURL` 사용 0건(전 파일 확인). **주석과 코드가 일치한다** — '실제 업로드는 없다. 파일을 고르면 이름·용량만 폼으로 넘긴다'(`FileUploadField.tsx:7`). 저장되는 것은 파일 **메타데이터뿐**이며 고객이 내려받을 실체가 없다(§7 #1).

### 4.1 공통 예외 규칙

| 규칙 | 내용 |
|---|---|
| 네트워크 단절 | 목록 조회 실패는 FS-030-EL-011 인라인 배너(툴바는 남는다), 상세 조회 실패는 FS-030-EL-023 대체 화면. 쓰기 실패는 각 요소 처리(저장=배너, 삭제=다이얼로그 배너, 토글=토스트) |
| 세션 만료 | 쿼리 계층 401 인터셉터가 앱 전역에서 재인증 경로로 보낸다(`shared/query/queryClient.ts:37-43`) — 이 화면에 개별 처리는 없다 |

| 중복 제출 | 폼 제출은 **동기 락 + `disabled`** 이중 방어(`useCrudForm`). 삭제·일괄 삭제는 요청 중 확인 버튼 비활성. 행 토글은 `pendingId` 로 그 행을 잠근다 |
| 실패 통지의 자리 | ① 조회 실패는 인라인 배너 ② 저장 실패는 폼 상단 배너 ③ 삭제 실패는 다이얼로그 안 배너 ④ 행 토글 실패는 토스트 ⑤ 삭제·저장 **성공**은 토스트 |
| 낙관적 업데이트 | **없다.** 행 토글이 비관적이다(`useCrudRowUpdate` — 서버 응답 후 무효화로 갱신). 롤백 대상이 없다 |
| 유령 저장 방지 | `createCrudAdapter.update`/`remove` 가 없는 id 를 조용히 통과시키지 않고 **409 를 던진다**(`shared/crud/crud.ts:71-73`·`:82-84`) — 다른 관리자가 지운 항목을 편집하면 '저장했습니다' 대신 충돌 다이얼로그가 뜬다 |
| 동시 조회 | 목록·상세 조회는 각각 동시에 1건만 유지된다(react-query, `staleTime: 30초`) |
| 권한 없음 | 역할 기반 분기 없음. 서버 권한 응답은 조회=인라인 배너, 쓰기=각 요소 실패로 떨어진다. 은닉 정책은 BE-030 이 확정 |
| 행 선택의 수명 | 선택은 화면 로컬 상태. **일괄 삭제 전원 성공 시에만 해제된다.** ⚠ **필터·검색 변경 시 해제되지 않는다** — 공용 훅의 계약 위반(§7 #2) |
| 파일 업로드 | **실제 업로드가 없다**(§1.1). 검증은 클라이언트 로컬(확장자·용량)이며 요청을 발사하지 않는다. 진행률·취소·실패 fallback 이 없다 — 보낼 것이 없기 때문이다 |
