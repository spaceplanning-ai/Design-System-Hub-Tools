# FS-029. 고객노출 FAQ 큐레이션

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 이미 **발행된** FAQ 를 고객센터에서 어떤 순서로 보여줄지(재정렬)·노출할지(토글)·BEST 로 상단 고정할지(토글)만 큐레이션한다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 고객센터 > 자주 묻는 질문 (`/support/faq`) |
| 포함 화면 | 목록 겸 큐레이션 `/support/faq` **한 개뿐**. 하위 라우트(상세·등록·수정) 없음 |
| 구현 경로 | `apps/admin/src/pages/support/faq/**` (`CustomerFaqPage.tsx` · `components/CustomerFaqTable.tsx` · `data-source.ts` · `types.ts` · `faq.test.ts`) |
| 대응 SCR | SCR-029 (미작성 — §7) |
| 공통 컴포넌트 | `shared/ui/{Alert,Button,Card,ToggleSwitch,useToast,hintStyle}` · `@tds/ui` 재수출 `{useReorderableRows,ReorderGripCell,ReorderGripHeaderCell,ReorderMoveButtons,SeqCell,SeqHeaderCell}` · `shared/async(isAbort)` · `shared/format(formatNumber)` · `shared/crud(failIfRequested·LATENCY_MS)` |

### 1.1 범위 — 이 화면은 FAQ 를 **작성하지 않는다**

**FS-010(`/content/faq`, 콘텐츠 관리 FAQ)과 다른 화면이다.** 두 화면의 경계를 코드가 명시적으로 선언한다(`types.ts:3-6` · `CustomerFaqPage.tsx:2-5`):

| 구분 | FS-010 `/content/faq` | **FS-029 `/support/faq` (이 문서)** |
|---|---|---|
| 성격 | FAQ **작성·관리** | 고객센터 **노출 큐레이션** |
| 등록·수정·삭제 | 있음 | **없다** — 폼·삭제 다이얼로그·행 선택 열이 존재하지 않는다 |
| 카테고리 관리 | 있음(모달) | 없다 — 카테고리는 표시용 라벨(`categoryLabel`)로만 읽는다 |
| 검색·필터·페이지네이션 | 있음 | **없다** |
| 고유 축 | 정렬 순서(`order`) | 표시 순서(`order`) + **BEST 고정(`pinned`)** ← 콘텐츠 FAQ 에 없는 축 |
| 답변(`answer`) | 편집·표시 | **다루지 않는다** — 이 화면의 도메인 타입에 `answer` 필드가 없다(`types.ts:9-20`) |

- **범위 밖**: FAQ 등록·수정·삭제, 카테고리 생성·삭제, 답변 본문 편집. 작성이 필요하면 FS-029-EL-001.1 링크로 `/content/faq` 에 위임한다.
- 이 경계는 **어댑터 심(`data-source.ts:63`)이 뒷받침한다** — 조회 1건 + 순서 변경 1건 + 부분 수정 1건뿐이고 **POST·DELETE 심이 없다**.
