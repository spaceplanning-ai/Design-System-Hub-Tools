# FS-028. 문의 답변 — 답변 템플릿 관리 (목록·등록/수정)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 자주 쓰는 답변 문구를 **템플릿(매크로·상용구)** 으로 등록·수정·삭제하고 유형 태그로 분류한다. 저장된 템플릿은 **1:1 문의 상세(FS-026-EL-025)가 답변 작성 시 골라 삽입**하며, 그때 치환 변수가 실제 값으로 채워진다 |
| **화면명과 실제 관리 대상** | GNB 명칭은 '**문의 답변**'(`nav-config.ts:168`)이지만 **이 화면이 관리하는 것은 개별 답변이 아니라 답변 템플릿**이다. 코드가 이를 일관되게 말한다: `ENTITY_LABEL = '답변 템플릿'`(`RepliesPage.tsx:16` · `ReplyFormPage.tsx:26`) · 도메인 타입 `ReplyTemplate` · 연동 심 경로 `reply-templates` · 폼 안내 '자주 쓰는 답변을 템플릿으로 저장해 두면 티켓 답변 작성 시 골라 삽입할 수 있습니다.'(`ReplyFormPage.tsx:83`) · 리소스 키 `'support-templates'`. **개별 고객 답변은 FS-026(1:1 문의)의 타임라인이 소유한다** — 이 화면에는 고객·문의번호·발송 개념이 없다. 화면 제목과 도메인이 어긋나는 문제는 §7 #1 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 고객센터 > 문의 답변 (`/support/replies`) |
| 포함 화면 | 목록 `/support/replies` · 등록 `/support/replies/new` · 수정 `/support/replies/:id/edit` |
| 구현 경로 | `apps/admin/src/pages/support/replies/**` (`RepliesPage.tsx` · `ReplyFormPage.tsx` · `data-source.ts` · `validation.ts`) |
| 도메인 타입 위치 | **`types.ts` 가 없다.** `ReplyTemplate` · `ReplyTemplateInput` · 상수(`TEMPLATE_TITLE_MAX`=60 · `TEMPLATE_BODY_MAX`=1000 · `TEMPLATE_ALL_LABEL`='전체')와 순수 규칙(`applyTemplate` · `templatesForCategory` · `searchTemplates`)이 **`apps/admin/src/pages/support/_shared/domain.ts`** 에 있다 — 1:1 문의·문의 유형과 도메인을 공유하기 때문이다 |
| 대응 SCR | SCR-028 (미작성 — §7) |
| 공통 컴포넌트 | `shared/crud/{useCrudList,CrudListShell,CrudTable,useCrudForm,FormPageShell,createStoreAdapter,requiredText}` · `shared/ui/{SearchField,SelectField,TextareaField,FormField,StatusBadge,Button,PlusCircleIcon,ConfirmDialog,SelectionBar,useRowSelection,useToast,useUnsavedChangesDialog,Empty,RowActions,errorIdOf,controlStyle}` · `shared/bulk(settleAll)` |
| 검증 정본 | `replies/validation.ts` 의 zod 스키마 `replyTemplateSchema` |

> **이 화면은 공용 CRUD 프레임워크를 거의 그대로 쓴다**: 목록은 `useCrudList` + `CrudListShell`, 폼은 `useCrudForm` + `FormPageShell`. 화면 고유 코드는 컬럼 정의·검색 툴바·필드 3개뿐이다. 그래서 FS-026(1:1 문의, 손조립)과 달리 로딩 파생·삭제 다이얼로그·충돌 처리·중복 제출 방지를 **프레임워크에서 상속**한다.

### 4.1 공통 예외 규칙

| 규칙 | 내용 |
|---|---|
| 네트워크 단절 | 목록 조회 실패는 인라인 배너(FS-028-EL-009, 툴바는 남는다). 폼 로드 실패는 폼 대체 배너(FS-028-EL-025). 저장 실패는 카드 배너(FS-028-EL-017). 삭제 실패는 다이얼로그 안 배너(FS-028-EL-010·EL-011). **오프라인 감지는 앱 전역에 없다**(`navigator.onLine` 0건) — §7 #21 |
