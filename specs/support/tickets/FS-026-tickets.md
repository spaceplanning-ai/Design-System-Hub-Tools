# FS-026. 1:1 문의 (목록·상세 처리)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 고객 채널로 접수된 1:1 문의를 상태·우선순위·채널·유형·키워드로 좁혀 훑고(트리아지), 개별 문의에서 담당을 배정하고 처리 상태를 바꾸며 고객답변·내부메모를 작성해 처리 이력에 남긴다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1) |
| 진입 경로 | 좌측 GNB > 고객센터 > 1:1 문의 (`/support/tickets`) |
| 포함 화면 | 목록 `/support/tickets` · 상세 처리 `/support/tickets/:id` |
| **범위 밖** | **문의 등록·삭제** — 문의는 고객 채널(웹·카카오톡·네이버톡톡·전화·이메일)이 접수하며 관리자 생성/삭제 진입점이 화면에 없다. 어댑터의 `create`/`remove` 는 인터페이스를 채우기 위해 존재하며 호출되면 거절한다(`'문의는 고객 채널에서 접수됩니다.'` / `'문의는 삭제할 수 없습니다.'` — `data-source.ts:27-29,35-37`). 저장소도 '문의는 고객이 만든다 — 관리자 생성/삭제 없음'을 못박는다(`_shared/store.ts:273`). BE-026 §1 이 이를 계약 범위 밖으로 확정한다 |
| 구현 경로 | `apps/admin/src/pages/support/tickets/**` · 도메인 `apps/admin/src/pages/support/_shared/{domain,store,usage}.ts` |
| 대응 SCR | SCR-026 (미작성 — §7) |
| 공통 컴포넌트 | `shared/ui/{SearchField,SelectField,StatusBadge,SeqCell,SeqHeaderCell,Alert,Button,Card,CardTitle,FormField,TextareaField,Timeline,ChevronLeftIcon,useToast,useUnsavedChangesDialog,controlStyle,errorIdOf,tableStyle,pageTitleStyle,buttonStyle}` · `shared/crud/{useCrudListQuery,useCrudUpdate,dev(LATENCY_MS·failIfRequested)}` · `shared/useRowNavigation` · `shared/async(isAbort)` · `shared/format` |

> **문의 유형·답변 템플릿은 이 화면의 소유가 아니다**: 유형 필터 선택지(FS-026-EL-005)는 문의 유형 화면(FS-027)의 데이터를, 답변 템플릿 선택지(FS-026-EL-025)는 문의 답변 화면(FS-028)의 데이터를 읽어 온다. 이 화면은 **소비자**다.
