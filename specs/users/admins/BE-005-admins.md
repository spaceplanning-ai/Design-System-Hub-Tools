# BE-005. 관리자(운영진) 관리 목록 백엔드 기능 명세

## 1. 개요

| 항목 | 내용 |
|---|---|
| 대상 화면 | FS-005 관리자(운영진) 관리 목록 (`/users/admins`) |
| 범위 | 운영자 목록 조회(그룹 필터 · 키워드 검색 · 페이징 · 전체/그룹별 건수), 운영진 그룹 목록 조회, 운영자 상세 조회(회원 상세 화면 재사용) |
| 범위 밖 | **운영진 그룹 생성** — 이 화면에 생성 진입점이 없다(FS-005 §1.1). **운영자 삭제 · 알림 발송 · 쓰기 전반** — 목록에 행 액션이 없고 `data-source.ts` 에 쓰기 함수가 없다. **운영자 계정 생성** — 이 계약에 존재하지 않는다 |
| 전제 | 인증은 세션 쿠키 기반. 모든 경로는 `/api` 프리픽스. 응답 본문은 `application/json; charset=utf-8`. 시각은 ISO 8601(오프셋 포함) |
| 프론트 어댑터 | `apps/admin/src/pages/admins/data-source.ts` |
| 도메인 타입 | `apps/admin/src/pages/admins/types.ts` |
| 상세 명세 재사용 | 운영자 상세(BE-005-EP-03)는 **회원 상세 계약(BE-004)** 을 그대로 따른다 — `fetchAdminDetail` 이 `MemberDetail` 을 돌려준다. §2 에러 봉투 · §3 권한 모델은 BE-003 을 상속한다 |

> **에러 봉투·권한 모델은 회원 도메인(BE-003 §2·§3)을 상속한다.** 운영자 도메인이 다른 형식을 만들면 백엔드가 두 벌을 구현하게 된다. 아래 §2·§3 은 BE-003 과 동일한 봉투·역할·은닉 원칙을 이 도메인의 리소스에 적용한 것이다.

## 2. 공통 에러 봉투

모든 4xx / 5xx 응답은 아래 봉투를 따른다(BE-003 §2 와 동일). 각 엔드포인트는 **에러코드 목록만** 나열한다.

```json
{
  "error": {
    "code": "ADMIN_NOT_FOUND",
    "message": "회원을 찾을 수 없습니다.",
    "fields": null,
    "traceId": "01J8X4K2M9P7Q3R5S6T8V0W2Y4"
  }
}
```

| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `error.code` | string | O | 기계 판정용 에러코드(SCREAMING_SNAKE_CASE) |
| `error.message` | string | O | 사용자에게 그대로 보여줄 수 있는 문구. **내부 정보 노출 금지** |
| `error.fields` | array \| null | X | 400 검증 실패 시에만. `name` 은 쿼리 파라미터명 |
| `error.traceId` | string | O | 로그 상관관계 ID |

### 2.1 공통 에러코드

| HTTP | 에러코드 | 발생 조건 | 프론트 처리 |
|---|---|---|---|
| 400 | `VALIDATION_FAILED` | 쿼리 파라미터 제약 위반 | `error.fields` 를 매핑. 프론트에 매핑 UI 가 없어 조회 실패로 표시된다 |
| 401 | `UNAUTHENTICATED` | 세션 없음·만료 | `/login?returnUrl=<현재경로>` 로 이동 (현재 프론트 미구현 — §7) |
| 403 | `FORBIDDEN` | 인증됐으나 운영진 도메인 조회 권한 부족 | FS-005-EL-010 조회 실패 배너 |
| 404 | `ADMIN_NOT_FOUND` | 운영자 id 없음·삭제됨, 또는 운영진 도메인 읽기 권한 없음(§3.2 은닉) | 어댑터가 `Error('회원을 찾을 수 없습니다')` 로 변환 → FS-004-EL-012 |
| 429 | `RATE_LIMITED` | 레이트리밋 초과. `Retry-After`(초) 헤더 동반 | 재시도 안내 |
| 500 | `INTERNAL_ERROR` | 서버 내부 오류 | 일반 문구 + `traceId`. **원인 노출 금지** |
| 504 | `REQUEST_TIMEOUT` | 서버 처리 상한 초과 | 500 과 동일한 일반 문구 + `traceId` |

## 3. 인증 · 권한 모델

### 3.1 역할

| 역할 | 설명 | 비고 |
|---|---|---|
| `admin` | 운영진 도메인 전체 권한 — 운영자 목록 · 그룹 목록 · 운영자 상세 조회 | 이 문서의 모든 엔드포인트를 호출할 수 있다 |
| 운영진 도메인 조회 권한이 없는 인증 관리자 | 운영자 명부에 접근할 수 없는 그 밖의 역할 | 컬렉션은 **403**, 개별 운영자는 **404** 은닉 (§3.2) |

> **운영자 명부 열람 자체가 민감하다.** 운영진 관리 화면은 '누가 관리 권한을 가졌는가' 를 드러낸다. 회원 목록보다 좁은 권한으로 보호해야 하며, 열람 권한 정책(어떤 역할이 운영자 목록을 볼 수 있는가)은 아키텍처 판정 사안으로 §7 에 남긴다. **프론트에는 역할 분기가 없다**(FS-005 §4.1) — 권한 강제는 전적으로 서버의 책임이다.

### 3.2 403 vs 404 은닉 정책

BE-003 §3.2 의 원칙을 이 도메인에 적용한다.

| 리소스 | 권한 부족 시 응답 | 근거 |
|---|---|---|
| `GET /api/admins` (컬렉션) | 403 `FORBIDDEN` | 운영자 컬렉션의 존재는 비밀이 아니다. 열거할 개별 id 가 노출되지 않는다 |
| `GET /api/admin-groups` (컬렉션) | 403 `FORBIDDEN` | 그룹 컬렉션의 존재는 비밀이 아니다 |
| `GET /api/admins/:id` (개별) | 운영진 도메인 읽기 권한 없음 → 404 `ADMIN_NOT_FOUND` | 운영자 id 열거를 막는다. 존재 여부 자체가 정보다. 어댑터가 이 코드를 '회원을 찾을 수 없습니다' 로 변환해 FS-004-EL-012 를 띄운다 |

### 3.3 CSRF · 3.4 타임아웃 상한

- **CSRF**: 이 도메인은 **읽기 전용**이다. 상태 변경 요청이 없으므로 `X-CSRF-Token` 요구 대상이 없다. 쓰기 계약이 생기면 BE-003 §3.3 을 그대로 적용한다.
- **타임아웃**: 조회 계열 서버 처리 상한 5초 → 초과 시 504 `REQUEST_TIMEOUT`. 프론트 권고 상한 10초(현재 프론트에 상한 없음 — §7).

## 4. 엔드포인트 명세

### BE-005-EP-01 · 운영자 목록 조회

| 항목 | 내용 |
|---|---|
| 근거 (FS) | FS-005-EL-001.1, FS-005-EL-001.2, FS-005-EL-004, FS-005-EL-005, FS-005-EL-006, FS-005-EL-009, FS-005-EL-010, FS-005-EL-010.1 |
| 메서드 · 경로 | `GET /api/admins` |
| 권한 | `admin` (운영진 도메인 조회 권한) |
| 멱등성 | 멱등 (GET). 재시도 안전. FS-005-EL-010.1('다시 시도')은 같은 조건으로 같은 요청을 다시 보낸다 |
| 페이징 | offset 방식. `page` 1-base, `size` 기본 10 · 상한 100 (프론트 `PAGE_SIZE = 10`) |
| 레이트리밋 | 관리자당 분당 120회 |

**요청 — 쿼리**

| 이름 | 타입 | 필수 | 제약 | 설명 |
|---|---|---|---|---|
| `groupId` | string | X | 그룹 id 또는 `all` (기본 `all`). 1–64자 | 그룹 필터. `all` 은 그룹 조건 없음 (프론트 상수 `GROUP_ALL = 'all'`) |
| `keyword` | string | X | 0–50자. 서버가 앞뒤 공백을 제거한다. 빈 문자열이면 조건 없음 | 닉네임 **또는** 계정에 대한 대소문자 무시 부분 일치 |
| `page` | number | X | 정수 ≥ 1 (기본 1) | 페이지 번호 |
| `size` | number | X | 정수 1–100 (기본 10) | 페이지 크기 |

- `page` 가 총 페이지 수를 넘으면 `admins` 는 빈 배열, `total` 은 조건에 걸린 실제 건수를 반환한다. 서버는 페이지를 자동 보정하지 않는다(FS-005 §7 #8 — 프론트가 빈 상태 FS-005-EL-008 을 표시한다).
- `keyword` 가 50자를 넘으면 400. 프론트에 길이 제약이 없으므로(FS-005-EL-004) 어댑터가 그대로 보내며, 400 은 조회 실패로 표시된다 — §7 에 변경 요청 로 기록한다.

**응답 200** — `AdminListResult`

```json
{
  "admins": [
    {
      "id": "A-00003",
      "nickname": "박백민",
      "account": "whitepark@example.com",
      "groupId": "admin-2",
      "group": "운영팀 admin_2",
      "joinedAt": "2024-06-26",
      "department": "운영본부",
      "position": "이사",
      "phone": "010-7173-7804",
      "memo": ""
    }
  ],
  "totalAll": 5,
  "groupCounts": { "admin": 3, "admin-2": 2 },
  "total": 2
}
```

| 필드 | 타입 (types.ts) | 설명 |
|---|---|---|
| `admins` | `readonly AdminUser[]` | 현재 페이지의 운영자. 필드명·타입이 `AdminUser` 와 일치한다 |
| `totalAll` | number | **검색·그룹 필터와 무관한 전체 운영자 수** (FS-005-EL-001.1 배지 — 좌측 배지가 검색할 때마다 흔들리면 안 된다) |
| `groupCounts` | `AdminGroupCounts` = `Record<string, number>` (키 = 그룹 id) | 그룹별 **전체** 운영자 수. `GET /api/admin-groups` 의 모든 그룹 id 를 키로 포함한다 |
| `total` | number | 필터·검색을 적용한 뒤의 전체 건수. 페이지네이션(FS-005-EL-009)과 조회 요약(FS-005-EL-005)이 쓴다 |

- `AdminUser.department` · `AdminUser.position` 은 **빈 문자열일 수 있다**(FS-005-EL-006.7 · 006.8 — 값을 지어내지 않는다). 서버는 값이 없으면 `""` 를 보낸다.
- `AdminUser.joinedAt` 은 `YYYY-MM-DD` 다. 목록은 상대 시각을 쓰지 않으므로(FS-005-EL-006.6) ISO date-time 필드가 없다 — 회원 목록(`joinedAtIso`)과 다른 지점이다.

**에러 목록**

| HTTP | 에러코드 | 조건 |
|---|---|---|
| 400 | `VALIDATION_FAILED` | `groupId` 형식·`keyword`(50자)·`page`(<1)·`size`(범위 밖) 위반 |
| 401 | `UNAUTHENTICATED` | 세션 없음·만료 |
| 403 | `FORBIDDEN` | 운영진 도메인 조회 권한 부족 (컬렉션은 은닉하지 않는다 — §3.2) |
| 429 | `RATE_LIMITED` | 분당 120회 초과 |
| 500 | `INTERNAL_ERROR` | 내부 오류 |
| 504 | `REQUEST_TIMEOUT` | 서버 처리 5초 초과 |

### BE-005-EP-02 · 운영진 그룹 목록 조회

| 항목 | 내용 |
|---|---|
| 근거 (FS) | FS-005-EL-001 |
| 메서드 · 경로 | `GET /api/admin-groups` |
| 권한 | `admin` (운영진 도메인 조회 권한) |
| 멱등성 | 멱등 (GET). 재시도 안전 |
| 페이징 | 없음 — 전량 반환. **그룹 수 상한 200개** (BE-003 §7.5 와 동일 근거). 생성 API 가 이 계약에 없으므로 상한 강제는 그룹을 만드는 도메인의 책임이다 |
| 레이트리밋 | 관리자당 분당 60회 |

**요청** — 파라미터 없음.

**응답 200** — `readonly AdminGroup[]`

```json
[
  { "id": "admin", "label": "운영팀 admin" },
  { "id": "admin-2", "label": "운영팀 admin_2" }
]
```

| 필드 | 타입 (types.ts) | 설명 |
|---|---|---|
| `[].id` | string | 그룹 id. `fetchAdmins` 의 `groupId` 쿼리와 `groupCounts` 의 키로 그대로 쓰인다 |
| `[].label` | string | 그룹 표시명. 좌측 필터는 `'운영 - <label>'` 로, 표는 `label` 을 그대로 쓴다(FS-005-EL-001.2 · FS-005-EL-006.5) |

- **이 엔드포인트는 운영진(staff) 그룹만 반환한다.** 회원 그룹(`type=member`)은 회원 목록(BE-003-EP-02)의 관심사이며 여기 섞이지 않는다. 두 도메인은 서로 다른 컬렉션을 소비한다.
- **그룹 생성 경로가 없다** — FS-005 §1.1 이 생성 진입점을 두지 않았다. 그룹은 별도 운영 정책으로 부여되며, 이 계약은 조회만 다룬다.

**에러 목록**

| HTTP | 에러코드 | 조건 |
|---|---|---|
| 401 | `UNAUTHENTICATED` | 세션 없음·만료 |
| 403 | `FORBIDDEN` | 운영진 도메인 조회 권한 부족 (컬렉션은 은닉하지 않는다 — §3.2) |
| 429 | `RATE_LIMITED` | 분당 60회 초과 |
| 500 | `INTERNAL_ERROR` | 내부 오류 |
| 504 | `REQUEST_TIMEOUT` | 서버 처리 5초 초과 |

### BE-005-EP-03 · 운영자 상세 조회

| 항목 | 내용 |
|---|---|
| 근거 (FS) | FS-005-EL-006.3, FS-005-EL-006.10, FS-005-EL-006.11 (→ FS-004 회원 상세 재사용) |
| 메서드 · 경로 | `GET /api/admins/:id` |
| 권한 | `admin` (운영진 도메인 조회 권한) |
| 멱등성 | 멱등 (GET). 재시도 안전 |
| 페이징 | N/A — 단건 조회 |
| 레이트리밋 | 관리자당 분당 120회 |

**요청 — 경로**

| 위치 | 이름 | 타입 | 필수 | 제약 |
|---|---|---|---|---|
| 경로 | `id` | string | O | 운영자 id (예: `A-00003`). 1–64자 |

**응답 200** — `MemberDetail` (회원 상세와 **동일한 스키마** — `apps/admin/src/shared/domain/member`)

- 상세 화면은 `MemberDetailPage`(FS-004)를 재사용하므로 응답 스키마·요소·예외는 **BE-004 를 그대로 따른다.** 이 문서는 그 계약을 새로 정의하지 않는다.
- **운영자↔회원 데이터 매핑의 부적합은 FS-005 §7 #3 미결이다**: 현재 어댑터(`buildAdminDetail`)는 운영자에게 회원 등급(`tier`) · 동의정보 · 쿠폰 · 적립금 개념을 빈 값/기본값으로 채운다. 서버가 붙을 때 운영자 상세의 필드 계약(부서·직급 포함 여부, 등급 필드 생략 여부)을 아키텍처 이 확정해야 한다 — 그 전까지 이 엔드포인트는 `MemberDetail` 형태를 유지한다.

**에러 목록**

| HTTP | 에러코드 | 조건 |
|---|---|---|
| 401 | `UNAUTHENTICATED` | 세션 없음·만료 |
| 404 | `ADMIN_NOT_FOUND` | 없는 id · 삭제된 운영자 · 운영진 도메인 읽기 권한 없음(은닉 — §3.2). 어댑터가 '회원을 찾을 수 없습니다' 로 변환 |
| 429 | `RATE_LIMITED` | 분당 120회 초과 |
| 500 | `INTERNAL_ERROR` | 내부 오류 |
| 504 | `REQUEST_TIMEOUT` | 서버 처리 5초 초과 |

## 7. 핵심 판정 (근거를 남긴다)

### 7.1 쓰기 엔드포인트가 없다 — 그것이 계약이다

이 도메인에는 생성·수정·삭제 엔드포인트가 **하나도 없다.** 근거: FS-005 §1.1 이 그룹 생성·행 액션·일괄 액션을 전부 두지 않았고, `data-source.ts` 에 쓰기 함수가 없다. **없는 프론트 호출에 대응하는 엔드포인트를 발명하지 않는다.** 운영자 계정 생성·삭제·권한 변경은 별도 화면(권한 관리 FS-006 및 미작성 운영자 관리 흐름)의 관심사이며, 필요해지면 FS 개정 + BE 신규 엔드포인트로 승격한다.

### 7.2 운영자 상세는 회원 상세 계약(BE-004)을 상속한다

`/users/admins/:id` 가 `MemberDetailPage` 를 재사용하므로, 상세의 요청/응답/에러/권한은 BE-004 를 그대로 쓴다. 다만 **운영자↔회원 데이터 매핑이 부적합**하다(FS-005 §7 #3): 운영자에게는 회원 등급·동의정보·쿠폰·적립금이 없는데 현재 어댑터가 빈 값으로 채운다. 그리고 상세의 ⋯ 메뉴(삭제·알림 발송)가 **회원 도메인 어댑터**를 호출한다(FS-005 §7 #4) — 운영자 삭제·알림이 회원 엔드포인트로 나간다. 두 문제는 운영자 전용 상세 계약이 정해지기 전까지 이 문서가 해결하지 않고 아키텍처 판정으로 남긴다.

### 7.3 운영자 명부 열람 권한 — 아키텍처 판정 사안

운영자 명부는 '누가 관리 권한을 가졌는가' 를 드러내므로 회원 목록보다 좁은 권한으로 보호해야 한다. 어떤 역할이 `GET /api/admins` 를 볼 수 있는지(예: `admin` 만, 또는 권한 관리 read 를 가진 역할)는 권한 매트릭스(FS-006/BE-006)와 함께 확정해야 한다 — §7.4 로 이관한다.

### 7.4 후속 이관 (프론트 구현 · UI 기획 · 아키텍처)

| # | 내용 | 이관 대상 |
|---|---|---|
| 1 | 401 `UNAUTHENTICATED` 를 감지해 `/login?returnUrl=` 으로 보내는 처리가 프론트에 없다 (FS-005 §4.1) | UI 기획 · 프론트 구현 쪽 변경 요청 |
| 2 | 프론트 요청 타임아웃 상한이 없다. 권고: 조회 10초 (§3.4) | UI 기획 · 프론트 구현 쪽 변경 요청 |
| 3 | `keyword` 50자 상한이 프론트에 없다 — 51자 이상이면 400 이 조회 실패 배너로 표시된다 | UI 기획 쪽 변경 요청 |
| 4 | 그룹 목록 조회 실패 시 전용 재시도 경로가 없다 · 페이지 초과 자동 보정이 없다 (FS-005 §7 #8) | UI 기획 쪽 변경 요청 |

| 6 | **운영자 명부 열람 권한 정책**(어떤 역할이 볼 수 있는가)이 미정 (§7.3) | 아키텍처 (권한 판정) |
