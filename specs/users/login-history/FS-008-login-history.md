# FS-008. 로그인 이력 (읽기 전용 감사 로그)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 회원·운영자의 로그인 시도(성공·실패)를 결과·계정 유형·기간·키워드로 좁혀 조회하고, 연속 실패로 드러나는 계정 탈취 시도를 확인하며, 필요하면 CSV 로 내보낸다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1 참조) |
| 진입 경로 | 좌측 GNB > 사용자 관리 > 로그인 이력 (`nav-config.ts`, `to: '/users/login-history'`) |
| 관련 화면 | FS-001 로그인(이 이력이 기록하는 시도의 출처. 실패 사유 정의는 BE-001) / FS-003 회원 상세·FS-005 운영자 상세(행 클릭 이동 대상) |
| 대응 SCR | SCR-008 (미작성 — §7) |
| 구현 경로 | `apps/admin/src/pages/login-history/LoginHistoryPage.tsx` 및 `components/**`. 기간 계산은 `period.ts` · 직접 지정 검증은 `validation.ts`(zod) · 타입·표시 규칙은 `types.ts` |

### 1.1 이 화면의 설계 원칙 (요구사항 — 설계 근거가 곧 명세다)

| 원칙 | 내용과 이유 |
|---|---|
| **읽기 전용 감사 로그 — 삭제·수정·행 ⋯ 메뉴가 없다** | 감사 기록은 **불변**이어야 한다. 지울 수 있는 감사 로그는 감사 로그가 아니다 — 침입자가 가장 먼저 지우는 것이 자기 흔적이다. 그래서 `data-source.ts` 에 쓰기 함수가 **하나도 없고**, 없는 것을 화면이 부를 수 없다. 이 부재가 불변성의 첫 방어선이다(`types.ts` · `LoginHistoryPage.tsx` 머리말) |
| **체크박스가 없다** | 일괄 액션이 없으므로 선택도 없다. 회원 관리에서 '체크박스가 아무 일괄 액션과도 연결되지 않았다' 가 결함으로 지적됐다(FS-003 검수 · FS-005 §7 #2) — 여기서 반복하지 않는다 |
| **미등록 계정도 이력에 남는다 (열거 오라클 방지)** | 미등록 이메일의 로그인 시도도 서버에서 세어지고 잠기며 이력에 남는다. 미등록 계정을 다르게 취급하면 그 차이가 계정 존재 여부를 알려주는 **열거 오라클**이 된다(BE-001 §3.2·§4). 로그인 화면은 미등록 계정에도 '이메일 또는 비밀번호가 일치하지 않습니다' 만 보이지만(FS-001-EL-019), **관리자용 감사 이력은 진짜 사유('미등록 계정')를 기록한다** — 그것을 보라고 이 화면이 있다 |
| **연속 실패 배지가 이 화면의 존재 이유다** | 같은 계정의 연속 실패 2회부터 '실패 N회 연속' 배지를 단다(1회는 오타, 2회부터가 신호). **연속 실패 횟수는 서버가 계산해 내려준다** — 페이지네이션된 목록을 프론트가 훑어 세면 페이지 경계에서 값이 왜곡된다(1페이지의 3연속이 2페이지에서 1연속이 됨) |
| **행 클릭 → 그 계정의 상세. 미등록 계정은 이동하지 않는다** | 회원이면 회원 상세, 운영자면 운영자 상세로 간다(두 화면 모두 존재). 미등록 계정은 가리킬 레코드가 없어 `subjectId=null` 이고, 그 행은 클릭해도 이동하지 않는다 — 없는 계정을 있는 것처럼 보이게 하지 않는다 |

### 4.1 공통 예외 규칙

| 규칙 | 내용 |
|---|---|
| 네트워크 단절 | 목록 조회는 실패로 처리되어 FS-008-EL-012(인라인 배너)로 대체 렌더된다. 내보내기는 FS-008-EL-013(실패 토스트)로 떨어진다. 서버 오류·네트워크 단절·타임아웃을 구분하는 문구가 구현에 없다. 개발용으로 `?fail=history`·`?fail=export`·`?fail=all` 쿼리로 실패 경로를 재현한다 |

| 중복 제출 | 이 화면의 유일한 명시적 작업인 내보내기(FS-008-EL-006)는 요청 중 버튼이 비활성되어 차단된다. 조회는 조건이 바뀌면 이전 요청을 abort 하고 최신만 반영한다 |

| 감사 불변성 | 이 화면에는 쓰기 경로가 없다 — 삭제·수정·상태 변경 함수가 `data-source.ts` 에 존재하지 않는다. 감사 기록은 조회·내보내기만 한다(§1.1). 서버도 이 불변성을 강제한다(BE-008 §7) |
