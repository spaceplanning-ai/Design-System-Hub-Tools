# BE-003. 회원 관리 목록 백엔드 기능 명세

## 1. 개요

| 항목 | 내용 |
|---|---|
| 대상 화면 | FS-003 회원 관리 목록 (`/users/members`) |
| 범위 | 회원 목록 조회(등급 × 그룹 AND 필터 · 키워드 검색 · 페이징 · 등급별/그룹별 건수), 회원 그룹 목록 조회, 회원 그룹 생성, 회원 목록 내보내기 조회, 회원 삭제, 알림 발송 |
| 범위 밖 | **회원 생성** — 고객은 회원가입으로만 유입된다(FS-003 §1.1). 관리자가 회원을 만드는 API 는 이 계약에 존재하지 않는다. **CSV 직렬화** — 프론트 `toCsv(members)` 가 브라우저에서 수행한다(§7.2 판정). **운영진 그룹 소비** — `type=staff` 그룹은 `/users/admins` 도메인의 관심사다(§7.4 판정) |
| 전제 | 인증은 세션 쿠키 기반. 모든 경로는 `/api` 프리픽스. 응답 본문은 `application/json; charset=utf-8`. 시각은 ISO 8601(오프셋 포함) |
| 프론트 어댑터 | `apps/admin/src/pages/members/data-source.ts` |
| 도메인 타입 | `apps/admin/src/pages/members/types.ts` |
| 회원 상세 명세 | BE-004 (`specs/users/members/detail/BE-004-member-detail.md`) — §2 에러 봉투 · §3 권한 모델을 이 문서와 **동일하게** 쓴다 |

> **BE-001(로그인 백엔드 명세) 부재**: `specs/login/` 에 BE-001 이 아직 없다(FS-001 만 존재). 따라서 이 문서의 §2 공통 에러 봉투와 §3 인증·권한 모델이 **회원 도메인의 기준**이며, D10 템플릿(`docs/_templates/D10-backend-spec.md`)의 봉투 정의를 그대로 따른다. BE-001 이 작성될 때 이 봉투·권한 모델을 재사용한다 — 도메인마다 다른 에러 형식을 만들면 백엔드가 두 벌을 구현하게 된다.

## 2. 공통 에러 봉투

모든 4xx / 5xx 응답은 아래 봉투를 따른다. 예외 없다. 각 엔드포인트(§4)는 **에러코드 목록만** 나열하며 자체 응답 형식을 정의하지 않는다.

```json
{
  "error": {
    "code": "MEMBER_NOT_FOUND",
    "message": "회원을 찾을 수 없습니다.",
    "fields": [
      { "name": "amount", "code": "OUT_OF_RANGE", "message": "1 이상 1,000,000 이하여야 합니다." }
    ],
    "traceId": "01J8X4K2M9P7Q3R5S6T8V0W2Y4"
  }
}
```

| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `error.code` | string | O | 기계 판정용 에러코드(SCREAMING_SNAKE_CASE). 프론트 분기의 유일한 근거 |
| `error.message` | string | O | 사용자에게 그대로 보여줄 수 있는 문구. **내부 정보(스택·쿼리·경로·저장소 구조) 노출 금지** |
| `error.fields` | array \| null | X | 400 / 422 검증 실패 시에만. 프론트가 입력 필드에 매핑한다. `name` 은 요청 파라미터명 또는 헤더명 |
| `error.traceId` | string | O | 로그 상관관계 ID. 500 문의 시 사용자가 그대로 전달한다 |

### 2.1 공통 에러코드

| HTTP | 에러코드 | 발생 조건 | 프론트 처리 |
|---|---|---|---|
| 400 | `VALIDATION_FAILED` | 요청 파라미터·헤더가 제약 위반 | `error.fields` 를 입력 필드에 매핑해 표시 |
| 401 | `UNAUTHENTICATED` | 세션 없음·만료 | `/login?returnUrl=<현재경로>` 로 이동 (§7.7 — 현재 프론트에 미구현) |
| 403 | `FORBIDDEN` | 인증됐으나 역할 부족 (존재 은닉이 무의미한 리소스 — §3.2) | 조회는 FS-003-EL-014 배너, 쓰기는 각 요소의 실패 처리 |
| 403 | `CSRF_TOKEN_INVALID` | 쓰기 요청의 `X-CSRF-Token` 헤더 누락·불일치 (§3.3) | 쓰기 실패 처리와 동일 |
| 404 | `NOT_FOUND` | 경로 리소스 없음 (또는 은닉 정책상 403 대신 404 — §3.2) | 목록으로 복귀 + 안내 |
| 404 | `MEMBER_NOT_FOUND` | 회원 id 가 없거나 삭제됨, 또는 회원 도메인 읽기 권한 없음 (§3.2 은닉) | 어댑터는 이 코드를 `Error('회원을 찾을 수 없습니다')` 로 변환한다 (§6.1) |
| 409 | `CONFLICT` | 동시 수정·중복 생성 | 최신 상태 재조회 후 재시도 안내 |
| 422 | `UNPROCESSABLE` | 형식은 맞으나 상태 위반 | 상태별 안내 문구 표시 |
| 429 | `RATE_LIMITED` | 레이트리밋 초과. `Retry-After`(초) 헤더를 함께 반환한다 | `Retry-After` 초 뒤 재시도 안내 |
| 500 | `INTERNAL_ERROR` | 서버 내부 오류 | 일반 문구 + `traceId` 표시. **원인 노출 금지** |
| 504 | `REQUEST_TIMEOUT` | 서버 처리 상한 초과 (§3.4) | 500 과 동일한 일반 문구 + `traceId` |

## 3. 인증 · 권한 모델

### 3.1 역할

| 역할 | 설명 | 비고 |
|---|---|---|
| `admin` | 회원 도메인 전체 권한 — 조회 · 내보내기 · 그룹 생성 · 회원 삭제 · 알림 발송 · 비밀번호 변경(BE-004) · 적립금 조정(BE-004) · 메모 저장(BE-004) | 이 문서의 모든 엔드포인트를 호출할 수 있다 |
| `operator` | 조회 계열(목록 · 그룹 목록 · 내보내기 · 상세) + 관리자 메모 저장(BE-004-EP-05) | 그 외 쓰기(그룹 생성 · 회원 삭제 · 알림 발송 · 비밀번호 변경 · 적립금 조정)는 **403** |
| 회원 도메인 권한이 없는 인증 관리자 | 회원 리소스에 접근할 수 없는 그 밖의 역할 | 개별 회원 리소스는 **404** 은닉, 컬렉션 리소스는 **403** (§3.2) |

> **프론트에는 역할 분기가 없다** (FS-003 §4.1 · FS-004 §4.1). 요소가 역할에 따라 숨거나 비활성되지 않는다. 권한 강제는 전적으로 서버의 책임이며, 프론트는 403 을 다른 실패와 같은 문구로 표시한다.

### 3.2 403 vs 404 은닉 정책

원칙 두 줄로 확정한다.

1. **컬렉션 리소스의 존재는 비밀이 아니다** → 권한 부족 시 `403 FORBIDDEN`.
2. **개별 회원 리소스의 존재 자체가 개인정보다** → 회원 도메인 **읽기 권한이 없는** 주체에게는 `404 MEMBER_NOT_FOUND` 로 은닉한다. 반대로 **읽기 권한이 있는 주체**(예: `operator`)가 쓰기 권한만 없어 거절될 때는 `403 FORBIDDEN` 을 준다 — 이미 회원의 존재를 아는 주체에게 존재를 숨기는 것은 의미가 없고, 정당한 운영자가 권한 문제를 인지하지 못하게 만든다.

| 리소스 | 권한 부족 시 응답 | 근거 |
|---|---|---|
| `GET /api/members` (컬렉션) | 403 `FORBIDDEN` | 회원 컬렉션의 존재는 비밀이 아니다. 열거할 개별 id 가 노출되지 않는다 |
| `GET /api/members/export` (컬렉션) | 403 `FORBIDDEN` | 위와 동일 |
| `GET /api/member-groups` (컬렉션) | 403 `FORBIDDEN` | 그룹 컬렉션의 존재는 비밀이 아니다 |
| `POST /api/member-groups` | 403 `FORBIDDEN` | 컬렉션 쓰기. 은닉할 개별 리소스가 없다 |
| `DELETE /api/members/:id` | `operator` → 403 `FORBIDDEN` / 회원 도메인 읽기 권한 없음 → 404 `MEMBER_NOT_FOUND` | `operator` 는 목록·상세에서 이미 이 회원을 본다. 읽기 권한이 없는 주체에게는 회원 id 열거를 차단한다 |
| `POST /api/members/:id/notifications` | `operator` → 403 `FORBIDDEN` / 회원 도메인 읽기 권한 없음 → 404 `MEMBER_NOT_FOUND` | 위와 동일 |
| `GET /api/members/:id` (BE-004) | 회원 도메인 읽기 권한 없음 → 404 `MEMBER_NOT_FOUND` | 회원 id 열거를 막는다. 존재 여부 자체가 개인정보다. 프론트는 이 코드를 FS-004-EL-012('회원을 찾을 수 없습니다.')로 표시한다 |

> **FS-003 §7 #8 · FS-004 §7 #7 미결 사항을 이 표로 확정한다.** FS-004-EL-012 는 '회원을 찾을 수 없습니다.' 문구로 **권한 부족의 은닉 응답을 겸한다**.

### 3.3 CSRF

세션 쿠키 기반 인증이므로 상태 변경 요청(POST · PUT · PATCH · DELETE)은 `X-CSRF-Token` 헤더를 요구한다. 누락·불일치 시 `403 CSRF_TOKEN_INVALID`. 프론트 어댑터의 **함수 시그니처는 바뀌지 않는다** — 어댑터 함수 본문이 헤더를 싣는다(§6.1).

### 3.4 타임아웃 상한

| 계열 | 서버 처리 상한 | 초과 시 | 프론트 상한(권고) |
|---|---|---|---|
| 조회(목록 · 그룹 목록 · 상세) | 5초 | 504 `REQUEST_TIMEOUT` | 10초 (현재 프론트에 상한 없음 — §7.7) |
| 내보내기 조회 | 30초 | 504 `REQUEST_TIMEOUT` | 60초 (현재 프론트에 상한 없음 — §7.7) |
| 쓰기(그룹 생성 · 회원 삭제 · 알림 발송) | 5초 | 504 `REQUEST_TIMEOUT` | 10초 (현재 프론트에 상한 없음 — §7.7) |

서버 상한이 프론트 권고 상한보다 항상 작다 — 서버가 먼저 응답을 확정하므로 프론트가 결과를 모른 채 끊는 구간이 없다.

## 4. 엔드포인트 명세

### BE-003-EP-01 · 회원 목록 조회

| 항목 | 내용 |
|---|---|
| 근거 (FS) | FS-003-EL-003, FS-003-EL-003.1, FS-003-EL-003.2, FS-003-EL-003.3, FS-003-EL-003.4, FS-003-EL-004.1, FS-003-EL-004.2, FS-003-EL-007, FS-003-EL-008, FS-003-EL-010, FS-003-EL-011, FS-003-EL-014, FS-003-EL-014.1, FS-003-EL-015, FS-003-EL-015.1, FS-003-EL-015.2, FS-003-EL-015.3 |
| 메서드 · 경로 | `GET /api/members` |
| 권한 | `admin`, `operator` |
| 멱등성 | 멱등 (GET). 재시도 안전. FS-003-EL-014.1('다시 시도')은 같은 조건으로 같은 요청을 다시 보낸다 |
| 페이징 | offset 방식. `page` 1-base, `size` 기본 10 · 상한 100 (프론트 `PAGE_SIZE = 10`) |
| 레이트리밋 | 관리자당 분당 120회 |

**요청 — 쿼리**

| 이름 | 타입 | 필수 | 제약 | 설명 |
|---|---|---|---|---|
| `tier` | string | X | `all` \| `normal` \| `vip` \| `vvip` (기본 `all`) | 등급 필터. `all` 은 등급 조건 없음 |
| `groupId` | string | X | 그룹 id 또는 `all` (기본 `all`). 1–64자 | 그룹 필터. `all` 은 그룹 조건 없음 (프론트 상수 `GROUP_ALL = 'all'`) |
| `keyword` | string | X | 0–50자. 서버가 앞뒤 공백을 제거한다. 빈 문자열이면 조건 없음 | 닉네임 **또는** 계정에 대한 대소문자 무시 부분 일치 |
| `page` | number | X | 정수 ≥ 1 (기본 1) | 페이지 번호 |
| `size` | number | X | 정수 1–100 (기본 10) | 페이지 크기 |

- `tier` 와 `groupId` 가 둘 다 `all` 이 아니면 **AND** 로 결합한다 (FS-003-EL-007).
- `page` 가 총 페이지 수를 넘으면 `members` 는 빈 배열, `total` 은 조건에 걸린 실제 건수를 반환한다. **서버는 페이지를 자동 보정하지 않는다** (FS-003 §7 #6 — 프론트가 빈 상태 FS-003-EL-013 을 표시한다).
- `keyword` 가 50자를 넘으면 400. 프론트에는 길이 제약이 없으므로(FS-003-EL-008) 어댑터가 50자로 자르지 않고 그대로 보내며, 400 은 조회 실패(FS-003-EL-014)로 표시된다 — §7.7 에 변경 요청 로 기록한다.

**응답 200** — `MemberListResult`

```json
{
  "members": [
    {
      "id": "M-00001",
      "nickname": "명재우",
      "account": "user1@naver.com",
      "tier": "normal",
      "groupId": "none",
      "group": "그룹 없음",
      "joinedAt": "2026-03-02",
      "joinedAtIso": "2026-03-02T09:14:00+09:00",
      "points": 12000,
      "activity": { "posts": 3, "comments": 12, "reviews": 1, "inquiries": 0 },
      "totalPurchase": 480000,
      "memo": ""
    }
  ],
  "counts": { "all": 497, "normal": 374, "vip": 93, "vvip": 30 },
  "groupCounts": { "none": 492, "free-shipping": 5 },
  "total": 42
}
```

| 필드 | 타입 (types.ts) | 설명 |
|---|---|---|
| `members` | `readonly Member[]` | 현재 페이지의 회원. 필드명·타입이 `Member` 와 일치한다 |
| `counts` | `TierCounts` = `Record<'all'\|'normal'\|'vip'\|'vvip', number>` | 등급별 **전체** 건수. **검색·그룹 필터와 무관하게 삭제되지 않은 전체 회원 기준이다** (FS-003-EL-003 비고 — 좌측 필터 배지가 검색할 때마다 흔들리면 안 된다) |
| `groupCounts` | `GroupCounts` = `Record<string, number>` (키 = 그룹 id) | 그룹별 **전체** 건수. **검색·등급 필터와 무관하게 전체 회원 기준이다** (FS-003-EL-004.2 비고). `GET /api/member-groups` 가 돌려주는 모든 그룹 id 를 키로 포함하며, 회원이 0명인 그룹도 `0` 으로 채운다 |
| `total` | number | 필터·검색을 **적용한 뒤**의 전체 건수. 페이지네이션(FS-003-EL-015)과 조회 요약(FS-003-EL-010)이 쓴다 |

`Member.joinedAt` 은 `YYYY-MM-DD`, `Member.joinedAtIso` 는 오프셋을 포함한 ISO 8601 date-time 이다 — 상대 시각 계산(FS-003-EL-011.7)에 시·분이 필요하다. `Member.memo` 는 목록에서 hover title 로만 쓰인다(FS-003-EL-011.11).

**에러 목록**

| HTTP | 에러코드 | 조건 |
|---|---|---|
| 400 | `VALIDATION_FAILED` | `tier` / `groupId` enum·형식 위반, `keyword` > 50자, `page` < 1, `size` 범위 밖. `error.fields[].name` 은 쿼리 파라미터명 |
| 401 | `UNAUTHENTICATED` | 세션 없음·만료 |
| 403 | `FORBIDDEN` | 역할 부족 (컬렉션은 은닉하지 않는다 — §3.2) |
| 429 | `RATE_LIMITED` | 분당 120회 초과 |
| 500 | `INTERNAL_ERROR` | 내부 오류 |
| 504 | `REQUEST_TIMEOUT` | 서버 처리 5초 초과 |

### BE-003-EP-02 · 회원 그룹 목록 조회

| 항목 | 내용 |
|---|---|
| 근거 (FS) | FS-003-EL-004 |
| 메서드 · 경로 | `GET /api/member-groups` |
| 권한 | `admin`, `operator` |
| 멱등성 | 멱등 (GET). 재시도 안전 |
| 페이징 | 없음 — 전량 반환. **그룹 수 상한 200개** (§7.5 판정). 상한에 도달하면 BE-003-EP-03 이 422 로 생성을 거절하므로 이 응답이 200개를 넘지 않는다 |
| 레이트리밋 | 관리자당 분당 60회 |

**요청** — 파라미터 없음.

**응답 200** — `readonly MemberGroup[]`

```json
[
```

**삭제 정책 — 소프트 삭제 (§7.1 판정)**

- 회원 레코드를 물리적으로 지우지 않고 **탈퇴 상태로 전이**한다. 주문·문의·구매평 이력이 회원을 참조하고 있고, 전자상거래 관련 기록은 법정 보존 대상이다 — 물리 삭제는 이 이력을 고아로 만든다.

- 개인정보(계정 · 이름 · 연락처 · 주소 · 생년월일 · 최종 로그인 IP)는 탈퇴 후 **30일 경과 시 비식별 처리**한다. 30일은 분쟁 대응 창구이며 **복구 창구가 아니다**.
- **되돌릴 수 없다.** 복구 엔드포인트는 이 계약에 존재하지 않는다 — FS-003-EL-017.1 · FS-004-EL-014.1 이 사용자에게 '이 작업은 되돌릴 수 없습니다.' 라고 고지한다.
- 삭제 행위는 감사 로그에 남긴다 (행위자 관리자 id · 대상 회원 id · 시각 · 행위자 IP · `traceId`).

- 큐는 실패한 발송을 최대 3회(지수 백오프) 재시도하고, 3회 실패하면 실패 이력으로 확정한다.

## 7. 핵심 판정 (근거를 남긴다)

### 7.1 회원 삭제 — 소프트 삭제로 확정한다

물리 삭제를 채택하지 않는다. 회원은 주문·문의·구매평의 참조 대상이고 그 기록은 법정 보존 대상이다 — 회원 행을 지우면 이력이 고아가 되거나 함께 지워진다. 따라서 **탈퇴 상태 전이(소프트 삭제)** 로 확정하고, 조회 대상에서는 즉시 제외하며, 개인정보는 30일 뒤 비식별 처리한다. **복구 엔드포인트를 만들지 않는다** — FS-003-EL-017.1 · FS-004-EL-014.1 이 '되돌릴 수 없습니다.' 라고 고지하고 있고, 복구 경로를 열면 그 고지가 거짓이 된다. 저장 방식(상태 컬럼·별도 보관)은 백엔드의 결정이며 이 문서가 정하지 않는다 — 계약이 정하는 것은 **삭제 후 조회에서 404 가 나오고, 재삭제가 204 로 멱등하며, 이력이 보존된다**는 관측 가능한 동작뿐이다.

### 7.2 내보내기 — 동기 응답 + 50,000행 상한. 비동기 큐를 v1.0 에 넣지 않는다

- 현재 데이터는 497명이다. 상한 50,000행에서 응답 크기는 회원 1건당 약 300바이트 × 50,000 ≈ 15MB(gzip 후 약 2MB)이며 브라우저 메모리에서 CSV 직렬화가 가능하다. 서버 처리 상한 30초 안에 끝난다.
- **비동기(작업 큐 + 다운로드 링크)를 채택하지 않는다.** 프론트는 `fetchMembersForExport(...): Promise<readonly Member[]>` 로 **회원 배열을 직접 받아** `toCsv` 로 파일을 만든다. 비동기로 바꾸면 반환 타입(작업 id) · 폴링 · 다운로드 링크 UI · 완료 알림이 모두 필요해져 **어댑터 시그니처와 화면 흐름이 함께 바뀐다** — 이는 프론트 구현 · UI 기획 쪽 변경 요청 사안이며, 프론트가 이미 기대하는 모양을 뒤집는 것이다. 현재 규모에서 그럴 근거가 없다.
- 50,000행을 넘으면 **422 `EXPORT_TOO_LARGE`** 로 거절하고 범위를 좁히라고 안내한다. 잘라서 일부만 주지 않는다 — 조용히 잘린 CSV 는 잘못된 운영 판단으로 이어진다.
- 50,000행 초과가 실제 운영에서 발생하면 그때 **비동기 내보내기를 별도 계약(FS 개정 + BE 신규 엔드포인트)으로 승격**한다. 이 문서는 그 시점까지의 계약이다.
- **CSV 직렬화는 프론트에 남긴다.** 서버가 CSV 를 내려주면 `toCsv` 가 사라지고 응답 Content-Type 이 바뀌며 화면의 Blob 다운로드 코드가 바뀐다. 파일명·BOM·7열 구성이 이미 FS-003-EL-009 에 고정돼 있고 프론트에 구현돼 있다 — 서버로 옮길 이득이 없다.

### 7.3 그룹명 중복 — 같은 유형 안에서 금지한다 (409)

FS-003 §7 #7 미결 사항을 확정한다. **같은 `type` 안에서 정규화(앞뒤 공백 제거 + 대소문자 무시)한 그룹명의 중복을 금지**하고 409 `GROUP_NAME_DUPLICATED` 를 반환한다. 근거: 좌측 필터(FS-003-EL-004.2)는 그룹을 **표시명으로만** 구분해 보여준다. 동명 그룹이 둘이면 관리자가 어느 쪽을 고르는지 화면에서 알 수 없다. `type` 이 다르면(회원 그룹 vs 운영진 그룹) 소비하는 화면이 달라 혼동이 없으므로 같은 이름을 허용한다.

### 7.4 운영진 그룹 — 여기서 만들 수 있다. 다만 회원 그룹 목록에는 나타나지 않는다

FS-003-EL-016.4 는 '운영진 그룹' 을 선택지로 고정했고 `types.ts` 의 `GroupType` 은 `'member' | 'staff'` 다. 따라서 **`POST /api/member-groups` 는 `type=staff` 를 받는다** — 프론트가 보낼 수 있는 값을 서버가 거절하면 화면에 도달 불가능한 선택지가 남는다.

그러나 **`GET /api/member-groups` 는 `type=member` 만 반환한다.** 근거: 이 엔드포인트의 유일한 소비자는 회원 목록의 좌측 **그룹 필터**이고, 운영진 그룹은 회원을 담지 않는다(FS-004 §1.1 — '운영진 그룹은 관리자 화면에서만 쓰이며, 회원 상세에는 나타나지 않는다'). 결과적으로 운영진 그룹을 만들면 성공 배너는 뜨지만 사이드바 항목은 늘지 않는다 — 이는 **의도된 동작**이며, 사용자에게 혼란을 줄 수 있으므로 §7.7 에 변경 요청 로 기록한다(모달 도움말 FS-003-EL-016.5 가 이미 이 사실을 설명한다).

### 7.5 그룹 수 상한 — 200개

FS-003 §7 #10(그룹 수 상한 없음 → 사이드바가 세로로 늘어난다)을 서버 측에서 막는다. 상한 200개에 도달하면 생성이 422 `GROUP_LIMIT_EXCEEDED` 로 거절된다. 200 은 사이드바가 스크롤 없이 다룰 수 있는 규모를 넘어서지만, 운영상 정당한 그룹 개수를 제약하지 않는 선이다. 사이드바 표현(스크롤·접기)은 UI 기획 의 소관이다.

### 7.6 알림 발송 — 비동기(큐). 202 는 접수만 보장한다

동기 발송을 채택하지 않는다. 외부 메일 게이트웨이의 지연이 관리자 화면의 응답 시간이 되며, 서버 처리 상한 5초를 지킬 수 없다. **202 Accepted 로 접수만 보장**하고 큐가 발송한다(실패 시 지수 백오프로 최대 3회 재시도).

**발송 실패를 이 엔드포인트가 알리지 않는다**는 것이 이 판정의 대가다. 회원 화면에는 발송 결과를 표시할 요소가 없고(FS-003 §3 · FS-004 §3), 프론트는 실패 문구조차 표시하지 않는다(FS §7). 발송 결과 이력은 마케팅 도메인의 관심사다 — 이 한계를 §7.7 에 미결로 남긴다. 다만 **발송 전에 판정 가능한 거절 사유**(수신 미동의 · 탈퇴 회원)는 202 를 주기 전에 422 · 404 로 즉시 응답한다.
