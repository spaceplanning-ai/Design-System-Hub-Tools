# FS-007. 고객 설정 (회원 등급 정책)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 목적 | 회원 등급(일반/VIP/VVIP)의 승급 조건(누적 구매금액)과 등급별 할인율, 그리고 등급 산정 기준(집계 기간·강등 허용·재계산 시점)을 정하고 저장한다. 저장 전에는 현재 회원이 이 정책으로 어떻게 나뉘는지 미리 본다 |
| 역할(주 사용자) | 관리자 (구현에 역할 분기 없음 — §4.1 참조) |
| 진입 경로 | 좌측 GNB > 사용자 관리 > 고객 설정 (`nav-config.ts`, `to: '/users/settings'`) |
| 관련 화면 | FS-003 회원 관리(등급 배지·필터가 이 정책의 결과를 소비) / FS-005 관리자 관리(사이드바 안내문 링크 대상) |
| 대응 SCR | SCR-007 (미작성 — §7) |
| 구현 경로 | `apps/admin/src/pages/customer-settings/CustomerSettingsPage.tsx` 및 `components/**`. 검증 규칙은 `validation.ts`(zod) · 모델·파서는 `types.ts` · 분포 계산은 `distribution.ts` |

### 1.1 이 화면의 설계 원칙 (요구사항 — 명세에 고정한다)

| 원칙 | 내용과 이유 |
|---|---|
| **검증의 정본은 모델이다 (화면이 아니라)** | 승급 조건 단조 증가·할인율 범위 규칙은 입력 컨트롤이 아니라 **zod 스키마**(`validation.ts` `tierPolicySchema`)가 강제한다. 화면은 입력을 막지 않고, 저장 시점에 모델이 정책을 만들어 주지 못하면 저장을 거부한다 |
| **에러(저장 거부)와 경고(저장 허용)를 나눈다** | **에러** — 금액·할인율이 정수로 안 읽힘, 승급 조건이 등급 오를수록 커지지 않음 → 저장 거부(zod). **경고** — 등급이 오르는데 할인율이 작아짐 → 저장은 허용하되 알린다(`policyWarnings`, zod 밖). 이 경계가 이 화면의 핵심이다 |
| **입력 중에는 숫자만, 포맷은 blur 에서** | 승급 조건 입력은 타이핑 중 숫자만 받고(`sanitizeDigits`), 포커스를 잃을 때 천 단위 구분을 붙인다(`formatAmountOnBlur`). 30자 붙여넣기가 조용히 잘리지 않는다 |
| **정책 저장은 파괴적이다 — 확인을 거친다** | 정책 변경은 전 회원의 등급·할인율을 움직이므로 클릭 한 번으로 저장하지 않고 확인 다이얼로그(intent="update")를 거친다. 저장하지 않은 채 이탈하면 discard 가드가 막는다 |
| **분포는 저장 전 미리보기다** | 우측 '현재 등급 분포' 는 지금 입력한 정책을 적용했을 때의 예상값이며, 저장 전에는 실제 등급이 바뀌지 않는다. 계산은 브라우저에서 순수 함수로 돈다(`distribution.ts`) |

| 요청 타임아웃 | 프론트 타임아웃 상한이 구현에 없다. 저장 요청의 abort 는 확인 다이얼로그를 닫거나 화면을 벗어날 때 발생한다 — §7 미결 사항(상한 자체는 미정) |
| 중복 제출 | 저장 버튼(FS-007-EL-004.1)과 확인 '저장' 버튼(FS-007-EL-005.1)은 요청 중 비활성되어 차단된다 |
| 검증의 자리 | 입력 유효성(에러)은 표 안 인라인(FS-007-EL-001.4), 저장 거부 요약은 토스트(FS-007-EL-009), 저장 요청 실패는 다이얼로그 안 배너(FS-007-EL-005.3), 조회 실패는 인라인 배너(FS-007-EL-008). 경고(할인율 역전)는 저장을 막지 않고 별도 Alert(FS-007-EL-001.5) |
| 권한 없음 | **이 화면의 구현에는 역할 기반 분기가 없다.** 서버가 권한 부족을 응답하면 조회는 FS-007-EL-008 로, 저장은 FS-007-EL-005.3 으로 떨어진다. 은닉 정책은 백엔드 명세(BE-007)가 확정한다 |
| 서버 재검증 | **프론트 검증은 UX 이지 보증이 아니다** — `saveTierPolicy` 주석이 명시한다. 서버도 같은 규칙(승급 조건 단조 증가·할인율 0~100)을 다시 검증해야 한다(BE-007 §7) |
